INTRO

Pattern Theory & When to Use Each

Two Pointers and Sliding Window are complementary techniques for solving array/string problems in linear time. Understanding the difference unlocks hundreds of interview problems.

Two Pointers Pattern

Use two pointers when you need to:

Two Pointers Strength
Find pairs/triplets that meet a condition
Validate or process from both ends
Exploit sorted/monotonic properties
Reverse or rearrange in-place
Skip redundant comparisons
Two Pointers Weakness
Requires sorted data (usually)
Can't maintain a "window" state easily
Harder to find variable-length subarrays

Sliding Window Pattern

Use sliding window when you need to:

Sliding Window Strength
Find optimal subarray/substring
Maintain a running state (hash, count)
Work on unsorted data
Expand and shrink dynamically
O(n) time, single pass often
Sliding Window Weakness
Not ideal for finding k-element pairs
Fixed-size windows are simpler
Variable-size requires shrink logic

Core Templates in C#

Two Pointers (Fixed):

csharp
int left = 0, right = arr.Length - 1;
while (left < right) {
    if (condition(arr[left], arr[right])) {
        // Found or process pair
        left++;
    } else {
        right--;
    }
}

Sliding Window (Variable):

csharp
int left = 0;
var windowState = new Dictionary<char, int>();

for (int right = 0; right < s.Length; right++) {
    // Add to window
    if (!windowState.ContainsKey(s[right]))
        windowState[s[right]] = 0;
    windowState[s[right]]++;

    // Shrink if needed
    while (!isValid(windowState)) {
        windowState[s[left]]--;
        if (windowState[s[left]] == 0)
            windowState.Remove(s[left]);
        left++;
    }

    // Process valid window
}
💡
Golden Rule: Two pointers converge, sliding window expands then shrinks. Think about whether you're searching for a specific relationship or optimizing a subarray.
1. Valid Palindrome
Check if string is palindrome (alphanumeric only, case-insensitive)
Easy Amazon Google
01
Time
O(n)
Space
O(1)
Pattern
Two Pointers

Problem Statement

Given a string s, determine if it is a valid palindrome, considering only alphanumeric characters and ignoring cases. Return true if palindrome, false otherwise.

Approach Comparison

Brute Force
Filter to alphanumeric, lowercase
Compare string with reverse
O(n) time, O(n) space
Optimal (Two Pointers)
Start from both ends
Skip non-alphanumeric on-the-fly
O(n) time, O(1) space

Algorithm

1
Initialize left = 0, right = s.Length - 1
2
While left < right, skip non-alphanumeric chars from both sides
3
Compare lowercase versions of s[left] and s[right]
4
If mismatch, return false; otherwise move both pointers inward
5
Return true if loop completes

C# Solution

csharp
public bool IsPalindrome(string s) {
    int left = 0, right = s.Length - 1;

    while (left < right) {
        // Skip non-alphanumeric from left
        while (left < right && !char.IsLetterOrDigit(s[left])) {
            left++;
        }
        // Skip non-alphanumeric from right
        while (left < right && !char.IsLetterOrDigit(s[right])) {
            right--;
        }

        // Compare lowercase chars
        if (char.ToLower(s[left]) != char.ToLower(s[right])) {
            return false;
        }
        left++;
        right--;
    }
    return true;
}

Example Trace

Input: "A man, a plan, a canal: Panama"
left=0 (A), right=32 (a) → Match ✓
left=1 (m), right=30 (m) → Match ✓
left=2 (a), right=28 (a) → Match ✓
... continues to center ...
Returns true
Input: "race a car"
left=0 (r), right=8 (r) → Match ✓
left=1 (a), right=6 (a) → Match ✓
left=2 (c), right=4 (c) → Match ✓
left=3 (e), right=3 → left ≥ right, exit
Returns false (not palindrome)

Edge Cases

⚠
Empty string or only special chars: Should return true. Single char: Always true. Mixed case & digits: Must be case-insensitive.

FAANG Interview Tip

💡
Many interviewers will ask "Can you do it without modifying the string?" This solution does exactly that—in-place two-pointer validation. Mention O(1) space upfront to impress.
2. Two Sum II - Sorted Array
Find two numbers that add to a target in sorted array; return 1-indexed positions
Medium Meta Apple
02
Time
O(n)
Space
O(1)
Pattern
Two Pointers

Problem Statement

Given a 1-indexed sorted array and a target integer, find indices of two numbers that add up to target. Exactly one solution exists, and you cannot use same element twice. Return array of indices in increasing order.

Algorithm

1
Place pointers at start (left=0) and end (right=n-1)
2
Calculate sum of elements at both pointers
3
If sum equals target, return [left+1, right+1] (1-indexed)
4
If sum is too small, move left pointer right; if too large, move right left
5
Repeat until found

C# Solution

csharp
public int[] TwoSum(int[] numbers, int target) {
    int left = 0, right = numbers.Length - 1;

    while (left < right) {
        int sum = numbers[left] + numbers[right];

        if (sum == target) {
            return new int[] { left + 1, right + 1 };
        } else if (sum < target) {
            left++;
        } else {
            right--;
        }
    }

    return new int[] {};
}

Example Trace

Input: [2,7,11,15], target=9
left=0 (2), right=3 (15): sum=17 > 9, right--
left=0 (2), right=2 (11): sum=13 > 9, right--
left=0 (2), right=1 (7): sum=9 ✓
Return [1, 2]
Input: [2,3,4], target=6
left=0 (2), right=2 (4): sum=6 ✓
Return [1, 3]
💡
Why not use hash map here? Sorted array is a huge hint—leverage it! Two pointers is O(n) time, O(1) space. Hash map would be O(n) time, O(n) space.
3. 3Sum
Find all unique triplets in array that sum to zero
Medium Google Amazon
03
Time
O(n²)
Space
O(1)
Pattern
Sort + Two Pointers

Problem Statement

Given an integer array nums, return all unique triplets that sum to 0. Do not include duplicate triplets in result.

Key Insight

Sort first to enable duplicate skipping and two-pointer search. For each fixed element, find two others using two pointers on remaining sorted array.

C# Solution

csharp
public IList<IList<int>> ThreeSum(int[] nums) {
    var result = new List<IList<int>>();
    Array.Sort(nums);

    for (int i = 0; i < nums.Length - 2; i++) {
        // Skip duplicates for first element
        if (i > 0 && nums[i] == nums[i - 1]) continue;

        // If smallest possible sum > 0, no solution
        if (nums[i] > 0) break;

        // Two sum on remaining elements
        int left = i + 1, right = nums.Length - 1;
        int target = -nums[i];

        while (left < right) {
            int sum = nums[left] + nums[right];

            if (sum == target) {
                result.Add(new List<int> { nums[i], nums[left], nums[right] });

                // Skip duplicates
                while (left < right && nums[left] == nums[left + 1]) left++;
                while (left < right && nums[right] == nums[right - 1]) right--;

                left++;
                right--;
            } else if (sum < target) {
                left++;
            } else {
                right--;
            }
        }
    }

    return result;
}

Example Trace

Input: [-1, 0, 1, 2, -1, -4]
After sort: [-4, -1, -1, 0, 1, 2]

i=0 (nums[0]=-4): target=4, no valid pairs
i=1 (nums[1]=-1): target=1
  left=2, right=5: -1+2=1 ✓ → [-1, -1, 2]
i=2: Skip (duplicate -1)
Result: [[-1, -1, 2], [-1, 0, 1]]
💡
FAANG Focus: Emphasize duplicate handling. Many candidates produce correct values but with duplicates. Show you understand why skipping matters.
4. Container With Most Water
Find two lines that form container holding maximum water
Medium Microsoft Bloomberg
04
Time
O(n)
Space
O(1)
Pattern
Two Pointers Greedy

Problem Statement

Given array height representing elevation map, where each bar's width is 1 and height is given value, find two lines forming container with maximum water area. Area = width × min(height[i], height[j]).

Greedy Insight

Start with maximum width (both pointers at edges). Water volume is limited by shorter bar. Moving the taller bar inward can't increase volume (width decreases, height at best stays same). So move the shorter bar.

C# Solution

csharp
public int MaxArea(int[] height) {
    int maxArea = 0;
    int left = 0, right = height.Length - 1;

    while (left < right) {
        // Width = right - left
        int width = right - left;
        // Height = min of both bars
        int h = Math.Min(height[left], height[right]);
        int area = width * h;
        maxArea = Math.Max(maxArea, area);

        // Move shorter bar inward
        if (height[left] < height[right]) {
            left++;
        } else {
            right--;
        }
    }

    return maxArea;
}

Example Trace

Input: [1,8,6,2,5,4,8,3,7]
left=0 (1), right=8 (7): width=8, h=min(1,7)=1, area=8
Move left (1 is shorter)
left=1 (8), right=8 (7): width=7, h=min(8,7)=7, area=49 ✓
Move right (7 is shorter)
Continue until convergence
Max area = 49
💡
This is a classic greedy problem. The insight is counterintuitive: moving the taller line doesn't help. Explaining the greedy logic clearly is what sets top candidates apart.
5. Trapping Rain Water
Calculate water trapped between elevation bars after raining
Hard Meta Goldman Sachs
05
Time
O(n)
Space
O(1)
Pattern
Two Pointers

Problem Statement

Given elevation map (array), where each bar has width 1, compute total units of water trapped after raining. Water level at position i = min(max_left, max_right) - height[i].

Key Insight

Track max height seen from left and right. At each position, water trapped depends on both sides. Use two pointers: advance the side with smaller max height to guarantee trapped water calculation is correct.

C# Solution

csharp
public int Trap(int[] height) {
    int trapped = 0;
    int left = 0, right = height.Length - 1;
    int leftMax = 0, rightMax = 0;

    while (left <= right) {
        if (height[left] < height[right]) {
            // Left side is limiting factor
            if (height[left] >= leftMax) {
                leftMax = height[left];
            } else {
                trapped += leftMax - height[left];
            }
            left++;
        } else {
            // Right side is limiting factor
            if (height[right] >= rightMax) {
                rightMax = height[right];
            } else {
                trapped += rightMax - height[right];
            }
            right--;
        }
    }

    return trapped;
}

Example Trace

Input: [0,1,0,2,1,0,1,3,2,1,2,1]
left=0, right=11: height[0]=0 < height[11]=1
leftMax becomes 0, trapped += 0
left=1, right=11: height[1]=1 >= height[11]=1
rightMax=1, trapped += 1-1=0
Continue with two pointers moving inward...
Total trapped = 6
⚠
Classic hard interview problem. Many candidates think they need prefix/suffix arrays. The two-pointer O(1) space solution is elegant and impresses interviewers.
6. Longest Substring Without Repeating Characters
Find length of longest substring with no repeating characters
Medium Amazon Google
06
Time
O(n)
Space
O(m)
Pattern
Sliding Window + Hash

Problem Statement

Given string s, find length of longest substring without repeating characters. Return integer length.

Approach

Sliding window expands right, tracking character positions in hash map. When duplicate found, shrink window from left until duplicate removed. Update max length during valid states.

C# Solution

csharp
public int LengthOfLongestSubstring(string s) {
    var charMap = new Dictionary<char, int>();
    int maxLen = 0, left = 0;

    for (int right = 0; right < s.Length; right++) {
        char c = s[right];

        // If char exists and is within current window
        if (charMap.ContainsKey(c) && charMap[c] >= left) {
            // Move left past duplicate
            left = charMap[c] + 1;
        }

        // Update or add char's latest position
        charMap[c] = right;

        // Update max length
        maxLen = Math.Max(maxLen, right - left + 1);
    }

    return maxLen;
}

Example Trace

Input: "abcabcbb"
right=0 (a): charMap={a:0}, maxLen=1
right=1 (b): charMap={a:0, b:1}, maxLen=2
right=2 (c): charMap={a:0, b:1, c:2}, maxLen=3
right=3 (a): Duplicate! left=1, charMap={a:3, b:1, c:2}, maxLen=3
right=4 (b): Duplicate! left=2, maxLen=3
right=5 (c): Duplicate! left=3, maxLen=3
Result = 3 ("abc")
💡
Store character positions, not just presence. This lets you jump the left pointer directly past duplicates instead of incrementing one by one.
7. Minimum Window Substring
Find minimum window substring containing all characters of pattern
Hard Google Amazon
07
Time
O(n + m)
Space
O(1)
Pattern
Sliding Window + Frequency

Problem Statement

Given strings s (source) and t (target), find minimum window substring in s that contains all characters from t (including duplicates). If no such window, return empty string.

Algorithm

1
Count frequency of each char in t
2
Expand right pointer, adding chars to window
3
When window is valid (contains all chars), try shrinking from left
4
Track minimum window seen
5
Return minimum window substring

C# Solution

csharp
public string MinWindow(string s, string t) {
    if (t.Length > s.Length) return "";

    var tCount = new Dictionary<char, int>();
    var windowCount = new Dictionary<char, int>();

    // Count chars in t
    foreach (char c in t) {
        if (!tCount.ContainsKey(c)) tCount[c] = 0;
        tCount[c]++;
    }

    int required = tCount.Count;
    int formed = 0;
    int left = 0;
    int minLen = int.MaxValue;
    int minStart = 0;

    for (int right = 0; right < s.Length; right++) {
        char c = s[right];
        if (!windowCount.ContainsKey(c)) windowCount[c] = 0;
        windowCount[c]++;

        // If frequency matches, increment formed count
        if (tCount.ContainsKey(c) && windowCount[c] == tCount[c]) {
            formed++;
        }

        // Try shrinking from left
        while (left <= right && formed == required) {
            int windowLen = right - left + 1;
            if (windowLen < minLen) {
                minLen = windowLen;
                minStart = left;
            }

            // Remove char from left
            char leftChar = s[left];
            windowCount[leftChar]--;
            if (tCount.ContainsKey(leftChar) && windowCount[leftChar] < tCount[leftChar]) {
                formed--;
            }
            left++;
        }
    }

    return minLen == int.MaxValue ? "" : s.Substring(minStart, minLen);
}

Example Trace

Input: s="ADOBECODEBANC", t="ABC"
tCount = {A:1, B:1, C:1}, required=3
Expand right until "ADOBEC" (formed=3)
Try shrinking: "DOBEC" (A removed, formed drops)
Continue expanding: "ADOBECODEBANC"
Find "BANC" at end (length=4)
Result = "BANC"
🔥
Hard Interview Problem: This appears in 90% of top-tier interviews. Practice the two-frequency-map approach thoroughly. Mistakes here kill interviews.
8. Sliding Window Maximum
Find max element in every k-size sliding window
Hard Bloomberg Apple
08
Time
O(n)
Space
O(k)
Pattern
Deque + Sliding Window

Problem Statement

Given array and window size k, slide window across array and return array of max values in each window position.

Key Insight

Use deque to store indices in decreasing order of values. Remove outdated indices (outside window), add new indices maintaining decreasing order. Front of deque always holds current max.

C# Solution

csharp
public int[] MaxSlidingWindow(int[] nums, int k) {
    var deque = new Deque<int>();
    var result = new int[nums.Length - k + 1];
    int idx = 0;

    for (int i = 0; i < nums.Length; i++) {
        // Remove indices outside window
        while (deque.Count > 0 && deque.Front() < i - k + 1) {
            deque.RemoveFront();
        }

        // Remove smaller elements from back
        while (deque.Count > 0 && nums[deque.Back()] < nums[i]) {
            deque.RemoveBack();
        }

        // Add current index
        deque.AddBack(i);

        // When window is full, store max (front of deque)
        if (i >= k - 1) {
            result[idx++] = nums[deque.Front()];
        }
    }

    return result;
}

Example Trace

Input: [1,3,-1,-3,5,3,6,7], k=3
i=0: deque=[0] (value=1)
i=1: deque=[1] (value=3, removed 1)
i=2: deque=[1,2] (value=-1) → window max = 3
i=3: deque=[3] (value=-3, removed 1,2) → window max = 3
i=4: deque=[4] (value=5, removed 3) → window max = 5
Continue...
Result = [3, 3, 5, 5, 6, 7]
⚠
C# doesn't have built-in Deque. Use Queue or LinkedList, or implement your own simple version. Practice the logic, not the language details.
9. Longest Repeating Character Replacement
Find longest substring with same character after at most k replacements
Medium Facebook Amazon
09
Time
O(n)
Space
O(1)
Pattern
Sliding Window

Problem Statement

Given string s and integer k, find length of longest substring with all same character where you can make at most k character replacements.

Approach

Track frequency of characters in window. At each step, check if window is valid: (window_length - max_frequency) <= k. If valid, expand; if not, shrink.

C# Solution

csharp
public int CharacterReplacement(string s, int k) {
    var charCount = new int[26];
    int maxCount = 0;
    int maxLen = 0;
    int left = 0;

    for (int right = 0; right < s.Length; right++) {
        int idx = s[right] - 'A';
        charCount[idx]++;
        maxCount = Math.Max(maxCount, charCount[idx]);

        // If changes needed > k, shrink window
        while ((right - left + 1) - maxCount > k) {
            int leftIdx = s[left] - 'A';
            charCount[leftIdx]--;
            left++;
        }

        maxLen = Math.Max(maxLen, right - left + 1);
    }

    return maxLen;
}

Example Trace

Input: s="ABAB", k=2
right=0 (A): count=[1,0,...], maxLen=1
right=1 (B): count=[1,1,...], maxLen=2
right=2 (A): count=[2,1,...], maxLen=3
right=3 (B): count=[2,2,...], changes=2 ✓, maxLen=4
Result = 4
💡
Use array instead of hash map for 26 uppercase letters. Much faster and O(1) space for alphabet.
10. Permutation in String
Check if string contains permutation of another string as substring
Medium Google Microsoft
10
Time
O(n)
Space
O(1)
Pattern
Fixed-Size Window

Problem Statement

Given strings s1 and s2, return true if s2 contains a permutation of s1 as a substring. Permutation includes any rearrangement.

Approach

Fixed-size sliding window. Window size = s1.length. Compare character frequencies in window with s1. If match, found permutation.

C# Solution

csharp
public bool CheckInclusion(string s1, string s2) {
    if (s1.Length > s2.Length) return false;

    var s1Count = new int[26];
    var windowCount = new int[26];

    // Count chars in s1
    for (int i = 0; i < s1.Length; i++) {
        s1Count[s1[i] - 'a']++;
    }

    // Slide window over s2
    for (int i = 0; i < s2.Length; i++) {
        int idx = s2[i] - 'a';
        windowCount[idx]++;

        // Remove leftmost char if window > s1.Length
        if (i >= s1.Length) {
            windowCount[s2[i - s1.Length] - 'a']--;
        }

        // Check if frequencies match
        if (Matches(s1Count, windowCount)) {
            return true;
        }
    }

    return false;
}

private bool Matches(int[] arr1, int[] arr2) {
    for (int i = 0; i < 26; i++) {
        if (arr1[i] != arr2[i]) return false;
    }
    return true;
}

Example Trace

Input: s1="ab", s2="eidbaooo"
s1Count: a=1, b=1
Window at [0,1]: "ei" ≠ permutation
Window at [1,2]: "id" ≠ permutation
Window at [2,3]: "db" ≠ permutation
Window at [3,4]: "ba" ✓ = permutation of "ab"
Result = true
💡
Fixed-size windows are simpler than variable-size. Useful when you're looking for a specific substring length or permutation.
11. Minimum Size Subarray Sum
Find minimum length subarray with sum >= target
Medium Amazon Google
11
Time
O(n)
Space
O(1)
Pattern
Sliding Window

Problem Statement

Given array of positive integers and target sum, find minimum length of subarray whose sum >= target. Return length or 0 if no such subarray.

C# Solution

csharp
public int MinSubArrayLen(int target, int[] nums) {
    int minLen = int.MaxValue;
    int left = 0;
    int sum = 0;

    for (int right = 0; right < nums.Length; right++) {
        sum += nums[right];

        // Shrink window while sum >= target
        while (sum >= target) {
            minLen = Math.Min(minLen, right - left + 1);
            sum -= nums[left];
            left++;
        }
    }

    return minLen == int.MaxValue ? 0 : minLen;
}

Example Trace

Input: target=7, nums=[2,3,1,2,4,3]
right=0: sum=2 < 7
right=1: sum=5 < 7
right=2: sum=6 < 7
right=3: sum=8 >= 7 → minLen=4, shrink: sum=6
right=4: sum=10 >= 7 → minLen=3 ([4,3,2]), sum=7 >= 7 → minLen=2 ([4,3])
right=5: sum=10 >= 7 → minLen=2 (no improvement)
Result = 2
12. Fruit Into Baskets
Pick maximum fruits with at most 2 basket types
Medium Google
12
Time
O(n)
Space
O(1)
Pattern
Sliding Window (At Most K)

Problem Statement

Given array of fruit types in row, pick maximum fruits using at most 2 baskets. Each basket can hold unlimited fruits of one type. Must pick contiguous fruits.

Approach

Sliding window: maintain at most 2 distinct characters in window. Expand right, shrink left when more than 2 types encountered. Track maximum window size.

C# Solution

csharp
public int TotalFruit(int[] fruits) {
    var basket = new Dictionary<int, int>();
    int maxFruits = 0;
    int left = 0;

    for (int right = 0; right < fruits.Length; right++) {
        if (!basket.ContainsKey(fruits[right]))
            basket[fruits[right]] = 0;
        basket[fruits[right]]++;

        // Shrink if more than 2 basket types
        while (basket.Count > 2) {
            basket[fruits[left]]--;
            if (basket[fruits[left]] == 0)
                basket.Remove(fruits[left]);
            left++;
        }

        maxFruits = Math.Max(maxFruits, right - left + 1);
    }

    return maxFruits;
}

Example Trace

Input: [1,2,1,2,3]
right=0 (1): basket={1:1}, maxFruits=1
right=1 (2): basket={1:1, 2:1}, maxFruits=2
right=2 (1): basket={1:2, 2:1}, maxFruits=3
right=3 (2): basket={1:2, 2:2}, maxFruits=4
right=4 (3): basket has 3 types, shrink left
After shrinking: basket={2:2, 3:1}, maxFruits=4
Result = 4
13. Subarrays with K Different Integers
Count subarrays with exactly k distinct integers
Hard Google Amazon
13
Time
O(n)
Space
O(k)
Pattern
AtMost(K) Technique

Problem Statement

Given array and integer k, count subarrays containing exactly k distinct integers.

Key Insight: AtMost(K) Technique

Exactly(K) = AtMost(K) - AtMost(K-1). Write helper to count subarrays with at most k distinct integers, then subtract.

C# Solution

csharp
public int SubarraysWithKDistinct(int[] nums, int k) {
    return AtMostKDistinct(nums, k) - AtMostKDistinct(nums, k - 1);
}

private int AtMostKDistinct(int[] nums, int k) {
    if (k == 0) return 0;

    var freq = new Dictionary<int, int>();
    int count = 0;
    int left = 0;

    for (int right = 0; right < nums.Length; right++) {
        if (!freq.ContainsKey(nums[right]))
            freq[nums[right]] = 0;
        freq[nums[right]]++;

        // Shrink window if more than k distinct
        while (freq.Count > k) {
            freq[nums[left]]--;
            if (freq[nums[left]] == 0)
                freq.Remove(nums[left]);
            left++;
        }

        // All subarrays ending at right with at most k distinct
        count += right - left + 1;
    }

    return count;
}

Example Trace

Input: [1,2,1,2,3], k=2
AtMost(2): [1], [1,2], [1,2,1], [1,2,1,2], [2,1,2] → count=9
AtMost(1): [1], [2], [1], [2] → count=4
Exactly(2) = 9 - 4 = 5
Subarrays: [1,2], [2,1], [1,2], [2,1,2], [1,2,1,2]
Result = 5
🔥

14. Remove Duplicates from Sorted Array
Remove duplicates in-place, return new length

Pattern
Two Pointers In-Place

Problem Statement

Given sorted array, remove duplicates in-place so each unique element appears once. Return new length. Do not use extra space (except for small variables).

Approach

Use two pointers: one tracks position to place next unique element, other scans array. When unique element found, place it at write position.

C# Solution

csharp
public int RemoveDuplicates(int[] nums) {
    int writeIdx = 1;

    for (int i = 1; i < nums.Length; i++) {
        if (nums[i] != nums[i - 1]) {
            nums[writeIdx] = nums[i];
            writeIdx++;
        }
    }

    return writeIdx;
}

Example Trace

writeIdx=1
i=1: nums[1]=1 == nums[0]=1, skip
i=2: nums[2]=2 != nums[1]=1 → nums[1]=2, writeIdx=2
i=3: nums[3]=2 == nums[2]=2, skip
i=4: nums[4]=3 != nums[3]=2 → nums[2]=3, writeIdx=3
Result = 3, array = [1,2,3,...]

15. Sort Colors
Sort array of 0s, 1s, 2s in-place (Dutch National Flag)
Medium Amazon Microsoft
15
Time
O(n)
Space
O(1)
Pattern
Dutch National Flag Algorithm

Problem Statement

Given array with values 0, 1, 2 (colors), sort in-place. Do not use library sort. Output must have all 0s, then all 1s, then all 2s.

Dutch National Flag Approach

Maintain three pointers: left (for 0s), mid (current), right (for 2s). Partition array into three regions: [0..left-1] = 0s, [left..mid-1] = 1s, [right+1..n-1] = 2s.

C# Solution

csharp
public void SortColors(int[] nums) {
    int left = 0, mid = 0, right = nums.Length - 1;

    while (mid <= right) {
        if (nums[mid] == 0) {
            // Swap with left, move both
            Swap(nums, left, mid);
            left++;
            mid++;
        } else if (nums[mid] == 1) {
            // Stay in middle region
            mid++;
        } else {
            // Swap with right, only move right
            Swap(nums, mid, right);
            right--;
        }
    }
}

private void Swap(int[] nums, int i, int j) {
    int temp = nums[i];
    nums[i] = nums[j];
    nums[j] = temp;
}

Example Trace

Input: [2,0,2,1,1,0]
left=0, mid=0, right=5
mid=0 (val=2): Swap with right → [0,0,2,1,1,2], right=4
mid=0 (val=0): Swap with left → [0,0,2,1,1,2], left=1, mid=1
mid=1 (val=0): Swap with left → [0,0,2,1,1,2], left=2, mid=2
mid=2 (val=2): Swap with right → [0,0,1,1,2,2], right=3
mid=2 (val=1): Stay, mid=3
mid=3 (val=1): Stay, mid=4
mid=4 (val=2): right already 3, done
Result = [0,0,1,1,2,2]
💡
