OOD Interview Framework

A systematic 4-step approach to designing object-oriented systems in 45-60 minutes.

The 4-Step Process

1
Clarify Requirements
Ask clarifying questions to understand scope, constraints, and user flows. What features are in scope? What are the scale and performance requirements?
2
Identify Objects & Relationships
List entities (classes), their attributes, and how they interact. Draw a class diagram. Identify parent-child relationships, composition, aggregation.
3
Define Core Methods & APIs
Write method signatures with parameters and return types. Focus on the most important operations. Show relationships with code.
4
Code Key Implementations
Implement the most complex or interesting methods. Show error handling, edge cases, and design trade-offs. Be ready to discuss alternatives.

Interview Tips:

  • Think out loud. Explain your design decisions.
  • Ask follow-up questions to clarify scope.
  • Draw diagrams (class relationships, data structures).
  • Discuss trade-offs: simplicity vs. flexibility, memory vs. speed.
  • Code is secondary to design. Show good OOP structure first.
  • Be ready to extend your design based on new requirements.

SOLID Principles

Five design principles for writing maintainable, testable, and flexible code. Each principle prevents a specific type of problem.

1. Single Responsibility Principle (SRP)

A class should have only one reason to change. Each class should be responsible for a single aspect of the system.

C#
// ❌ BAD: User class has multiple responsibilities
public class User {
  public string Name { get; set; }
  public string Email { get; set; }

  public void SaveToDatabase() { }      // Database responsibility
  public void SendEmail() { }            // Email responsibility
  public void ValidateEmail() { }        // Validation responsibility
}

// ✓ GOOD: Separated concerns
public class User {
  public string Name { get; set; }
  public string Email { get; set; }
}

public class UserRepository {
  public void Save(User user) { }
}

public class EmailService {
  public void SendEmail(string email, string message) { }
}

public class EmailValidator {
  public bool IsValid(string email) { }
}

2. Open/Closed Principle (OCP)

Classes should be open for extension but closed for modification. Use inheritance and interfaces instead of modifying existing code.

C#
// ❌ BAD: Adding new shape types requires modifying AreaCalculator
public class AreaCalculator {
  public double CalculateArea(Shape shape) {
    if (shape is Circle c) return Math.PI * c.Radius * c.Radius;
    if (shape is Rectangle r) return r.Width * r.Height;
    if (shape is Triangle t) return 0.5 * t.Base * t.Height;
    throw new Exception("Unknown shape");
  }
}

// ✓ GOOD: Use polymorphism; extend without modifying
public abstract class Shape {
  public abstract double Area();
}

public class Circle : Shape {
  public double Radius { get; set; }
  public override double Area() => Math.PI * Radius * Radius;
}

public class Rectangle : Shape {
  public double Width { get; set; }
  public double Height { get; set; }
  public override double Area() => Width * Height;
}

public class AreaCalculator {
  public double CalculateArea(Shape shape) => shape.Area();
  // No changes needed when adding new shapes!
}

3. Liskov Substitution Principle (LSP)

Derived classes should be substitutable for their base classes. If B is a subtype of A, then objects of type A can be replaced with objects of type B without breaking the system.

C#
// ❌ BAD: Penguin violates LSP (can't fly)
public class Bird {
  public virtual void Fly() { }
}

public class Penguin : Bird {
  public override void Fly() {
    throw new InvalidOperationException("Penguins can't fly!");
  }
}

// Client code breaks with Penguin
List<Bird> birds = new List<Bird> { new Eagle(), new Penguin() };
foreach (var bird in birds) {
  bird.Fly(); // Exception for Penguin!
}

// ✓ GOOD: Separate flying behavior
public abstract class Bird { }

public interface IFlyable {
  void Fly();
}

public class Eagle : Bird, IFlyable {
  public void Fly() { Console.WriteLine("Eagle flying"); }
}

public class Penguin : Bird {
  // No Fly() method
}

public class BirdWatcher {
  public void MakeBirdsFly(List<IFlyable> birds) {
    foreach (var bird in birds) bird.Fly();
  }
}

4. Interface Segregation Principle (ISP)

Clients should not depend on interfaces they don't use. Break large interfaces into smaller, specific ones.

C#
// ❌ BAD: Fat interface forces unnecessary implementations
public interface IWorker {
  void Work();
  void Eat();
  void Code();
  void Sleep();
}

public class Human : IWorker {
  public void Work() { }
  public void Eat() { }
  public void Code() { }
  public void Sleep() { }
}

public class Robot : IWorker {
  public void Work() { }
  public void Eat() { } // Robot doesn't eat!
  public void Code() { }
  public void Sleep() { } // Robot doesn't sleep!
}

// ✓ GOOD: Segregate interfaces by responsibility
public interface IWorker {
  void Work();
}

public interface IEater {
  void Eat();
}

public interface IDeveloper {
  void Code();
}

public interface ISleeper {
  void Sleep();
}

public class Human : IWorker, IEater, IDeveloper, ISleeper {
  public void Work() { }
  public void Eat() { }
  public void Code() { }
  public void Sleep() { }
}

public class Robot : IWorker, IDeveloper {
  public void Work() { }
  public void Code() { }
}

5. Dependency Inversion Principle (DIP)

High-level modules should not depend on low-level modules. Both should depend on abstractions. Depend on interfaces, not concrete implementations.

C#
// ❌ BAD: PaymentProcessor depends on concrete MySQLDatabase
public class MySQLDatabase {
  public void Save(string data) { }
}

public class PaymentProcessor {
  private MySQLDatabase _db = new MySQLDatabase();

  public void ProcessPayment(decimal amount) {
    _db.Save($"Payment: {amount}");
  }
}

// Can't swap database without changing PaymentProcessor!

// ✓ GOOD: PaymentProcessor depends on IDatabase abstraction
public interface IDatabase {
  void Save(string data);
}

public class MySQLDatabase : IDatabase {
  public void Save(string data) { }
}

public class MongoDatabase : IDatabase {
  public void Save(string data) { }
}

public class PaymentProcessor {
  private IDatabase _db;

  // Inject dependency via constructor
  public PaymentProcessor(IDatabase db) {
    _db = db;
  }

  public void ProcessPayment(decimal amount) {
    _db.Save($"Payment: {amount}");
  }
}

// Now we can use any IDatabase implementation:
IDatabase db1 = new MySQLDatabase();
IDatabase db2 = new MongoDatabase();
var processor1 = new PaymentProcessor(db1);
var processor2 = new PaymentProcessor(db2);

Common Design Patterns

Six essential patterns frequently asked in OOD interviews. Recognize when to use each one.

1. Singleton Pattern

Ensure a class has only one instance and provide a global point of access to it.

C#
public class Logger {
  private static Logger _instance;
  private static readonly object _lock = new object();

  private Logger() { } // Private constructor

  public static Logger GetInstance() {
    if (_instance == null) {
      lock (_lock) {
        if (_instance == null) {
          _instance = new Logger();
        }
      }
    }
    return _instance;
  }

  public void Log(string message) {
    Console.WriteLine($"[LOG] {message}");
  }
}

// Usage
Logger logger1 = Logger.GetInstance();
Logger logger2 = Logger.GetInstance();
// logger1 and logger2 are the same object

2. Factory Pattern

Create objects without specifying their exact classes. Useful when object creation logic is complex or varies by type.

C#
// Abstract product
public abstract class Vehicle {
  public abstract void Drive();
}

public class Car : Vehicle {
  public override void Drive() { Console.WriteLine("Driving car"); }
}

public class Truck : Vehicle {
  public override void Drive() { Console.WriteLine("Driving truck"); }
}

public class Bike : Vehicle {
  public override void Drive() { Console.WriteLine("Riding bike"); }
}

// Factory class
public class VehicleFactory {
  public static Vehicle CreateVehicle(string type) {
    return type.ToLower() switch {
      "car" => new Car(),
      "truck" => new Truck(),
      "bike" => new Bike(),
      _ => throw new ArgumentException("Unknown vehicle type")
    };
  }
}

// Usage
Vehicle car = VehicleFactory.CreateVehicle("car");
car.Drive(); // "Driving car"

3. Strategy Pattern

Define a family of algorithms and make them interchangeable. Let the client choose the algorithm at runtime.

C#
// Strategy interface
public interface ISortingStrategy {
  void Sort(int[] arr);
}

public class QuickSort : ISortingStrategy {
  public void Sort(int[] arr) {
    Console.WriteLine("Sorting with QuickSort");
  }
}

public class MergeSort : ISortingStrategy {
  public void Sort(int[] arr) {
    Console.WriteLine("Sorting with MergeSort");
  }
}

// Context class
public class DataProcessor {
  private ISortingStrategy _strategy;

  public void SetSortStrategy(ISortingStrategy strategy) {
    _strategy = strategy;
  }

  public void ProcessData(int[] data) {
    _strategy.Sort(data);
  }
}

// Usage
var processor = new DataProcessor();
processor.SetSortStrategy(new QuickSort());
processor.ProcessData(new int[] { 3, 1, 4, 1, 5 }); // "Sorting with QuickSort"

processor.SetSortStrategy(new MergeSort());
processor.ProcessData(new int[] { 3, 1, 4, 1, 5 }); // "Sorting with MergeSort"

4. Observer Pattern

Define a one-to-many relationship where multiple observers listen for changes in a subject.

C#
// Observer interface
public interface IObserver {
  void Update(string message);
}

// Concrete observer
public class EmailObserver : IObserver {
  public void Update(string message) {
    Console.WriteLine($"Email sent: {message}");
  }
}

public class LogObserver : IObserver {
  public void Update(string message) {
    Console.WriteLine($"Logged: {message}");
  }
}

// Subject
public class EventPublisher {
  private List<IObserver> _observers = new List<IObserver>();

  public void Subscribe(IObserver observer) {
    _observers.Add(observer);
  }

  public void Unsubscribe(IObserver observer) {
    _observers.Remove(observer);
  }

  public void PublishEvent(string message) {
    foreach (var observer in _observers) {
      observer.Update(message);
    }
  }
}

// Usage
var publisher = new EventPublisher();
publisher.Subscribe(new EmailObserver());
publisher.Subscribe(new LogObserver());
publisher.PublishEvent("System started");
// Output: Email sent: System started
//         Logged: System started

5. Decorator Pattern

Add behavior to objects dynamically without modifying their structure.

C#
// Component interface
public interface IDataSource {
  string Read();
}

// Concrete component
public class FileDataSource : IDataSource {
  private string _filename;
  public FileDataSource(string filename) { _filename = filename; }
  public string Read() => $"Reading from {_filename}";
}

// Decorator base
public abstract class DataSourceDecorator : IDataSource {
  protected IDataSource _dataSource;
  public DataSourceDecorator(IDataSource dataSource) {
    _dataSource = dataSource;
  }
  public virtual string Read() => _dataSource.Read();
}

// Concrete decorators
public class EncryptionDecorator : DataSourceDecorator {
  public EncryptionDecorator(IDataSource dataSource) : base(dataSource) { }
  public override string Read() => $"Encrypted({_dataSource.Read()})";
}

public class CompressionDecorator : DataSourceDecorator {
  public CompressionDecorator(IDataSource dataSource) : base(dataSource) { }
  public override string Read() => $"Compressed({_dataSource.Read()})";
}

// Usage
IDataSource source = new FileDataSource("data.txt");
source = new EncryptionDecorator(source);
source = new CompressionDecorator(source);
Console.WriteLine(source.Read());
// Output: Compressed(Encrypted(Reading from data.txt))

6. Builder Pattern

Construct complex objects step-by-step. Useful when objects have many optional parameters.

C#
public class House {
  public string Foundation { get; set; }
  public string Walls { get; set; }
  public string Roof { get; set; }
  public string Interior { get; set; }
  public string Garage { get; set; }

  public override string ToString() =>
    $"House: {Foundation}, {Walls}, {Roof}, {Interior}, {Garage}";
}

public class HouseBuilder {
  private House _house = new House();

  public HouseBuilder WithFoundation(string foundation) {
    _house.Foundation = foundation;
    return this;
  }

  public HouseBuilder WithWalls(string walls) {
    _house.Walls = walls;
    return this;
  }

  public HouseBuilder WithRoof(string roof) {
    _house.Roof = roof;
    return this;
  }

  public HouseBuilder WithInterior(string interior) {
    _house.Interior = interior;
    return this;
  }

  public HouseBuilder WithGarage(string garage) {
    _house.Garage = garage;
    return this;
  }

  public House Build() => _house;
}

// Usage
var house = new HouseBuilder()
  .WithFoundation("concrete")
  .WithWalls("brick")
  .WithRoof("wood")
  .WithInterior("modern")
  .WithGarage("2-car")
  .Build();
Console.WriteLine(house);

Problem 1: Parking Lot System
Design a parking lot that supports multiple vehicle types, reservation, and payment. Users can search for available spots, park vehicles, and exit with payment.
1

Requirements

  • Support multiple vehicle types: Car, Motorcycle, Truck, van
  • Support multiple levels in the parking lot
  • Find the nearest available spot for a vehicle
  • Track occupancy and generate reports
  • Handle payment at exit
  • Implement hourly rates based on vehicle type

Key Classes & Design

C#
// Enums
public enum VehicleType {
  Motorcycle = 0,
  Car = 1,
  Truck = 2
}

public enum ParkingSpotType {
  Motorcycle = 0,
  Compact = 1,
  Large = 2
}

public enum ParkingTicketStatus {
  Active = 0,
  Paid = 1,
  Lost = 2
}

// Vehicle hierarchy
public abstract class Vehicle {
  public string LicensePlate { get; set; }
  public abstract VehicleType GetType();
  public abstract ParkingSpotType GetRequiredSpotType();
}

public class Car : Vehicle {
  public override VehicleType GetType() => VehicleType.Car;
  public override ParkingSpotType GetRequiredSpotType() => ParkingSpotType.Compact;
}

public class Motorcycle : Vehicle {
  public override VehicleType GetType() => VehicleType.Motorcycle;
  public override ParkingSpotType GetRequiredSpotType() => ParkingSpotType.Motorcycle;
}

public class Truck : Vehicle {
  public override VehicleType GetType() => VehicleType.Truck;
  public override ParkingSpotType GetRequiredSpotType() => ParkingSpotType.Large;
}

// Parking spot
public class ParkingSpot {
  public int Id { get; set; }
  public int Level { get; set; }
  public ParkingSpotType SpotType { get; set; }
  public Vehicle ParkedVehicle { get; set; }
  public bool IsAvailable => ParkedVehicle == null;

  public bool CanFitVehicle(Vehicle vehicle) =>
    IsAvailable && CanFitType(vehicle.GetRequiredSpotType());

  private bool CanFitType(ParkingSpotType requiredType) =>
    SpotType >= requiredType; // Larger spots can fit smaller vehicles
}

// Parking ticket
public class ParkingTicket {
  public int Id { get; set; }
  public Vehicle Vehicle { get; set; }
  public ParkingSpot Spot { get; set; }
  public DateTime IssuedTime { get; set; }
  public DateTime ExitTime { get; set; }
  public decimal Price { get; set; }
  public ParkingTicketStatus Status { get; set; }
}

// Main parking lot class
public class ParkingLot {
  private List<ParkingLevel> _levels;
  private Dictionary<string, ParkingTicket> _ticketsByPlate;
  private PaymentProcessor _paymentProcessor;

  public ParkingLot(int numLevels, int spotsPerLevel) {
    _levels = new List<ParkingLevel>();
    _ticketsByPlate = new Dictionary<string, ParkingTicket>();

    for (int i = 0; i < numLevels; i++) {
      _levels.Add(new ParkingLevel(i, spotsPerLevel));
    }

    _paymentProcessor = new PaymentProcessor();
  }

  public ParkingTicket ParkVehicle(Vehicle vehicle) {
    var spot = FindAvailableSpot(vehicle);
    if (spot == null) {
      throw new Exception("No available spot for this vehicle");
    }

    spot.ParkedVehicle = vehicle;
    var ticket = new ParkingTicket {
      Id = Guid.NewGuid().GetHashCode(),
      Vehicle = vehicle,
      Spot = spot,
      IssuedTime = DateTime.Now,
      Status = ParkingTicketStatus.Active
    };

    _ticketsByPlate[vehicle.LicensePlate] = ticket;
    return ticket;
  }

  public decimal UnparkVehicle(string licensePlate) {
    if (!_ticketsByPlate.ContainsKey(licensePlate)) {
      throw new Exception("No ticket found for this vehicle");
    }

    var ticket = _ticketsByPlate[licensePlate];
    ticket.ExitTime = DateTime.Now;
    ticket.Price = CalculatePrice(ticket);
    ticket.Status = ParkingTicketStatus.Paid;

    ticket.Spot.ParkedVehicle = null;
    _ticketsByPlate.Remove(licensePlate);

    return ticket.Price;
  }

  private ParkingSpot FindAvailableSpot(Vehicle vehicle) {
    foreach (var level in _levels) {
      var spot = level.FindAvailableSpot(vehicle);
      if (spot != null) return spot;
    }
    return null;
  }

  private decimal CalculatePrice(ParkingTicket ticket) {
    var duration = ticket.ExitTime - ticket.IssuedTime;
    var hours = Math.Ceiling(duration.TotalHours);

    decimal hourlyRate = ticket.Vehicle.GetType() switch {
      VehicleType.Motorcycle => 2m,
      VehicleType.Car => 5m,
      VehicleType.Truck => 10m,
      _ => 0m
    };

    return (decimal)hours * hourlyRate;
  }
}

// Parking level
public class ParkingLevel {
  private int _id;
  private List<ParkingSpot> _spots;

  public ParkingLevel(int id, int numSpots) {
    _id = id;
    _spots = new List<ParkingSpot>();

    for (int i = 0; i < numSpots; i++) {
      var spotType = i % 3 switch {
        0 => ParkingSpotType.Motorcycle,
        1 => ParkingSpotType.Compact,
        _ => ParkingSpotType.Large
      };

      _spots.Add(new ParkingSpot {
        Id = i,
        Level = _id,
        SpotType = spotType
      });
    }
  }

  public ParkingSpot FindAvailableSpot(Vehicle vehicle) {
    foreach (var spot in _spots) {
      if (spot.CanFitVehicle(vehicle)) {
        return spot;
      }
    }
    return null;
  }
}

// Payment processor
public class PaymentProcessor {
  public bool ProcessPayment(decimal amount) {
    // Simplified: assume payment always succeeds
    Console.WriteLine($"Processing payment of ${amount}");
    return true;
  }
}

Design Trade-offs

Strengths
Polymorphic vehicle types make it easy to add new vehicle types
Spot type hierarchy allows flexible spot assignment
Separate Level and Lot classes for scalability
Price calculation strategy can be extended
Limitations
No reservation system (could add ReservationService)
Payment validation is simplified (real system needs PCI compliance)
No persistence (add database layer in production)
Spot finding is O(n) (could add spatial indexing)

  • Concurrency: How would you handle multiple vehicles parking simultaneously?
  • Reports: How would you generate occupancy reports by level and spot type?

Problem 2: LRU Cache
Implement a cache with GET and PUT operations in O(1) time and space. Evict least-recently-used items when capacity is exceeded.
2

Requirements

  • Get(key): Return value in O(1), mark as recently used
  • Put(key, value): Insert/update in O(1)
  • When capacity exceeded, evict least-recently-used
  • Both ops must be constant time

Key Data Structures

ℹ
DoublyLinkedList + Dictionary: Dictionary gives O(1) lookup; DoublyLinkedList maintains insertion/access order with O(1) move operations.
C#
public class LRUCache {
  private class Node {
    public int Key { get; set; }
    public int Value { get; set; }
    public Node Prev { get; set; }
    public Node Next { get; set; }

    public Node(int key, int value) {
      Key = key;
      Value = value;
    }
  }

  private int _capacity;
  private Dictionary<int, Node> _cache;
  private Node _head; // Most recently used
  private Node _tail; // Least recently used

  public LRUCache(int capacity) {
    _capacity = capacity;
    _cache = new Dictionary<int, Node>();

    // Sentinel nodes to simplify edge cases
    _head = new Node(0, 0);
    _tail = new Node(0, 0);
    _head.Next = _tail;
    _tail.Prev = _head;
  }

  public int Get(int key) {
    if (!_cache.ContainsKey(key)) {
      return -1;
    }

    var node = _cache[key];
    MoveToHead(node); // Mark as recently used
    return node.Value;
  }

  public void Put(int key, int value) {
    if (_cache.ContainsKey(key)) {
      // Update existing node
      var node = _cache[key];
      node.Value = value;
      MoveToHead(node);
    } else {
      // Create new node
      var newNode = new Node(key, value);
      _cache[key] = newNode;
      AddToHead(newNode);

      // Evict if over capacity
      if (_cache.Count > _capacity) {
        var evicted = RemoveTail();
        _cache.Remove(evicted.Key);
      }
    }
  }

  private void MoveToHead(Node node) {
    RemoveNode(node);
    AddToHead(node);
  }

  private void AddToHead(Node node) {
    node.Prev = _head;
    node.Next = _head.Next;
    _head.Next.Prev = node;
    _head.Next = node;
  }

  private void RemoveNode(Node node) {
    var prev = node.Prev;
    var next = node.Next;
    prev.Next = next;
    next.Prev = prev;
  }

  private Node RemoveTail() {
    var lru = _tail.Prev;
    RemoveNode(lru);
    return lru;
  }
}

// Usage
var cache = new LRUCache(2);
cache.Put(1, 1);
cache.Put(2, 2);
Console.WriteLine(cache.Get(1)); // 1
cache.Put(3, 3); // Evicts key 2
Console.WriteLine(cache.Get(2)); // -1 (evicted)

Complexity Analysis

OperationTimeSpaceNotes
GetO(1)—Dictionary lookup + O(1) list move
PutO(1)—Dictionary insert + list manipulation
Space—O(capacity)Dictionary + DoublyLinkedList nodes

Alternative Approach: OrderedDictionary (C# 6.0+)

C#
// .NET 6+ has OrderedDictionary that maintains insertion order
using System.Collections.Generic;

public class SimpleLRUCache {
  private int _capacity;
  private Dictionary<int, int> _cache;

  public SimpleLRUCache(int capacity) {
    _capacity = capacity;
    _cache = new Dictionary<int, int>();
  }

  public int Get(int key) {
    if (!_cache.ContainsKey(key)) return -1;

    // Remove and re-add to move to end (most recent)
    var value = _cache[key];
    _cache.Remove(key);
    _cache[key] = value;
    return value;
  }

  public void Put(int key, int value) {
    if (_cache.ContainsKey(key)) {
      _cache.Remove(key);
    } else if (_cache.Count >= _capacity) {
      // Remove first key (least recently used)
      var lruKey = _cache.Keys.First();
      _cache.Remove(lruKey);
    }

    _cache[key] = value;
  }
}

Design Trade-offs

DoublyLinkedList Approach
True O(1) for all operations
More efficient in production
Full control over data structure
Dictionary.First() Approach
Simpler code, fewer bugs
Requires .NET 6+
Slightly slower (First() is O(1) but less optimal)

Problem 3: Design a Deck of Cards
Implement a card deck and game (e.g., Blackjack). Support shuffling, dealing, and checking hand values with polymorphic card and game logic.
3

Requirements

  • Standard 52-card deck (4 suits, 13 ranks)
  • Shuffle deck randomly
  • Deal cards to players
  • Support Blackjack-specific rules
  • Calculate hand value (Ace=1 or 11)
  • Extensible for other card games
C#
// Enums
public enum Suit {
  Hearts,
  Diamonds,
  Clubs,
  Spades
}

public enum Rank {
  Ace = 1, Two, Three, Four, Five, Six, Seven, Eight, Nine, Ten,
  Jack, Queen, King
}

// Card class
public class Card {
  public Suit Suit { get; set; }
  public Rank Rank { get; set; }

  public Card(Suit suit, Rank rank) {
    Suit = suit;
    Rank = rank;
  }

  public int GetValue() {
    return Rank switch {
      Rank.Ace => 1,        // Ace can be 1 or 11 (handled in Hand)
      Rank.Two => 2,
      Rank.Three => 3,
      Rank.Four => 4,
      Rank.Five => 5,
      Rank.Six => 6,
      Rank.Seven => 7,
      Rank.Eight => 8,
      Rank.Nine => 9,
      Rank.Ten or Rank.Jack or Rank.Queen or Rank.King => 10,
      _ => 0
    };
  }

  public override string ToString() => $"{Rank} of {Suit}";
}

// Deck class
public class Deck {
  private List<Card> _cards;
  private Random _random;

  public Deck() {
    _cards = new List<Card>();
    _random = new Random();
    Initialize();
  }

  private void Initialize() {
    foreach (Suit suit in Enum.GetValues(typeof(Suit))) {
      foreach (Rank rank in Enum.GetValues(typeof(Rank))) {
        _cards.Add(new Card(suit, rank));
      }
    }
  }

  public void Shuffle() {
    for (int i = _cards.Count - 1; i > 0; i--) {
      int randomIndex = _random.Next(i + 1);
      (_cards[i], _cards[randomIndex]) = (_cards[randomIndex], _cards[i]);
    }
  }

  public Card DealCard() {
    if (_cards.Count == 0) {
      throw new Exception("No cards left in deck");
    }
    var card = _cards[0];
    _cards.RemoveAt(0);
    return card;
  }

  public int GetRemainingCards() => _cards.Count;
}

// Hand class
public class Hand {
  private List<Card> _cards;

  public Hand() {
    _cards = new List<Card>();
  }

  public void AddCard(Card card) {
    _cards.Add(card);
  }

  public List<Card> GetCards() => new List<Card>(_cards);

  public int GetValue() {
    int value = 0;
    int aces = 0;

    // Sum all cards
    foreach (var card in _cards) {
      value += card.GetValue();
      if (card.Rank == Rank.Ace) {
        aces++;
      }
    }

    // Adjust for Aces (make one Ace = 11 if possible)
    while (value + 10 <= 21 && aces > 0) {
      value += 10;
      aces--;
    }

    return value;
  }

  public bool IsBlackjack() => _cards.Count == 2 && GetValue() == 21;

  public bool IsBust() => GetValue() > 21;
}

// Blackjack game
public class BlackjackGame {
  private Deck _deck;
  private Hand _playerHand;
  private Hand _dealerHand;

  public BlackjackGame() {
    _deck = new Deck();
    _deck.Shuffle();
    _playerHand = new Hand();
    _dealerHand = new Hand();
  }

  public void DealInitialCards() {
    _playerHand.AddCard(_deck.DealCard());
    _dealerHand.AddCard(_deck.DealCard());
    _playerHand.AddCard(_deck.DealCard());
    _dealerHand.AddCard(_deck.DealCard());
  }

  public void PlayerHit() {
    _playerHand.AddCard(_deck.DealCard());
  }

  public void DealerHit() {
    while (_dealerHand.GetValue() < 17) {
      _dealerHand.AddCard(_deck.DealCard());
    }
  }

  public string GetGameResult() {
    int playerValue = _playerHand.GetValue();
    int dealerValue = _dealerHand.GetValue();

    if (_playerHand.IsBlackjack() && !_dealerHand.IsBlackjack()) {
      return "Player wins with Blackjack!";
    }

    if (_playerHand.IsBust()) {
      return "Player busts. Dealer wins!";
    }

    if (_dealerHand.IsBust()) {
      return "Dealer busts. Player wins!";
    }

    if (playerValue > dealerValue) {
      return "Player wins!";
    } else if (playerValue < dealerValue) {
      return "Dealer wins!";
    } else {
      return "Push (tie)!";
    }
  }

  public void PrintGameState() {
    Console.WriteLine($"Player hand: {string.Join(", ", _playerHand.GetCards())} (Value: {_playerHand.GetValue()})");
    Console.WriteLine($"Dealer hand: {string.Join(", ", _dealerHand.GetCards())} (Value: {_dealerHand.GetValue()})");
  }
}
