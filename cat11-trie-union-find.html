Trie (Prefix Tree)

A tree structure optimized for prefix-based lookups. Each node represents a character; paths from root spell words. Insert is O(m) where m is word length. Search is O(m). Space is O(alphabet × depth).

C# 
public class TrieNode
{
    public Dictionary<char, TrieNode> children = new();
    public bool isWord = false;
}

public class Trie
{
    private TrieNode root = new();

    public void Insert(string word)
    {
        var node = root;
        foreach (var c in word)
        {
            if (!node.children.ContainsKey(c))
                node.children[c] = new TrieNode();
            node = node.children[c];
        }
        node.isWord = true;
    }

    public bool Search(string word)
    {
        var node = root;
        foreach (var c in word)
        {
            if (!node.children.ContainsKey(c)) return false;
            node = node.children[c];
        }
        return node.isWord;
    }

    public bool StartsWith(string prefix)
    {
        var node = root;
        foreach (var c in prefix)
        {
            if (!node.children.ContainsKey(c)) return false;
            node = node.children[c];
        }
        return true;
    }
}

Union-Find (Disjoint Set Union)

Tracks connected components efficiently. Union(a, b) merges sets; Find(x) returns representative. With path compression and union by rank: nearly O(1) per operation. Critical for cycle detection and connectivity.

C# 
public class UnionFind
{
    private int[] parent, rank;

    public UnionFind(int n)
    {
        parent = new int[n];
        rank = new int[n];
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    public int Find(int x)
    {
        if (parent[x] != x)
            parent[x] = Find(parent[x]); // Path compression
        return parent[x];
    }

    public bool Union(int x, int y)
    {
        int px = Find(x), py = Find(y);
        if (px == py) return false;

        // Union by rank
        if (rank[px] < rank[py]) (px, py) = (py, px);
        parent[py] = px;
        if (rank[px] == rank[py]) rank[px]++;
        return true;
    }
}
★
When to use: Trie excels at prefix queries and autocomplete. Union-Find shines in connectivity, grouping, and cycle detection. Choose based on problem structure.

public class Trie
{

    {

    }

    public void Insert(string word)
    {

        foreach (var c in word)
        {
            if (!node.children.ContainsKey(c))
                node.children[c] = new TrieNode();
            node = node.children[c];
        }
        node.isWord = true;
    }

    public bool Search(string word)
    {
        var node = root;
        foreach (var c in word)
        {
            if (!node.children.ContainsKey(c))
                return false;
            node = node.children[c];
        }

    }

        }
        return true;
    }

Complexity & Edge Cases

✓ Strong
Insert/search in O(m) regardless of trie size
Memory efficient for many words with common prefixes
⚠ Watch out
Empty string: handle as valid prefix but not word
Case sensitivity: decide upfront (usually lowercase)
💡

Design Add and Search Words

Approach

1

C# 
public class WordDictionary
{
    private class Node
    {
        public Dictionary<char, Node> children = new();
        public bool isWord = false;
    }

    private Node root = new();

    public void AddWord(string word)
    {
        var node = root;
        foreach (var c in word)
        {
            if (!node.children.ContainsKey(c))
                node.children[c] = new Node();
            node = node.children[c];
        }
        node.isWord = true;
    }

    public bool Search(string word)
    {
        return DFS(word, 0, root);
    }

    private bool DFS(string word, int idx, Node node)
    {
        if (idx == word.Length)
            return node.isWord;

        char c = word[idx];
        if (c == '.')
        {
            foreach (var child in node.children.Values)
                if (DFS(word, idx + 1, child))
                    return true;
            return false;
        }

        if (!node.children.ContainsKey(c))
            return false;
        return DFS(word, idx + 1, node.children[c]);
    }
}

Complexity & Edge Cases

✓ Strong
Add is O(m), search is O(26^m) worst case (all dots)
DFS naturally handles wildcard recursion
⚠ Watch out
Pattern all dots on large trie = exponential time
Must verify isWord at end, not just path existence
💡
FAANG tip: Asked by Meta. Pattern matching with wildcards is common in autocomplete filtering. Optimize by memoizing search results if same patterns repeat.
Word Search II
Given a grid and list of words, return all words found in grid (can traverse adjacent cells, each cell used once per word).
Hard Google Amazon
3
Time
O(n×m×4^l)
Space
O(k×l)

Approach

1
Build trie from all words for efficient prefix checking.
2
DFS from each cell: follow trie paths in grid.
3
Add found words to result, mark as visited to avoid duplicates.
C# 
public IList<string> FindWords(char[][] board, string[] words)
{
    var trie = new TrieNode();
    foreach (var word in words)
    {
        var node = trie;
        foreach (var c in word)
        {
            if (!node.children.ContainsKey(c))
                node.children[c] = new TrieNode();
            node = node.children[c];
        }
        node.word = word;
    }

    var result = new List<string>();
    int rows = board.Length, cols = board[0].Length;

    for (int i = 0; i < rows; i++)
        for (int j = 0; j < cols; j++)
            DFS(board, i, j, trie, result);

    return result;
}

private void DFS(char[][] board, int r, int c, TrieNode node, List<string> result)
{
    char ch = board[r][c];
    if (ch == '#' || !node.children.ContainsKey(ch))
        return;

    node = node.children[ch];
    if (node.word != null)
    {
        result.Add(node.word);
        node.word = null; // Avoid duplicates
    }

    board[r][c] = '#';
    foreach (var (dr, dc) in new[] { (-1, 0), (1, 0), (0, -1), (0, 1) })
    {
        int nr = r + dr, nc = c + dc;
        if (nr >= 0 && nr < board.Length && nc >= 0 && nc < board[0].Length)
            DFS(board, nr, nc, node, result);
    }
    board[r][c] = ch;
}

Complexity & Edge Cases

✓ Strong
Trie prunes impossible branches early
Backtracking marks visited with #, restores after
⚠ Watch out
Mark word as null to avoid adding duplicate results
Board mutation: restore state after DFS
💡
FAANG tip: Hard problem—Google, Amazon love it. Key insight: trie avoids checking entire word list per cell. Set word=null instead of deleting from trie for efficiency.
Replace Words
Given dictionary and sentence, replace each word with shortest matching prefix from dictionary.
Medium Amazon
4
Time
O(n×m)
Space
O(d)

Approach

1
Build trie from dictionary prefixes.
2
For each word, traverse trie greedily finding shortest match.
3
Replace or keep original based on match.
C# 
public string ReplaceWords(IList<string> dictionary, string sentence)
{
    var trie = new TrieNode();
    foreach (var word in dictionary)
    {
        var node = trie;
        foreach (var c in word)
        {
            if (!node.children.ContainsKey(c))
                node.children[c] = new TrieNode();
            node = node.children[c];
        }
        node.isWord = true;
    }

    var words = sentence.Split(' ');
    var result = new List<string>();

    foreach (var word in words)
    {
        var prefix = FindPrefix(word, trie);
        result.Add(prefix ?? word);
    }

    return string.Join(" ", result);
}

private string FindPrefix(string word, TrieNode trie)
{
    var node = trie;
    var prefix = new StringBuilder();

    foreach (var c in word)
    {
        if (!node.children.ContainsKey(c) || node.isWord)
            break;
        prefix.Append(c);
        node = node.children[c];
    }

    return node.isWord ? prefix.ToString() : null;
}

Complexity & Edge Cases

✓ Strong
Single pass per word yields shortest prefix
Trie lookup is O(prefix length), fast and efficient
⚠ Watch out
Break when word found (isWord=true) or child missing
Return original word if no prefix found
💡
FAANG tip: Amazon asks this. Classic trie application—don't overthink. Stop at first word found, not longest; greedy works.
Number of Provinces
Given adjacency matrix of cities, return number of provinces (groups of directly/indirectly connected cities).
Medium Amazon Meta
5
Time
O(n²α(n))
Space
O(n)

Approach

1
Initialize Union-Find with n cities, each its own set.
2
For each edge (i,j), union sets containing i and j.
3
Count unique roots—each root is one province.
C# 
public int FindCircleNum(int[][] isConnected)
{
    int n = isConnected.Length;
    var uf = new UnionFind(n);

    for (int i = 0; i < n; i++)
        for (int j = i + 1; j < n; j++)
            if (isConnected[i][j] == 1)
                uf.Union(i, j);

    var roots = new HashSet<int>();
    for (int i = 0; i < n; i++)
        roots.Add(uf.Find(i));

    return roots.Count;
}

Complexity & Edge Cases

✓ Strong
Union-Find is nearly O(1) per operation with optimizations
Handles cycles naturally—no explicit cycle tracking
⚠ Watch out
Single city (n=1) has 1 province
isConnected[i][i] is always 1 but ignored
💡
FAANG tip: Union-Find shines here—simple and elegant. Amazon, Meta love connectivity problems. Path compression + rank make it near-linear.
Redundant Connection
In a tree with one redundant edge, find and return the edge that creates a cycle.
Medium Google Apple
6
Time
O(nα(n))
Space
O(n)

Approach

1
Process edges in order, union their endpoints.
2
If endpoints already in same set, that edge is redundant.
3
Return first redundant edge found (array order).
C# 
public int[] FindRedundantConnection(int[][] edges)
{
    int n = edges.Length;
    var uf = new UnionFind(n + 1);

    foreach (var edge in edges)
    {
        int u = edge[0], v = edge[1];
        if (!uf.Union(u, v))
            return edge; // Cycle detected
    }

    return new int[] { };
}

Complexity & Edge Cases

✓ Strong
Detects cycle on first occurrence (linear time)
No need to build graph structure
⚠ Watch out
Union must return false for existing connections
1-indexed nodes: UnionFind size = n+1
💡
FAANG tip: Google favorite for testing Union-Find understanding. If problem asks "find the edge," Union-Find is the signal. Much faster than DFS/BFS cycle detection.
Accounts Merge
Given list of accounts with names and emails, merge accounts with common emails. Return grouped merged accounts sorted.
Medium Amazon Meta
7
Time
O(n×m×log m)
Space
O(n×m)

Approach

1
Map each email to account index; union all emails in same account.
2
Group emails by root account; build result list per root.
3
Sort emails alphabetically within each merged account.
C# 
public IList<IList<string>> AccountsMerge(IList<IList<string>> accounts)
{
    int n = accounts.Count;
    var uf = new UnionFind(n);
    var emailToAccount = new Dictionary<string, int>();

    for (int i = 0; i < n; i++)
    {
        foreach (var email in accounts[i].Skip(1))
        {
            if (emailToAccount.ContainsKey(email))
                uf.Union(i, emailToAccount[email]);
            else
                emailToAccount[email] = i;
        }
    }

    var emailsByRoot = new Dictionary<int, List<string>>();
    foreach (var (email, accountIdx) in emailToAccount)
    {
        int root = uf.Find(accountIdx);
        if (!emailsByRoot.ContainsKey(root))
            emailsByRoot[root] = new List<string>();
        emailsByRoot[root].Add(email);
    }

    var result = new List<IList<string>>();
    foreach (var (root, emails) in emailsByRoot)
    {
        emails.Sort();
        var merged = new List<string> { accounts[root][0] };
        merged.AddRange(emails);
        result.Add(merged);
    }

    return result;
}

Complexity & Edge Cases

✓ Strong
Union-Find elegantly groups accounts by email overlap
Sorting emails is O(m log m) per account—acceptable
⚠ Watch out
Skip first element (name) when iterating emails
Account name always from original (first email's account root)
💡
FAANG tip: Amazon, Meta love this. Classic Union-Find + hash map pattern. Key: email → account mapping + union, then group by root.
Longest Word in Dictionary
Given dictionary of words, return longest word formed by concatenating other dictionary words (can reuse).
Medium Apple
8
Time
O(n×m²)
Space
O(n×m)

Approach

1
Build trie from all dictionary words.
2
For each word, BFS to check if all prefixes exist in trie.
3
Track longest valid word (longest first, then lexicographic).
C# 
public string LongestWord(string[] words)
{
    var trie = new TrieNode();
    foreach (var word in words)
    {
        var node = trie;
        foreach (var c in word)
        {
            if (!node.children.ContainsKey(c))
                node.children[c] = new TrieNode();
            node = node.children[c];
        }
        node.isWord = true;
    }

    Array.Sort(words, (a, b) => b.Length != a.Length ?
        b.Length.CompareTo(a.Length) : a.CompareTo(b));

    foreach (var word in words)
    {
        var node = trie;
        bool valid = true;
        foreach (var c in word)
        {
            node = node.children[c];
            if (!node.isWord)
            {
                valid = false;
                break;
            }
        }
        if (valid) return word;
    }

    return "";
}

Complexity & Edge Cases

✓ Strong
Sort by length descending first—exit early on first valid
Lexicographic tie-break built into sort
⚠ Watch out
Check each prefix is a word (isWord), not just exists
Empty result: return "" if no valid word
💡
FAANG tip: Apple, Google ask this. Sorting trick: descending length + lexicographic. Trie lookup confirms all prefixes are valid words.
Most Stones Removed from Grid
In a 2D grid, stones sharing row or column form a connected component. Return max stones removable (all but one per component).
Hard Amazon Google
9
Time
O(n×α(n))
Space
O(n)

Approach

1
Create nodes for each row and column (offset to avoid collision).
2
For each stone at (r,c), union row-node with column-node.
3
Stones removed = total stones - number of components.
C# 
public int RemoveStones(int[][] stones)
{
    int n = stones.Length;
    var uf = new UnionFind(20001); // 10000 rows + 10000 cols offset

    foreach (var stone in stones)
    {
        int row = stone[0], col = stone[1] + 10000;
        uf.Union(row, col); // Stones in same row/col connected
    }

    var roots = new HashSet<int>();
    foreach (var stone in stones)
        roots.Add(uf.Find(stone[0])); // Find root via row

    return n - roots.Count; // Remove all but root
}

Complexity & Edge Cases

✓ Strong
Linear time—each stone processed once
Elegant: union rows and columns, not stones directly
⚠ Watch out
Offset column indices to avoid collision with rows
Count components (unique roots), not stones
💡
FAANG tip: Hard but elegant solution—Amazon, Google ask. Key insight: union rows and columns, not stone pairs. Stones = components, removable = stones - components.
Swim in Rising Water
Grid where cell (i,j) has elevation grid[i][j]. Water rises by time. Find minimum time to swim from top-left to bottom-right.
Hard Google Meta
10
Time
O(n²α(n))
Space
O(n²)

Approach

1
Process cells in ascending elevation order (sort grid values).
2
Union cell with neighbors already processed (lower elevation).
3
When start and end connected, return current elevation (time).
C# 
public int SwimInWater(int[][] grid)
{
    int n = grid.Length;
    var uf = new UnionFind(n * n);

    var cells = new List<(int, int, int)>(); // (elevation, row, col)
    for (int i = 0; i < n; i++)
        for (int j = 0; j < n; j++)
            cells.Add((grid[i][j], i, j));

    cells.Sort();

    foreach (var (elev, r, c) in cells)
    {
        foreach (var (dr, dc) in new[] { (-1, 0), (1, 0), (0, -1), (0, 1) })
        {
            int nr = r + dr, nc = c + dc;
            if (nr >= 0 && nr < n && nc >= 0 && nc < n && grid[nr][nc] <= elev)
                uf.Union(r * n + c, nr * n + nc);
        }

        if (uf.Find(0) == uf.Find(n * n - 1))
            return elev;
    }

    return 0;
}

Complexity & Edge Cases

✓ Strong
Binary search alternative would be O(n² log max(grid))
Union-Find approach is cleaner and faster
⚠ Watch out
Only union with neighbors having lower elevation (already processed)
Start (0,0) and end (n-1,n-1) map to indices 0 and n²-1
💡
FAANG tip: Hard problem—Google, Meta favor Union-Find solution over binary search + Dijkstra. Process cells by elevation ascending, union as water rises. Returns on first connectivity.
