INTRO

Introduction: Stack & Queue Essentials

Stacks and queues are linear data structures defined by how you access elements. Understanding when to use each, and mastering the monotonic stack pattern, will accelerate your interview performance.

Stack<T> in C#

Last-In-First-Out (LIFO). Elements added and removed from the same end (the top).

C# 
var stack = new Stack<int>();
stack.Push(5);     // O(1)
int top = stack.Pop();  // O(1), throws if empty
int peek = stack.Peek();  // O(1), throws if empty
int count = stack.Count;  // O(1)
bool empty = stack.Count == 0;

Queue<T> in C#

First-In-First-Out (FIFO). Elements added to the rear, removed from the front.

C# 
var queue = new Queue<int>();
queue.Enqueue(5);     // O(1) amortized
int front = queue.Dequeue();  // O(1) amortized, throws if empty
int peek = queue.Peek();  // O(1), throws if empty
int count = queue.Count;  // O(1)

Monotonic Stack Pattern

A stack that maintains elements in a specific order (increasing or decreasing). Useful for "next greater/smaller element" problems and span problems.

C# (Template) 
// Find next greater element for each element
int[] nums = new[] { 1, 3, 2, 4 };
int[] result = new int[nums.Length];
var stack = new Stack<int>(); // store indices

for (int i = nums.Length - 1; i >= 0; i--) {
    // Pop smaller elements
    while (stack.Count > 0 && nums[stack.Peek()] <= nums[i]) {
        stack.Pop();
    }
    result[i] = stack.Count > 0 ? nums[stack.Peek()] : -1;
    stack.Push(i);
}
When to use Stack
LIFO Access
Matching pairs (parentheses), undo/redo, DFS, backtracking, evaluating expressions.
When to use Queue
FIFO Access
Level-order traversal, BFS, task scheduling, buffering data streams.
Monotonic Stack
Optimize Span Queries
Next/previous greater/smaller, histogram area, stock span. Reduces O(n²) to O(n).

Valid Parentheses

Given a string containing only '(', ')', '{', '}', '[', ']', determine if it is valid. A valid string has every opening bracket matched by a closing bracket of the same type in the correct order.

Easy Google Meta Amazon
1

Problem Statement

Given a string s containing characters '(', ')', '{', '}', '[', and ']', return true if the string is valid, false otherwise.

Approach: Stack

Push opening brackets onto a stack. When you encounter a closing bracket, check if the stack top matches. If yes, pop; if no or stack is empty, return false. At the end, stack must be empty.

C# Solution

C# 
public bool IsValid(string s) {
    var stack = new Stack<char>();

    foreach (char c in s) {
        if (c == '(' || c == '{' || c == '[') {
            stack.Push(c);
        } else {
            if (stack.Count == 0) return false;
            char top = stack.Pop();
            if ((c == ')' && top != '(') ||
                (c == '}' && top != '{') ||
                (c == ']' && top != '[')) {
                return false;
            }
        }
    }

    return stack.Count == 0;
}
Time
O(n)
Space
O(n)

Trace: s = "{[]}"

1
c='{': opening, push. Stack=['{']
2
c='[': opening, push. Stack=['{', '[']
3
c=']': closing, pop '[', matches. Stack=['{']
4
c='}': closing, pop '{', matches. Stack=[]
5
Stack empty → return true

Edge Cases

  • Single closing bracket → return false (stack empty when popping)
  • Mismatched closing bracket → return false (top doesn't match)
  • Unclosed opening bracket → return false (stack not empty at end)
  • Empty string → return true (vacuously valid)
→
FAANG Tip: Mention using a HashMap to map closing to opening brackets for cleaner matching. This scales better if you add new bracket types.
Min Stack

Design a stack that supports push, pop, top, and retrieving the minimum element in constant time.

Medium Google Apple
2

Problem Statement

Implement a MinStack class with methods Push(val), Pop(), Top(), and GetMin(). All operations must run in O(1).

Approach: Dual Stack

Maintain two stacks: one for all values, one for minimums. When pushing, also push the new minimum (min of current value and previous minimum) to the min stack.

C# Solution

C# 
public class MinStack {
    private Stack<int> stack;
    private Stack<int> minStack;

    public MinStack() {
        stack = new Stack<int>();
        minStack = new Stack<int>();
    }

    public void Push(int val) {
        stack.Push(val);
        int minVal = minStack.Count == 0
            ? val
            : Math.Min(val, minStack.Peek());
        minStack.Push(minVal);
    }

    public void Pop() {
        stack.Pop();
        minStack.Pop();
    }

    public int Top() => stack.Peek();

    public int GetMin() => minStack.Peek();
}
Time
O(1) each
Space
O(n)

Trace: Push 3, 1, 4, 1, 5; GetMin(); Pop

1
Push(3): stack=[3], minStack=[3]
2
Push(1): stack=[3,1], minStack=[3,1]
3
Push(4): stack=[3,1,4], minStack=[3,1,1]
4
GetMin(): minStack.Peek()=1 ✓
5
Pop(): stack=[3,1], minStack=[3,1]

Edge Cases

  • Single element → getMin equals that element
  • Duplicates → minStack stores each duplicate
  • Negative numbers → minStack handles correctly
→
FAANG Tip: Can optimize space by only pushing to minStack when value ≤ current min. Requires storing the value in the main pop logic.
Evaluate Reverse Polish Notation

Given an array of tokens representing a mathematical expression in Reverse Polish Notation (RPN), evaluate and return the result.

Medium Microsoft Apple
3

Problem Statement

In RPN, the operator follows the operands. For example, "3 4 +" means 3 + 4. Evaluate the expression and return the result as an integer.

Approach: Stack

Iterate through tokens. Push numbers; when you see an operator, pop two operands, apply the operation, and push the result back. Final stack contains one element: the answer.

C# Solution

C# 
public int EvalRPN(string[] tokens) {
    var stack = new Stack<long>();
    var ops = new HashSet<string>() { "+", "-", "*", "/" };

    foreach (string token in tokens) {
        if (ops.Contains(token)) {
            long b = stack.Pop();
            long a = stack.Pop();
            long result = token switch {
                "+" => a + b,
                "-" => a - b,
                "*" => a * b,
                "/" => a / b,
                _ => 0
            };
            stack.Push(result);
        } else {
            stack.Push(long.Parse(token));
        }
    }

    return (int)stack.Peek();
}
Time
O(n)
Space
O(n)

Trace: ["2", "1", "+", "3", "*"]

1
Push 2: stack=[2]
2
Push 1: stack=[2, 1]
3
Op '+': pop 1, pop 2, push 2+1=3. stack=[3]
4
Push 3: stack=[3, 3]
5
Op '*': pop 3, pop 3, push 3*3=9. Result=9

Edge Cases

  • Division by zero → undefined (assume valid input)
  • Integer overflow in intermediate results → use long
  • Negative operands → handle correctly with subtraction/division
→
FAANG Tip: Use long to avoid overflow before casting back to int. Mention division truncates towards zero in C#, which differs from Python.
Generate Parentheses

Given n pairs of parentheses, generate all combinations of well-formed parentheses.

Medium Google Facebook
4

Problem Statement

Given an integer n, generate all combinations of n pairs of well-matched parentheses as a list of strings.

Approach: Backtracking

Use a recursive helper that tracks open and close counts. Add '(' if open < n, add ')' if close < open. This ensures we never close more than we open and always have valid parentheses.

C# Solution

C# 
public IList<string> GenerateParenthesis(int n) {
    var result = new List<string>();
    Backtrack(result, "", 0, 0, n);
    return result;
}

private void Backtrack(IList<string> result, string current,
                        int open, int close, int n) {
    if (open == n && close == n) {
        result.Add(current);
        return;
    }

    if (open < n) {
        Backtrack(result, current + "(", open + 1, close, n);
    }

    if (close < open) {
        Backtrack(result, current + ")", open, close + 1, n);
    }
}
Time
O(4^n / √n)
Space
O(n)

Trace: n=2

1
Start: "", open=0, close=0
2
Add '(': "(", open=1, close=0
3
Add '(': "((", open=2, close=0
4
Add ')': "(()", open=2, close=1 → "())" → valid
5
Backtrack, try "()" → "()()" valid

Edge Cases

  • n=0 → empty string (one valid combination)
  • n=1 → one valid combination "()"
  • n=3 → 5 valid combinations (Catalan number C_3)
→
FAANG Tip: The count of valid parentheses strings of length n is the nth Catalan number. This grows exponentially; mention why optimization via memoization isn't helpful here (state space is huge).
Daily Temperatures

Given daily temperatures, for each day find the number of days you have to wait until you get a warmer temperature. If no future day is warmer, put 0.

Medium Google Meta
5

Problem Statement

Given an array of daily temperatures, return an array where each element is the number of days you have to wait from that day to get a warmer temperature.

Approach: Monotonic Stack

Traverse from right to left, maintaining a decreasing stack of indices. For each temperature, pop all smaller temperatures (they're blocked by the current one), then the stack top is the next warmer day.

C# Solution

C# 
public int[] DailyTemperatures(int[] temperatures) {
    int n = temperatures.Length;
    int[] result = new int[n];
    var stack = new Stack<int>(); // store indices

    for (int i = n - 1; i >= 0; i--) {
        // Pop all indices with smaller or equal temperatures
        while (stack.Count > 0 &&
               temperatures[stack.Peek()] <= temperatures[i]) {
            stack.Pop();
        }

        // Stack top is next warmer day (or empty if none)
        result[i] = stack.Count > 0 ? stack.Peek() - i : 0;
        stack.Push(i);
    }

    return result;
}
Time
O(n)
Space
O(n)

Trace: [73, 74, 75, 71, 69, 72, 76, 73]

1
i=7 (73): stack empty → result[7]=0. Push 7. stack=[7]
2
i=6 (76): 73 < 76, pop. stack empty → result[6]=0. Push 6. stack=[6]
3
i=5 (72): 76 > 72, don't pop → result[5]=6-5=1. Push 5. stack=[6,5]
4
Continue rightward...
5
Final: [1, 1, 4, 2, 1, 1, 0, 0]

Edge Cases

  • Strictly increasing temperatures → stack grows, all pops happen right-to-left
  • Strictly decreasing temperatures → all results 0 (no warmer day)
  • All equal temperatures → all results 0
→
FAANG Tip: This is the canonical monotonic stack problem. Emphasize the right-to-left traversal and the invariant that the stack is always decreasing.
Largest Rectangle in Histogram

Given an array of bar heights, find the largest area of a rectangle that can be formed.

Hard Google Microsoft
6

Problem Statement

Given an array heights representing bar heights, find the maximal area of a rectangle that can be formed by selecting contiguous bars.

Approach: Monotonic Stack

For each bar, find the leftmost and rightmost bars that are at least as tall. Use a monotonic stack to track indices. For each bar popped, calculate the area with that bar's height using the span between the popped index and current index.

C# Solution

C# 
public int LargestRectangleArea(int[] heights) {
    var stack = new Stack<int>();
    int maxArea = 0;
    int n = heights.Length;

    for (int i = 0; i <= n; i++) {
        int h = i == n ? 0 : heights[i];

        // Pop and calculate area for all taller bars
        while (stack.Count > 0 && heights[stack.Peek()] > h) {
            int idx = stack.Pop();
            int width = stack.Count > 0 ? i - stack.Peek() - 1 : i;
            int area = heights[idx] * width;
            maxArea = Math.Max(maxArea, area);
        }

        if (i < n) stack.Push(i);
    }

    return maxArea;
}
Time
O(n)
Space
O(n)

Trace: [2, 1, 5, 6, 2, 3]

1
Push 0 (h=2). stack=[0]
2
i=1 (h=1): 2>1, pop 0. Width=1, area=2. Push 1. stack=[1]
3
i=2 (h=5): 1<5, push 2. stack=[1,2]
4
i=3 (h=6): 5<6, push 3. stack=[1,2,3]
5
i=4 (h=2): pop 3,2; max area=10 (h=6, w=1; h=5, w=2)

Edge Cases

  • Single bar → area = height of that bar
  • All equal heights → area = height × length
  • Empty array → area = 0
→
FAANG Tip: The key insight is adding a sentinel 0 at the end to flush the stack. When you pop, the width spans from the new stack top to current index.
Car Fleet

N cars are heading to a destination. Each car has a position and speed. A car catches up to the car in front and they form a fleet. Find the number of fleets.

Medium Google Meta
7

Problem Statement

Given target distance, positions, and speeds of N cars, determine how many car fleets will reach the target. Cars that catch up form a single fleet.

Approach: Sort & Stack

Sort by position. For each car from right to left, compute time to target. If a car's time is greater than the previous (slower) car's time, it's a new fleet. Use a stack to track fleet times.

C# Solution

C# 
public int CarFleet(int target, int[] position, int[] speed) {
    if (position.Length == 0) return 0;

    // Create array of (position, speed) and sort by position
    var cars = new List<(int, int)>();
    for (int i = 0; i < position.Length; i++) {
        cars.Add((position[i], speed[i]));
    }
    cars.Sort();

    var stack = new Stack<double>(); // store times

    for (int i = cars.Count - 1; i >= 0; i--) {
        double time = (double)(target - cars[i].Item1) / cars[i].Item2;

        // If faster than car ahead, it's a new fleet
        if (stack.Count == 0 || time > stack.Peek()) {
            stack.Push(time);
        }
    }

    return stack.Count;
}
Time
O(n log n)
Space
O(n)

Trace: target=12, position=[10,8,0,5,3], speed=[2,4,1,1,3]

1
Sort cars by position: [(0,1), (3,3), (5,1), (8,4), (10,2)]
2
From right: (10,2) time=1. stack=[1]
3
(8,4) time=1. 1≤1, merge. stack=[1]
4
(5,1) time=7. 7>1, new fleet. stack=[1,7]
5
Result: 3 fleets

Edge Cases

  • Single car → 1 fleet
  • All cars have same speed → 1 fleet (if sorted by position)
  • Car already at target → time = 0
→
FAANG Tip: The stack stores times. A new fleet is formed only if the current car's time exceeds the previous (slower) fleet's time.
Implement Queue using Stacks

Implement a queue using two stacks. Support enqueue, dequeue, peek, and empty operations.

Easy Apple Meta
8

Problem Statement

Design a queue class that uses two stacks. Enqueue should be O(1), and amortized dequeue should be O(1).

Approach: Two Stacks

Use one stack for enqueue, and flip everything to another stack for dequeue. On dequeue, if the dequeue stack is empty, reverse the enqueue stack into it.

C# Solution

C# 
public class MyQueue {
    private Stack<int> inStack;   // for enqueue
    private Stack<int> outStack;  // for dequeue

    public MyQueue() {
        inStack = new Stack<int>();
        outStack = new Stack<int>();
    }

    public void Push(int x) {
        inStack.Push(x);
    }

    public int Pop() {
        Peek(); // Ensure outStack is filled
        return outStack.Pop();
    }

    public int Peek() {
        if (outStack.Count == 0) {
            // Transfer all from inStack to outStack
            while (inStack.Count > 0) {
                outStack.Push(inStack.Pop());
            }
        }
        return outStack.Peek();
    }

    public bool Empty() {
        return inStack.Count == 0 && outStack.Count == 0;
    }
}
Push
O(1)
Pop
O(1) amortized
Space
O(n)

Trace: Push 1,2,3; Pop; Pop

1
Push(1): inStack=[1], outStack=[]
2
Push(2): inStack=[1,2], outStack=[]
3
Push(3): inStack=[1,2,3], outStack=[]
4
Pop(): Peek() flips. inStack=[], outStack=[3,2,1]
5
Return 1, Pop outStack

Edge Cases

  • Queue with one element → Pop returns it
  • Multiple Peek calls → outStack unchanged after first flip
  • Interleaved Push/Pop → both stacks can be non-empty
→
FAANG Tip: Each element is pushed once and popped once. The amortized analysis gives O(1) per operation. Explain why flipping is lazy (only done when outStack empties).
Implement Stack using Queues

Implement a stack using one or two queues. Support push, pop, top, and empty operations.

Easy Google
9

Problem Statement

Design a stack class using queue(s). Push, pop, top, and empty must work correctly.

Approach: Single Queue

Use one queue. On push, add the element and rotate the queue so the new element is at the front. On pop, simply dequeue. This makes pop O(1) but push O(n).

C# Solution

C# 
public class MyStack {
    private Queue<int> queue;

    public MyStack() {
        queue = new Queue<int>();
    }

    public void Push(int x) {
        queue.Enqueue(x);
        // Rotate so new element is at front
        for (int i = 0; i < queue.Count - 1; i++) {
            queue.Enqueue(queue.Dequeue());
        }
    }

    public int Pop() {
        return queue.Dequeue();
    }

    public int Top() {
        return queue.Peek();
    }

    public bool Empty() {
        return queue.Count == 0;
    }
}
Push
O(n)
Pop
O(1)
Space
O(n)

Trace: Push 1, 2; Pop; Top

1
Push(1): queue=[1]
2
Push(2): enqueue 2, queue=[1,2]. Rotate: queue=[2,1]
3
Pop(): dequeue from queue=[2,1]. Return 2
4
Top(): peek queue=[1]. Return 1

Edge Cases

  • Single element → Pop and Top work correctly
  • Empty stack → Pop/Top throw (inherit Queue behavior)
→
FAANG Tip: Trade-off: push is expensive (O(n)) but pop is cheap (O(1)). Contrast with a two-queue solution where push is O(1) but pop is O(n).
Next Greater Element I

Given two arrays, find the next greater element in the second array for each element in the first array.

Easy Google
10

Problem Statement

Given two arrays (first is a subset of second), find the next greater element in the second array for each element in the first array. If no such element exists, return -1.

Approach: Monotonic Stack + HashMap

Process the second array right-to-left using a decreasing stack. For each element, pop all smaller elements, then the stack top (if exists) is the next greater. Store results in a HashMap.

C# Solution

C# 
public int[] NextGreaterElement(int[] nums1, int[] nums2) {
    var map = new Dictionary<int, int>();
    var stack = new Stack<int>();

    // Process nums2 from right to left
    for (int i = nums2.Length - 1; i >= 0; i--) {
        // Pop all smaller elements
        while (stack.Count > 0 && stack.Peek() <= nums2[i]) {
            stack.Pop();
        }

        // Stack top is next greater (or -1 if empty)
        map[nums2[i]] = stack.Count > 0 ? stack.Peek() : -1;
        stack.Push(nums2[i]);
    }

    // Build result for nums1
    int[] result = new int[nums1.Length];
    for (int i = 0; i < nums1.Length; i++) {
        result[i] = map[nums1[i]];
    }
    return result;
}
Time
O(n+m)
Space
O(m)

Trace: nums1=[1,3,4], nums2=[1,3,4,2]

1
i=3 (2): stack empty, map[2]=-1. Push 2. stack=[2]
2
i=2 (4): 2≤4, pop. stack empty, map[4]=-1. Push 4. stack=[4]
3
i=1 (3): 4>3, no pop. map[3]=4. Push 3. stack=[4,3]
4
i=0 (1): 3>1, 4>1, no pop. map[1]=4. Push 1.
5
Result: [4, 4, -1]

Edge Cases

  • nums1 and nums2 are the same → all or mostly -1
  • nums2 is strictly decreasing → all -1
  • nums2 is strictly increasing → all have next greater (except last)
→
FAANG Tip: HashMap decouples the two arrays. You could also use TryGetValue for safe lookups if nums1 had elements not in nums2 (violates problem spec but good defensive coding).
Decode String

Given an encoded string, decode it. For example, "3[a2[c]]" becomes "accaccacc".

Medium Google Facebook
11

Problem Statement

The pattern is k[encoded_string] where k is a number. Nested encodings are allowed. Decode and return the final string.

Approach: Stack

Use a stack of strings. Iterate through characters: push non-bracket characters, push strings when hitting ']' by popping and reconstructing with the multiplier.

C# Solution

C# 
public string DecodeString(string s) {
    var stack = new Stack<string>();
    string current = "";
    int num = 0;

    foreach (char c in s) {
        if (char.IsDigit(c)) {
            num = num * 10 + (c - '0');
        } else if (c == '[') {
            // Push current string and number to stack
            stack.Push(current);
            stack.Push(num.ToString());
            current = "";
            num = 0;
        } else if (c == ']') {
            // Pop multiplier and previous string
            int count = int.Parse(stack.Pop());
            string prev = stack.Pop();
            current = prev + Repeat(current, count);
        } else {
            current += c;
        }
    }

    return current;
}

private string Repeat(string s, int count) {
    var sb = new StringBuilder();
    for (int i = 0; i < count; i++) {
        sb.Append(s);
    }
    return sb.ToString();
}
Time
O(n×m)
Space
O(n)

Trace: "3[a2[c]]"

1
See '3': num=3
2
See '[': push "", push "3". current="", num=0
3
See 'a': current="a"
4
See '2': num=2
5
See '[': push "a", push "2". current="", num=0
6
See 'c': current="c"
7
See ']': pop 2, pop "a". current="a"+"cc"="acc"
8
See ']': pop 3, pop "". current=""+repeat("acc",3)="accaccacc"

Edge Cases

  • Multi-digit numbers (e.g., "23[a]") → parse correctly with num*10+digit
  • Deeply nested (e.g., "2[2[2[a]]]") → stack handles recursion
  • No brackets (e.g., "abc") → return as-is
→
FAANG Tip: Use StringBuilder for repeated string concatenation to avoid O(n²) quadratic time. The stack of strings interleaves multipliers and decoded substrings elegantly.
Basic Calculator II

Implement a basic calculator supporting +, -, *, / with correct operator precedence. No parentheses.

Medium Google Microsoft
12

Problem Statement

Given a string expression with integers and operators (+, -, *, /), evaluate and return the result respecting operator precedence (* and / before + and -).

Approach: Stack

Parse the expression left-to-right. Track the last operator. For +/-, push the number (or its negation) to the stack. For */, pop the stack, compute, and push back. At the end, sum the stack.

C# Solution

C# 
public int Calculate(string s) {
    var stack = new Stack<long>();
    long num = 0;
    char op = '+';

    for (int i = 0; i < s.Length; i++) {
        char c = s[i];

        // Parse number
        if (char.IsDigit(c)) {
            num = num * 10 + (c - '0');
        }

        // Process when we hit an operator or end of string
        if (!char.IsDigit(c) && c != ' ' || i == s.Length - 1) {
            switch (op) {
                case '+':
                    stack.Push(num);
                    break;
                case '-':
                    stack.Push(-num);
                    break;
                case '*':
                    stack.Push(stack.Pop() * num);
                    break;
                case '/':
                    stack.Push(stack.Pop() / num);
                    break;
            }
            num = 0;
            if (c == '+' || c == '-' || c == '*' || c == '/') {
                op = c;
            }
        }
    }

    // Sum the stack
    long result = 0;
    while (stack.Count > 0) {
        result += stack.Pop();
    }
    return (int)result;
}
Time
O(n)
Space
O(n)

Trace: "14-12/2"

1
Parse 1, num=1
2
Parse 4, num=14. See '-': push +14, op='-'
3
Parse 12, num=12. See '/': push -12 (op was '-'), op='/'
4
Parse 2, num=2. End: pop, 12/2=6, push 6
5
Sum: 14 + (-12) + 6 = 8

Edge Cases

  • Spaces in input (e.g., "3 + 5") → skip them
  • Multi-digit numbers → parse with num*10+digit
  • Integer division → truncates towards zero (use long and cast)
  • Leading zeros (e.g., "007") → parsed as 7
→
FAANG Tip: The trick is processing * and / immediately (by popping and re-pushing) while deferring + and - to the final summation. This elegantly handles precedence. Use long to avoid overflow.
