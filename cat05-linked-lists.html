INTRO

Linked List Fundamentals

Linked lists are the foundation of pointer-based data structures. Unlike arrays, they excel at insertion and deletion but require careful pointer manipulation.

C# ListNode Definition

Every linked list problem requires a ListNode class:

C# 
public class ListNode {
    public int val;
    public ListNode next;
    public ListNode(int val = 0, ListNode next = null) {
        this.val = val;
        this.next = next;
    }
}

Core Techniques

TECHNIQUE 01
Dummy Head Pattern
Create a sentinel node before the actual head to simplify edge cases. Eliminates special handling for operations on the first node.
TECHNIQUE 02
Two-Pointer Technique
Use fast and slow pointers for cycle detection, finding the middle, or identifying the k-th node from end without double traversal.
TECHNIQUE 03
In-Place Reversal
Reverse segments by updating three pointers: prev, current, and next. Track the previous node before breaking the link forward.
01
Reverse Linked List

Reverse all pointers to reverse the entire linked list. Implement both iterative and recursive solutions.

Easy Apple Google
1

Problem Statement

Given the head of a singly linked list, reverse the list and return the new head. Do this in-place with O(1) extra space.

Approach: Iterative Reversal

1
Initialize prev = null, current = head
2
For each node, save the next pointer before reversing
3
Point current.next to prev, move both pointers forward
4
Return prev as the new head

C# Solution - Iterative

C# 
public ListNode ReverseList(ListNode head) {
    ListNode prev = null;
    ListNode current = head;

    while (current != null) {
        ListNode nextTemp = current.next;  // Save next
        current.next = prev;             // Reverse the link
        prev = current;                // Move prev forward
        current = nextTemp;             // Move current forward
    }

    return prev;
}

C# Solution - Recursive

C# 
public ListNode ReverseListRecursive(ListNode head) {
    if (head == null || head.next == null) {
        return head;
    }

    ListNode newHead = ReverseListRecursive(head.next);
    head.next.next = head;  // Reverse the link
    head.next = null;         // Remove forward link

    return newHead;
}
Time
O(n)
Space
O(1) iter / O(n) rec

Trace: 1→2→3→null

Iterative
prev=null, curr=1 → prev=1, curr=2 → prev=2, curr=3 → prev=3, curr=null. Return 3.
Recursive
Unwind stack: 3 becomes head. Reverse 3→null, 2→3, 1→2. Result: 3→2→1→null.
💡
FAANG Tip: Interviewers often ask for both iterative and recursive. Iterative is preferred in production for space efficiency. Recursive shows understanding of stack unwinding.
02
Merge Two Sorted Lists

Merge two ascending linked lists into one. Use dummy head to avoid edge cases at the beginning.

Easy Facebook Amazon
2

Problem Statement

Merge two sorted linked lists into a single sorted list. Return the head of the merged list.

Approach: Dummy Head Pattern

1
Create a dummy node to simplify head insertion
2
Compare heads of both lists and append the smaller
3
Advance the pointer in the list we just took from
4
Attach remaining list and return dummy.next

C# Solution

C# 
public ListNode MergeTwoLists(ListNode list1, ListNode list2) {
    ListNode dummy = new ListNode(0);
    ListNode current = dummy;

    while (list1 != null && list2 != null) {
        if (list1.val <= list2.val) {
            current.next = list1;
            list1 = list1.next;
        } else {
            current.next = list2;
            list2 = list2.next;
        }
        current = current.next;
    }

    // Attach remaining
    current.next = list1 != null ? list1 : list2;

    return dummy.next;
}
Time
O(n + m)
Space
O(1)

Edge Cases

Empty lists: Either list can be null. Attach the non-null list directly.

Single elements: Dummy node handles the first append without special logic.

💡
FAANG Tip: The dummy head pattern is essential for simplifying list construction. Mention it explicitly during coding to show you understand this idiom.
03
Linked List Cycle

Detect if a linked list contains a cycle using Floyd's tortoise and hare algorithm.

Easy Facebook Microsoft
3

Problem Statement

Determine if a linked list has a cycle. Return true if there is a cycle, false otherwise. Use O(1) space.

Approach: Floyd's Cycle Detection

1
slow moves 1 step, fast moves 2 steps each iteration
2
If a cycle exists, fast and slow will eventually meet
3
If fast reaches null, no cycle exists

C# Solution

C# 
public bool HasCycle(ListNode head) {
    if (head == null || head.next == null) {
        return false;
    }

    ListNode slow = head;
    ListNode fast = head;

    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;

        if (slow == fast) {
            return true;
        }
    }

    return false;
}
Time
O(n)
Space
O(1)

Why Floyd's Works

In a cycle of length C, the relative speed difference (1 step) means fast catches slow after at most n iterations. The key insight: if they meet, a cycle exists; if fast reaches null, no cycle.

💡
FAANG Tip: Floyd's algorithm is fundamental for cycle detection. Understanding why the relative speed guarantees a meeting is crucial. Expect follow-ups about finding the cycle start node.
04
Linked List Cycle II

Find the node where the cycle begins. Extend Floyd's algorithm with a second phase.

Medium Amazon Apple
4

Problem Statement

Given a linked list, return the node where the cycle begins. If no cycle, return null.

Approach: Floyd's + Phase Two

1
Detect cycle using Floyd's algorithm
2
Reset one pointer to head, keep other at meeting point
3
Move both one step at a time until they meet again
4
Meeting point is the cycle start

C# Solution

C# 
public ListNode DetectCycleStart(ListNode head) {
    if (head == null) return null;

    ListNode slow = head, fast = head;

    // Phase 1: Detect cycle
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) break;
    }

    if (fast == null || fast.next == null) {
        return null;  // No cycle
    }

    // Phase 2: Find cycle start
    slow = head;
    while (slow != fast) {
        slow = slow.next;
        fast = fast.next;
    }

    return slow;
}
Time
O(n)
Space
O(1)

Mathematical Insight

If distance from head to cycle start is a and cycle length is c, when slow and fast meet, slow has traveled a + b (where b is distance in cycle) and fast has traveled a + b + c. Since fast = 2 × slow, we can derive that resetting slow to head and moving both one step will meet at the cycle start.

⚠️
Edge Case: Ensure phase 2 only runs if a cycle was actually detected. Check if fast is null or fast.next is null before proceeding.
05
Remove Nth Node From End

Remove the n-th node from the end of a list in a single pass using two pointers.

Medium Facebook Google
5

Problem Statement

Remove the n-th node from the end of the list. Do it in one pass. If n equals the list length, remove the head.

Approach: Two-Pointer Gap

1
Create a dummy node pointing to head to handle edge cases
2
Advance first pointer n+1 steps ahead
3
Move both pointers until first reaches the end
4
Remove the target node by updating pointers

C# Solution

C# 
public ListNode RemoveNthFromEnd(ListNode head, int n) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode first = dummy;
    ListNode second = dummy;

    // Advance first pointer n+1 steps
    for (int i = 0; i <= n; i++) {
        if (first == null) return head;  // n > length
        first = first.next;
    }

    // Move both until first reaches end
    while (first != null) {
        first = first.next;
        second = second.next;
    }

    // Remove the node
    second.next = second.next.next;

    return dummy.next;
}
Time
O(n)
Space
O(1)
Passes
1

Why The Dummy Node?

The dummy node ensures we can safely remove the head. Without it, removing the first node requires special logic. The dummy acts as a sentinel.

💡
FAANG Tip: Always mention the dummy node pattern when handling head removal. Show you understand both the space constraint (O(1)) and the one-pass requirement.
06
Reorder List

Reorder list as: L0 → Ln → L1 → Ln-1 → ... by finding middle, reversing, and merging.

Medium Google Amazon
6

Problem Statement

Given a list, reorder it to: L0 → Ln → L1 → Ln-1 → L2 → Ln-2 → ... Modify in place.

Approach: Three-Step Process

1
Find the middle of the list using slow/fast pointers
2
Reverse the second half of the list
3
Merge the two halves by interleaving nodes

C# Solution

C# 
public void ReorderList(ListNode head) {
    if (head == null || head.next == null) return;

    // Step 1: Find middle
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    // Step 2: Reverse second half
    ListNode mid = slow;
    ListNode prev = null, curr = slow;
    while (curr != null) {
        ListNode temp = curr.next;
        curr.next = prev;
        prev = curr;
        curr = temp;
    }

    // Step 3: Merge halves
    ListNode l1 = head, l2 = prev;
    while (l2.next != null) {
        ListNode t1 = l1.next, t2 = l2.next;
        l1.next = l2;
        l2.next = t1;
        l1 = t1;
        l2 = t2;
    }
}
Time
O(n)
Space
O(1)

Trace: 1→2→3→4→5

After finding middle: slow at 3. After reversing second half: second half is 5→4→3. After merging: 1→5→2→4→3.

🔴
Common Mistake: Not breaking the first half from the second. Ensure mid.next = null after finding the middle to avoid cycles.
07
Add Two Numbers

Add two numbers represented as linked lists. Simulate digit-by-digit addition with carry.

Medium Apple Facebook
7

Problem Statement

Add two numbers represented by two lists where each node contains a single digit in reverse order. Return the sum as a linked list in reverse order.

Approach: Carry Propagation

1
Initialize a dummy node and carry variable
2
Traverse both lists, adding digits with carry
3
Create new node with (sum % 10) and update carry
4
If carry remains after both lists, create extra node

C# Solution

C# 
public ListNode AddTwoNumbers(ListNode l1, ListNode l2) {
    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;
    int carry = 0;

    while (l1 != null || l2 != null || carry > 0) {
        int sum = carry;

        if (l1 != null) {
            sum += l1.val;
            l1 = l1.next;
        }

        if (l2 != null) {
            sum += l2.val;
            l2 = l2.next;
        }

        carry = sum / 10;
        curr.next = new ListNode(sum % 10);
        curr = curr.next;
    }

    return dummy.next;
}
Time
O(max(n, m))
Space
O(max(n, m))

Example: 342 + 465

Represented as 2→4→3 and 5→6→4. Sum: 7→0→8 (807). Digit-by-digit: 2+5=7, 4+6+0=10 (carry 1), 3+4+1=8.

💡
FAANG Tip: The while condition includes carry to handle the final digit. A common mistake is exiting when both lists are null but carry is non-zero, resulting in lost digits.

Copy List with Random Pointer

Clone a linked list where each node has a random pointer to any node or null.

Medium Microsoft Facebook
8

Problem Statement

Clone a linked list where each node has a next and a random pointer. The random pointer can point to any node or null.

Approach: Hash Map Solution

1
First pass: create all nodes and map old→new
2
Second pass: set next and random pointers using map
3
Return the cloned head

C# Solution - Hash Map

C# 
public class Node {
    public int val;
    public Node next;
    public Node random;
}

public Node CopyRandomList(Node head) {
    if (head == null) return null;

    var map = new Dictionary<Node, Node>();

    // Pass 1: Create all nodes
    Node curr = head;
    while (curr != null) {
        map[curr] = new Node(curr.val);
        curr = curr.next;
    }

    // Pass 2: Set pointers
    curr = head;
    while (curr != null) {
        map[curr].next = curr.next != null ? map[curr.next] : null;
        map[curr].random = curr.random != null ? map[curr.random] : null;
        curr = curr.next;
    }

    return map[head];
}
Time
O(n)
Space
O(n)

Alternative: Interweave + Unweave

In-place solution using O(1) space: interweave the original and cloned lists, copy random pointers, then unweave. More complex but demonstrates deep understanding.

⚠️
Note: The interweave approach is interview gold for showing problem-solving creativity, but the hash map approach is clearer and less error-prone.

Merge K Sorted Lists

Merge K sorted linked lists into one using min-heap or divide-and-conquer.

Hard Google Amazon
9

Problem Statement

Merge K sorted linked lists and return as one sorted list. Lists may be empty.

Approach: Min-Heap Priority Queue

1
Push the head of each list into a min-heap
2
Pop the minimum node, append to result
3
Push the next node of the popped node into heap
4
Repeat until heap is empty

C# Solution - Min-Heap

C# 
public ListNode MergeKLists(ListNode[] lists) {
    if (lists == null || lists.Length == 0) return null;

    PriorityQueue<ListNode, int> heap = new PriorityQueue<ListNode, int>();

    // Add all list heads to heap
    foreach (var list in lists) {
        if (list != null) {
            heap.Enqueue(list, list.val);
        }
    }

    ListNode dummy = new ListNode(0);
    ListNode curr = dummy;

    while (heap.Count > 0) {
        ListNode minNode = heap.Dequeue();
        curr.next = minNode;
        curr = curr.next;

        if (minNode.next != null) {
            heap.Enqueue(minNode.next, minNode.next.val);
        }
    }

    return dummy.next;
}
Time
O(n log k)
Space
O(k)
k
Number of lists

Alternative: Divide and Conquer

Merge lists pairwise until one remains. Time: O(n log k). More space-efficient alternative to heap when k is large.

💡
FAANG Tip: Heap approach is intuitive. Mention divide-and-conquer as an optimization if you want to showcase depth. Interviewers appreciate both.
10
Reverse Nodes in K-Group

Reverse every k nodes in the list. Groups at the end that have fewer than k nodes are left unchanged.

Hard Google Facebook
10

Problem Statement

Given a linked list, reverse the nodes in groups of k. If the number of nodes is not a multiple of k, leave the last group in original order.

Approach: Iterative K-Group Reversal

1
Check if there are k nodes ahead to process
2
Reverse the k nodes iteratively
3
Connect previous group to reversed group
4
Move to next group and repeat

C# Solution

C# 
public ListNode ReverseKGroup(ListNode head, int k) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode prev = dummy;

    while (true) {
        // Check if there are k nodes ahead
        ListNode kth = prev;
        for (int i = 0; i < k; i++) {
            kth = kth.next;
            if (kth == null) return dummy.next;
        }

        ListNode groupStart = prev.next;

        // Reverse k nodes
        ListNode p = null, c = groupStart;
        for (int i = 0; i < k; i++) {
            ListNode temp = c.next;
            c.next = p;
            p = c;
            c = temp;
        }

        // Connect
        prev.next = p;
        groupStart.next = c;
        prev = groupStart;
    }
}
Time
O(n)
Space
O(1)

Trace: 1→2→3→4→5, k=2

Group 1 (1,2): reverse to 2→1. Group 2 (3,4): reverse to 4→3. Result: 2→1→4→3→5. Node 5 is left unchanged.

🔴
Critical: Always check if k nodes exist before reversing. Failing to do so causes incorrect reversals of partial groups.

LRU Cache

Design an LRU Cache with get and put operations in O(1) time using a doubly linked list and hash map.

Medium Amazon Google
11

Problem Statement

Design an LRU (Least Recently Used) Cache with capacity limit. Implement Get(key) and Put(key, value) in O(1) time.

Approach: Doubly Linked List + Hash Map

1
Use doubly linked list for efficient add/remove at both ends
2
Use hash map for O(1) node lookup by key
3
On access, move node to end (most recent)
4
When full, remove node at head (least recent)

C# Solution

C# 
public class Node {
    public int key, val;
    public Node prev, next;
}

public class LRUCache {
    int capacity;
    Dictionary<int, Node> cache;
    Node head, tail;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        cache = new Dictionary<int, Node>();
        head = new Node();
        tail = new Node();
        head.next = tail;
        tail.prev = head;
    }

    public int Get(int key) {
        if (!cache.ContainsKey(key)) return -1;
        Node node = cache[key];
        Remove(node);
        AddToEnd(node);
        return node.val;
    }

    public void Put(int key, int value) {
        if (cache.ContainsKey(key)) {
            Remove(cache[key]);
        }
        Node newNode = new Node { key = key, val = value };
        cache[key] = newNode;
        AddToEnd(newNode);
        if (cache.Count > capacity) {
            Node lru = head.next;
            Remove(lru);
            cache.Remove(lru.key);
        }
    }

    private void Remove(Node node) {
        node.prev.next = node.next;
        node.next.prev = node.prev;
    }

    private void AddToEnd(Node node) {
        node.prev = tail.prev;
        node.next = tail;
        tail.prev.next = node;
        tail.prev = node;
    }
}
Get Time
O(1)
Put Time
O(1)
Space
O(capacity)

Why Doubly Linked List?

Doubly linked list allows O(1) removal from the middle (when moving accessed nodes). A singly linked list cannot efficiently remove a node without knowing its predecessor.

💡
FAANG Tip: LRU Cache is a classic system design question. Mentioning sentinel nodes (head/tail) and the removal pattern shows deep understanding of data structure design.

Palindrome Linked List

Check if a linked list is a palindrome by reversing the second half and comparing with the first.

Medium Facebook Amazon
12

Problem Statement

Given a singly linked list, determine if it is a palindrome. Do it in O(n) time and O(1) space.

Approach: Reverse Second Half

1
Find the middle using slow/fast pointers
2
Reverse the second half of the list
3
Compare first half with reversed second half
4
Return true if all values match

C# Solution

C# 
public bool IsPalindrome(ListNode head) {
    if (head == null || head.next == null) return true;

    // Find middle
    ListNode slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }

    // Reverse second half
    ListNode prev = null, curr = slow;
    while (curr != null) {
        ListNode temp = curr.next;
        curr.next = prev;
        prev = curr;
        curr = temp;
    }

    // Compare
    ListNode l1 = head, l2 = prev;
    while (l2 != null) {  // l2 is shorter for odd-length lists
        if (l1.val != l2.val) return false;
        l1 = l1.next;
        l2 = l2.next;
    }

    return true;
}
Time
O(n)
Space
O(1)

Trace: 1→2→2→1

After finding middle: slow at second 2. After reversing: 1→2. Compare: 1==1, 2==2. Result: true.

⚠️
Note: For odd-length lists (1→2→3→2→1), slow starts at the middle element. The comparison works because l2 (reversed second half) is shorter and we check l2 != null.
💡
FAANG Tip: This problem tests mastery of three concepts: finding the middle (slow/fast), reversing, and two-pointer comparison. Solving it cleanly in one pass demonstrates strong linked list intuition.
