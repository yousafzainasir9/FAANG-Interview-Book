Intro

Heap & Priority Queue Fundamentals

Heaps are complete binary trees that maintain a specific ordering property. In C#, PriorityQueue<T,P> (.NET 6+) provides O(log n) insertions and deletions while maintaining efficient access to the extremum.

Core Concepts

Min-Heap: Parent ≤ children. Root is smallest. Use for "find minimum" or "n smallest" problems.

Max-Heap: Parent ≥ children. Root is largest. Use for "find maximum" or "n largest" problems.

Complete Binary Tree: All levels filled except possibly the last, which is filled left-to-right. Enables array-based storage with O(1) parent/child access.

C# PriorityQueue Usage

C# 
// Min-heap: smaller priority comes out first
var minHeap = new PriorityQueue<int, int>();
minHeap.Enqueue(5, 5);
minHeap.Enqueue(3, 3);
int min = minHeap.Dequeue(); // 3

// Max-heap: negate priority for reverse ordering
var maxHeap = new PriorityQueue<int, int>();
maxHeap.Enqueue(5, -5);
maxHeap.Enqueue(3, -3);
int max = maxHeap.Dequeue(); // 5

Key Patterns

Pattern 1
Top-K Elements
Use min-heap of size K. Insert all, keep top K by evicting smallest when size exceeds K.
Pattern 2
Two-Heap Pattern
Max-heap for lower half, min-heap for upper half. Maintain balance to find median.
Pattern 3
Greedy with Max-Heap
Extract max repeatedly and apply a constraint or operation. Common in scheduling and reorganization.

Complexity Reference

Operation Time Notes
Insert (Enqueue) O(log n) Bubble up from leaf
Delete Min/Max (Dequeue) O(log n) Replace root, sink down
Peek Min/Max O(1) Access root without removal
Heapify n items O(n) Build heap bottom-up
Heap sort O(n log n) Heapify + extract n times
Kth Largest Element in Array
Given unsorted array and integer k, return the kth largest element (not kth distinct).
MEDIUM Amazon Apple
1

Approach: Min-Heap of Size K

Maintain a min-heap with exactly k largest elements. When heap size exceeds k, remove the smallest (root). Final root is kth largest. Time O(n log k), space O(k).

Solution

C# 
public class Solution {
    public int FindKthLargest(int[] nums, int k) {
        // Min-heap stores k largest elements
        var heap = new PriorityQueue<int, int>();

        foreach (int num in nums) {
            heap.Enqueue(num, num);
            if (heap.Count > k) {
                heap.Dequeue();
            }
        }

        return heap.Peek();
    }
}

Last Stone Weight
Stones with weights. Each turn, take two heaviest, smash them. If weights differ, put remainder back. Return weight of last stone or 0.
EASY Google
2

Approach: Max-Heap Greedy

Use max-heap to always extract two largest stones. Continue until one or zero stones remain. Simple max-heap simulation with negate priority.

C# 
public class Solution {
    public int LastStoneWeight(int[] stones) {
        // Max-heap via negated priority
        var heap = new PriorityQueue<int, int>();

        foreach (int stone in stones) {
            heap.Enqueue(stone, -stone);
        }

        while (heap.Count > 1) {
            int first = heap.Dequeue();
            int second = heap.Dequeue();

            int diff = Math.Abs(first - second);
            if (diff > 0) {
                heap.Enqueue(diff, -diff);
            }
        }

        return heap.Count == 1 ? heap.Peek() : 0;
    }
}

O(n log n)
Space
O(n)

Max-heap greedy

Approach: Min-Heap of Distances

Compute squared distances (avoid sqrt overhead). Maintain min-heap of k closest points by distance. Time O(n log k).

C# 
public class Solution {
    public int[][] KClosest(int[][] points, int k) {
        var heap = new PriorityQueue<int[], int>();

        foreach (int[] point in points) {
            int distSq = point[0] * point[0] + point[1] * point[1];
            heap.Enqueue(point, distSq);

            if (heap.Count > k) {
                heap.Dequeue();
            }
        }

        int[][] result = new int[k][];
        int idx = 0;
        while (heap.Count > 0) {
            result[idx++] = heap.Dequeue();
        }

        return result;
    }
}

O(k)

Squared distance

Approach: Two-Heap Pattern

Max-heap for lower half, min-heap for upper half. Balance so sizes differ by at most 1. Median is max of lower half (odd n) or average of both roots (even n).

C# 
public class MedianFinder {
    private PriorityQueue<int, int> lower;  // max-heap
    private PriorityQueue<int, int> upper;  // min-heap

    public MedianFinder() {
        lower = new PriorityQueue<int, int>();
        upper = new PriorityQueue<int, int>();
    }

    public void AddNum(int num) {
        lower.Enqueue(num, -num);  // max-heap

        // Ensure lower max <= upper min
        if (lower.Count > 0 && upper.Count > 0
            && lower.Peek() > upper.Peek()) {
            int val = lower.Dequeue();
            upper.Enqueue(val, val);
        }

        // Balance: lower.Count == upper.Count or lower.Count + 1
        if (lower.Count > upper.Count + 1) {
            upper.Enqueue(lower.Dequeue(), lower.Peek());
        } else if (upper.Count > lower.Count) {
            lower.Enqueue(upper.Dequeue(), -upper.Peek());
        }
    }

    public double FindMedian() {
        if (lower.Count > upper.Count) {
            return lower.Peek();
        }
        return (lower.Peek() + upper.Peek()) / 2.0;
    }
}

Approach: Min-Heap of List Nodes

Maintain min-heap of at most k nodes (heads of each list). Extract minimum node, advance that list's pointer, re-insert new head. Build result iteratively.

C# 
public class Solution {
    public ListNode MergeKLists(ListNode[] lists) {
        // Min-heap by node value
        var heap = new PriorityQueue<(ListNode, int), int>();

        foreach (ListNode list in lists) {
            if (list != null) {
                heap.Enqueue((list, 0), list.val);
            }
        }

        ListNode dummy = new ListNode(0);
        ListNode current = dummy;

        while (heap.Count > 0) {
            (ListNode node, _) = heap.Dequeue();
            current.next = node;
            current = current.next;

            if (node.next != null) {
                heap.Enqueue((node.next, 0), node.next.val);
            }
        }

        return dummy.next;
    }
}
Time
O(N log k)
Space
O(k)
N = total nodes
k = num lists
💡 FAANG Tip: Contrast with naive approach (sort all + rebuild). Heap keeps only k pointers in memory, processes sequentially.

Approach: Min-Heap by Frequency

Count frequencies with HashMap. Maintain min-heap of size k with elements by frequency. Time O(n log k), space O(n).

C# 
public class Solution {
    public int[] TopKFrequent(int[] nums, int k) {
        // Count frequencies
        var freq = new Dictionary<int, int>();
        foreach (int num in nums) {
            freq[num] = freq.GetValueOrDefault(num, 0) + 1;
        }

        // Min-heap of (element, frequency) by frequency
        var heap = new PriorityQueue<int, int>();
        foreach (var (element, count) in freq) {
            heap.Enqueue(element, count);
            if (heap.Count > k) {
                heap.Dequeue();
            }
        }

        int[] result = new int[k];
        int idx = 0;
        while (heap.Count > 0) {
            result[idx++] = heap.Dequeue();
        }

        return result;
    }
}

Approach: Max-Heap + Greedy

Count frequencies. Use max-heap (by count). Always pick most frequent available char (not same as last char). Return result if valid.

C# 
public class Solution {
    public string ReorganizeString(string s) {
        // Count char frequencies
        var freq = new Dictionary<char, int>();
        foreach (char c in s) {
            freq[c] = freq.GetValueOrDefault(c, 0) + 1;
        }

        // Max-heap by frequency
        var heap = new PriorityQueue<(char, int), int>();
        foreach (var (c, count) in freq) {
            heap.Enqueue((c, count), -count);
        }

        if (heap.Peek().Item2 > (s.Length + 1) / 2) {
            return "";
        }

        StringBuilder result = new StringBuilder();
        (char prev, int prevCount) = (' ', 0);

        while (heap.Count > 0) {
            var (c, count) = heap.Dequeue();

            if (c == prev && prevCount > 0) {
                if (heap.Count == 0) return "";
                var (c2, count2) = heap.Dequeue();
                result.Append(c2);
                heap.Enqueue((c, count), -count);
                heap.Enqueue((c2, count2 - 1), -(count2 - 1));
                (prev, prevCount) = (c2, 1);
            } else {
                result.Append(c);
                (prev, prevCount) = (c, 1);
                if (count > 1) {
                    heap.Enqueue((c, count - 1), -(count - 1));
                }
            }
        }

        return result.ToString();
    }
}

Space
O(A)
A = 26 chars
Alphabet size
⚠️ Edge Case: If most frequent char appears > (n+1)/2 times, impossible to arrange. Validate early.

Approach: Min-Heap of End Times

Sort by start time. For each meeting, check if earliest-ending room is free. If so, reuse; else allocate new. Heap tracks room end times.

C# 
public class Solution {
    public int MinMeetingRooms(int[][] intervals) {
        // Sort by start time
        Array.Sort(intervals, (a, b) => a[0].CompareTo(b[0]));

        // Min-heap of end times
        var heap = new PriorityQueue<int, int>();

        foreach (int[] interval in intervals) {
            int start = interval[0], end = interval[1];

            // Earliest room's end time
            if (heap.Count > 0 && heap.Peek() <= start) {
                heap.Dequeue();  // Reuse room
            }

            heap.Enqueue(end, end);
        }

        return heap.Count;
    }
}
Time
O(n log n)
Space
O(n)
Key Insight
Min-heap end times
💡 FAANG Tip: This is a classic heap problem. The greedy choice (reuse earliest-ending room) is optimal. Prove it prevents unnecessary allocations.
Kth Smallest Element in Sorted Matrix
Given m×n matrix where rows and columns are sorted, find kth smallest element.
MEDIUM Amazon Microsoft
9

Approach: Min-Heap with Visited Set

Insert top-left (0,0) into min-heap. Repeatedly extract min, add its unvisited neighbors (right, down). Track visited to avoid duplicates.

C# 
public class Solution {
    public int KthSmallest(int[][] matrix, int k) {
        int m = matrix.Length, n = matrix[0].Length;

        // Min-heap of (value, row, col)
        var heap = new PriorityQueue<(int, int, int), int>();
        var visited = new HashSet<(int, int)>();

        heap.Enqueue((matrix[0][0], 0, 0), matrix[0][0]);
        visited.Add((0, 0));

        int count = 0;
        while (heap.Count > 0) {
            (int val, int r, int c) = heap.Dequeue();
            count++;

            if (count == k) return val;

            // Add right neighbor
            if (c + 1 < n && !visited.Contains((r, c + 1))) {
                heap.Enqueue((matrix[r][c + 1], r, c + 1), matrix[r][c + 1]);
                visited.Add((r, c + 1));
            }

            // Add down neighbor
            if (r + 1 < m && !visited.Contains((r + 1, c))) {
                heap.Enqueue((matrix[r + 1][c], r + 1, c), matrix[r + 1][c]);
                visited.Add((r + 1, c));
            }
        }

        return -1;
    }
}
Time
O(k log k)
Space
O(k)
Best for
Small k
💡 FAANG Tip: Visited set prevents re-adding same (r,c). Only explore right and down because rows/cols are sorted globally.

Approach: Max-Heap + Queue Cooldown

Count frequencies. Simulate: each cycle, extract most frequent available task (not in cooldown). Decrement, re-queue with cooldown delay. Track time.

C# 
public class Solution {
    public int LeastInterval(char[] tasks, int n) {
        // Count frequencies
        var freq = new Dictionary<char, int>();
        foreach (char task in tasks) {
            freq[task] = freq.GetValueOrDefault(task, 0) + 1;
        }

        // Max-heap by frequency
        var maxHeap = new PriorityQueue<(char, int), int>();
        foreach (var (task, count) in freq) {
            maxHeap.Enqueue((task, count), -count);
        }

        var cooldown = new Queue<(char, int)>();  // (task, available_time)
        int time = 0;

        while (maxHeap.Count > 0 || cooldown.Count > 0) {
            if (cooldown.Count > 0 && cooldown.Peek().Item2 == time) {
                var (task, _) = cooldown.Dequeue();
                maxHeap.Enqueue((task, freq[task]), -freq[task]);
            }

            if (maxHeap.Count > 0) {
                var (task, count) = maxHeap.Dequeue();
                count--;
                if (count > 0) {
                    cooldown.Enqueue((task, time + n + 1));
                }
                freq[task] = count;
            }

            time++;
        }

        return time;
    }
}
Time
O(t log A)
Space
O(A)
t = total tasks
A = 26 chars
⚠️ Edge Case: When n is large, many idle slots. Result can be far ≥ task count. Simulate carefully to avoid off-by-one.
Ugly Number II
Write program to find nth ugly number. Ugly = factors only 2, 3, 5. (1 is first ugly).
MEDIUM Google Amazon

Approach: Min-Heap Iteration

Start with 1. For each ugly number, generate next three (multiply by 2, 3, 5). Use min-heap and set to track seen numbers. Extract n times.

C# 
public class Solution {
    public int NthUglyNumber(int n) {
        // Min-heap of ugly candidates
        var heap = new PriorityQueue<long, long>();
        var seen = new HashSet<long>();

        heap.Enqueue(1, 1);
        seen.Add(1);

        long ugly = 1;
        int[] factors = { 2, 3, 5 };

        for (int i = 0; i < n; i++) {
            ugly = heap.Dequeue();

            foreach (int factor in factors) {
                long next = ugly * factor;
                if (!seen.Contains(next)) {
                    seen.Add(next);
                    heap.Enqueue(next, next);
                }
            }
        }

        return (int)ugly;
    }
}

O(n log n)
Space
O(n)

IPO

HARD Google Amazon
12

Approach: Two Heaps Greedy

Sort projects by capital requirement. Maintain max-heap of available profits (projects with requirement ≤ current capital). Greedily pick k most profitable. Update capital after each.

C# 
public class Solution {
    public int FindMaximizedCapital(int k, int w,
                                      int[] profits, int[] capital) {
        // Pair projects (capital, profit) and sort by capital
        var projects = new List<(int, int)>();
        for (int i = 0; i < profits.Length; i++) {
            projects.Add((capital[i], profits[i]));
        }
        projects.Sort();

        // Max-heap of profits
        var maxHeap = new PriorityQueue<int, int>();

        int idx = 0;
        for (int i = 0; i < k; i++) {
            // Add all available projects
            while (idx < projects.Count && projects[idx].Item1 <= w) {
                maxHeap.Enqueue(projects[idx].Item2, -projects[idx].Item2);
                idx++;
            }

            // No more projects available
            if (maxHeap.Count == 0) break;

            // Take most profitable
            w += maxHeap.Dequeue();
        }

        return w;
    }
}
Time
O(n log n + k log n)
Space
O(n)
Greedy
Max profit always
💡 FAANG Tip: This is the signature "two heaps + greedy" problem. Sort by one dimension, use heap for another. Greedy choice (max profit among available) is optimal.
Summary

Key Takeaways

When to Use Heap
— Dynamic "find extreme" (min/max) — Top-K or K-th smallest/largest — Merging sorted sequences — Scheduling with constraints — Always-available greedy choice
Min vs Max Heap
— Min-heap: Extract smallest repeatedly — Max-heap: Negate priority in C# PriorityQueue — Two-heap: Balance halves (median, balance tree) — Top-K: Use opposite-type heap of size K
Common Patterns
— Top-K: min-heap of size K — Merge K: min-heap of heads — Median: max-heap + min-heap balanced — Greedy: extract max, apply operation
Interview Tips
— Complexity: O(n log k) for top-K, O(n log n) for merge — Edge case: k=1, k=n, empty input — Validate greedy choice: prove it's always optimal — Compare alternatives: quickselect, sorting, DP

Final Checklist

1
Identify the operation: Are you finding min/max? K elements? Merging? Scheduling?
2
Choose heap type: Min for smallest, max for largest. Use negation in C# for max-heap.
3
Size constraint: Top-K or unlimited? Maintain size K for bounded memory.
4
Validate greedy: If extracting max repeatedly, why is this always optimal?
5
Code & test: Handle empty heaps, single element, ties, large values. Trace a small example.
