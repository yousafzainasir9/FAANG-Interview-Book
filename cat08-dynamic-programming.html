Dynamic Programming Framework

Dynamic Programming is a technique for solving problems with overlapping subproblems and optimal substructure. Success depends on correctly identifying state, recurrence, and base cases.

The Five-Step DP Framework

1
Define State: What exactly does dp[i] represent? Be precise—state ambiguity is the root of DP mistakes.
2
Write Recurrence: Express dp[i] in terms of previously computed states. This is the core logic.
3
Identify Base Cases: What are the simplest subproblems? (Usually dp[0] or small values.)
4
Choose Evaluation Order: Determine if you'll compute bottom-up (iterative) or top-down (memoization).
5
Optimize Space: Can you reduce space by storing only the last k values instead of the entire array?

Top-Down vs. Bottom-Up

Top-Down (Recursion + Memoization)
Start from the end goal and recursively break down into subproblems. Use a memo dictionary to cache results.
Pros: Intuitive, only computes needed states.
Cons: Recursion overhead, stack space.
Bottom-Up (Iteration)
Build solutions from base cases upward. Fill a table row by row or fill by fill.
Pros: No recursion, better cache locality, space control.
Cons: Compute all states, requires careful ordering.

Common DP Patterns

Pattern A
1D Array DP
State depends on one index. Examples: Climbing Stairs, House Robber, Coin Change (unbounded), LIS.
Pattern B
2D Array DP
State depends on two indices (typically two sequences). Examples: LCS, Edit Distance, Unique Paths.
Pattern C
Knapsack Variants
0/1 knapsack, unbounded, bounded. Use a dimension for item count or weight.

Additional patterns include interval DP (e.g., Burst Balloons) where state is a range, and bitmask DP for small subsets.

Climbing Stairs
You are climbing a staircase with n steps. Each time you can climb 1 or 2 steps. How many distinct ways can you climb to the top?
Easy Meta Google Amazon
1
Time
O(n)
Space
O(1) optimized
Type
1D DP

Problem Statement

Given a staircase with n steps, you can climb 1 or 2 steps at a time. Calculate the number of distinct ways to reach the top.

Approach Comparison

Brute Force (Recursion)
Explore all paths recursively
Time: O(2^n) exponential
Space: O(n) stack depth
Hits same subproblems repeatedly
Dynamic Programming
Compute each state once
Time: O(n) linear
Space: O(1) constant
Overlapping subproblems solved once

DP State & Recurrence

State: dp[i] = number of ways to reach step i.

Recurrence: dp[i] = dp[i-1] + dp[i-2] (you can reach step i from step i-1 or i-2).

Base Cases: dp[0] = 1, dp[1] = 1.

Solution

C# 
public class Solution {
    public int ClimbStairs(int n) {
        if (n <= 1) return 1;

        int prev2 = 1, prev1 = 1;
        for (int i = 2; i <= n; i++) {
            int curr = prev1 + prev2;
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }
}

House Robber
Rob houses in a line, but you cannot rob two adjacent houses. Maximize the total amount of money stolen.
Medium Meta Apple Amazon
2
Time
O(n)
Space
O(1)
Type
1D DP (Max)

Problem Statement

You are a professional robber planning to rob houses. You cannot rob two adjacent houses. Given an array nums where nums[i] is the amount of money in house i, return the maximum amount you can rob.

Approach Comparison

Brute Force
Try all subsets of non-adjacent houses
Time: O(2^n)
Space: O(n) for recursion
1D DP
dp[i] = max(rob i, skip i)
Time: O(n)
Space: O(1) with optimization

DP State & Recurrence

State: dp[i] = maximum money robbed considering houses up to i.

Recurrence: dp[i] = max(dp[i-1], nums[i] + dp[i-2]) (rob or skip house i).

Base Cases: dp[0] = nums[0], dp[1] = max(nums[0], nums[1]).

Solution

C# 
public class Solution {
    public int Rob(int[] nums) {
        if (nums.Length == 1) return nums[0];

        int prev2 = nums[0];
        int prev1 = Math.Max(nums[0], nums[1]);

        for (int i = 2; i < nums.Length; i++) {
            int curr = Math.Max(prev1, nums[i] + prev2);
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }
}

House Robber II
Houses are arranged in a circle; you cannot rob the first and last house together. Maximize total money.
Medium Apple
3
Time
O(n)
Space
O(1)
Type
Circular DP

Problem Statement

Same as House Robber, but now houses are arranged in a circle. You cannot rob both the first and last house.

Approach & Solution

The circular constraint means house n-1 and house 0 are adjacent. Solution: compute max for two cases—(1) rob [0...n-2], (2) rob [1...n-1]—and return the maximum.

C# 
public class Solution {
    private int RobLinear(int start, int end, int[] nums) {
        int prev2 = 0, prev1 = 0;
        for (int i = start; i <= end; i++) {
            int curr = Math.Max(prev1, nums[i] + prev2);
            prev2 = prev1;
            prev1 = curr;
        }
        return prev1;
    }

    public int Rob(int[] nums) {
        if (nums.Length == 1) return nums[0];
        if (nums.Length == 2) return Math.Max(nums[0], nums[1]);

        // Case 1: Rob [0...n-2] (exclude last)
        int case1 = RobLinear(0, nums.Length - 2, nums);

        // Case 2: Rob [1...n-1] (exclude first)
        int case2 = RobLinear(1, nums.Length - 1, nums);

        return Math.Max(case1, case2);
    }
}

Edge Cases & FAANG Tips

ℹ️
Key Insight: By considering both cases (exclude first, exclude last), we ensure we never rob both endpoints. This is a common technique for circular DP problems.
Coin Change
Given coins of different denominations, find the fewest coins needed to make a given amount.
Medium Google Meta
4
Time
O(n × m)
Space
O(n)
Type
Unbounded Knapsack

Problem Statement

Given an integer array coins and an integer amount, return the fewest number of coins needed to make that amount. Return -1 if impossible.

DP State & Recurrence

State: dp[i] = minimum number of coins to make amount i.

Recurrence: dp[i] = 1 + min(dp[i - coin]) for each coin ≤ i.

Base Case: dp[0] = 0 (zero coins for amount 0).

Solution

C# 
public class Solution {
    public int CoinChange(int[] coins, int amount) {
        int[] dp = new int[amount + 1];
        for (int i = 1; i <= amount; i++) {
            dp[i] = amount + 1; // Initialize to impossible value
            foreach (int coin in coins) {
                if (coin <= i) {
                    dp[i] = Math.Min(dp[i], dp[i - coin] + 1);
                }
            }
        }
        return dp[amount] > amount ? -1 : dp[amount];
    }
}

Longest Increasing Subsequence
Find the length of the longest strictly increasing subsequence in an array.
Medium Amazon Google
5
Time (DP)
O(n²)
Time (Binary Search)
O(n log n)
Space
O(n)

Problem Statement

Given an integer array nums, return the length of the longest strictly increasing subsequence.

DP State & Recurrence (O(n²) solution)

State: dp[i] = length of LIS ending at index i.

Recurrence: dp[i] = 1 + max(dp[j]) for all j < i where nums[j] < nums[i].

Solution (DP)

C# 
public class Solution {
    public int LengthOfLIS(int[] nums) {
        int[] dp = new int[nums.Length];
        int maxLen = 0;

        for (int i = 0; i < nums.Length; i++) {
            dp[i] = 1;
            for (int j = 0; j < i; j++) {
                if (nums[j] < nums[i]) {
                    dp[i] = Math.Max(dp[i], dp[j] + 1);
                }
            }
            maxLen = Math.Max(maxLen, dp[i]);
        }
        return maxLen;
    }
}

Optimized Solution (Binary Search, O(n log n))

Maintain an array where tails[i] is the smallest tail of all increasing subsequences of length i+1.

C# 
public class Solution {
    public int LengthOfLIS(int[] nums) {
        var tails = new List<int>();

        foreach (int num in nums) {
            int pos = tails.BinarySearch(num);
            if (pos < 0) pos = ~pos; // Convert to insertion point

            if (pos == tails.Count) {
                tails.Add(num);
            } else {
                tails[pos] = num;
            }
        }
        return tails.Count;
    }
}

Edge Cases & FAANG Tips

💡
Interview Tip: Start with O(n²) DP, then optimize with binary search if asked. Explain why the greedy strategy (keeping smallest tails) is valid.
Word Break
Determine if a string can be segmented into words from a given dictionary.
Medium Meta Google
6
Time
O(n² × m)
Space
O(n)
Type
1D DP + Set

Problem Statement

Given a string s and a list of words wordDict, return true if s can be segmented into space-separated words from the dictionary.

DP State & Recurrence

State: dp[i] = true if substring s[0...i-1] can be segmented.

Recurrence: dp[i] = true if dp[j] == true and s[j...i-1] is in the word dictionary, for some j < i.

Solution

C# 
public class Solution {
    public bool WordBreak(string s, IList<string> wordDict) {
        var wordSet = new HashSet<string>(wordDict);
        bool[] dp = new bool[s.Length + 1];
        dp[0] = true;

        for (int i = 1; i <= s.Length; i++) {
            for (int j = 0; j < i; j++) {
                string word = s.Substring(j, i - j);
                if (dp[j] && wordSet.Contains(word)) {
                    dp[i] = true;
                    break;
                }
            }
        }
        return dp[s.Length];
    }
}

Longest Common Subsequence
Find the length of the longest subsequence common to two strings.
Medium Google
7
Time
O(m × n)
Space
O(m × n)
Type
2D DP

Problem Statement

Given two strings text1 and text2, return the length of their longest common subsequence. A subsequence is a sequence that appears in the same order but not necessarily contiguous.

DP State & Recurrence

State: dp[i][j] = LCS length of text1[0...i-1] and text2[0...j-1].

Recurrence:
If text1[i-1] == text2[j-1]: dp[i][j] = dp[i-1][j-1] + 1
Else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])

Solution

C# 
public class Solution {
    public int LongestCommonSubsequence(string text1, string text2) {
        int[][] dp = new int[text1.Length + 1][];
        for (int i = 0; i <= text1.Length; i++) {
            dp[i] = new int[text2.Length + 1];
        }

        for (int i = 1; i <= text1.Length; i++) {
            for (int j = 1; j <= text2.Length; j++) {
                if (text1[i - 1] == text2[j - 1]) {
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    dp[i][j] = Math.Max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }
        return dp[text1.Length][text2.Length];
    }
}

Edge Cases & FAANG Tips

💡
Follow-up: Reconstruct the actual LCS by backtracking from dp[m][n]. Keep a separate array to store parent pointers or directions.
Edit Distance (Levenshtein Distance)
Find the minimum number of operations (insert, delete, replace) to transform one string into another.
Medium Meta Apple
8
Time
O(m × n)
Space
O(m × n) or O(n)
Type
2D DP

Problem Statement

Given two strings word1 and word2, return the minimum number of single-character edits (insert, delete, replace) needed to change word1 into word2.

DP State & Recurrence

State: dp[i][j] = min edits to transform word1[0...i-1] to word2[0...j-1].

Recurrence:
If word1[i-1] == word2[j-1]: dp[i][j] = dp[i-1][j-1] (no edit needed)
Else: dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1]) (delete, insert, replace)

Solution

C# 
public class Solution {
    public int MinDistance(string word1, string word2) {
        int[][] dp = new int[word1.Length + 1][];
        for (int i = 0; i <= word1.Length; i++) {
            dp[i] = new int[word2.Length + 1];
        }

        // Base cases
        for (int i = 0; i <= word1.Length; i++) dp[i][0] = i;
        for (int j = 0; j <= word2.Length; j++) dp[0][j] = j;

        for (int i = 1; i <= word1.Length; i++) {
            for (int j = 1; j <= word2.Length; j++) {
                if (word1[i - 1] == word2[j - 1]) {
                    dp[i][j] = dp[i - 1][j - 1];
                } else {
                    dp[i][j] = 1 + Math.Min(Math.Min(
                        dp[i - 1][j], // delete
                        dp[i][j - 1]  // insert
                    ), dp[i - 1][j - 1]); // replace
                }
            }
        }
        return dp[word1.Length][word2.Length];
    }
}

Edge Cases & FAANG Tips

💡
Space Optimization: Use a rolling array (only keep current and previous rows) to reduce space from O(m×n) to O(n).
Unique Paths
Count paths from top-left to bottom-right in a grid, moving only right or down.
Medium Meta Google
9
Time
O(m × n)
Space
O(m × n) or O(n)
Type
2D Grid DP

Problem Statement

Given an m × n grid, return the number of unique paths from top-left [0, 0] to bottom-right [m-1, n-1]. You can only move right or down.

DP State & Recurrence

State: dp[i][j] = number of unique paths to cell (i, j).

Recurrence: dp[i][j] = dp[i-1][j] + dp[i][j-1] (come from top or left).

Solution

C# 
public class Solution {
    public int UniquePaths(int m, int n) {
        int[][] dp = new int[m][];
        for (int i = 0; i < m; i++) {
            dp[i] = new int[n];
            for (int j = 0; j < n; j++) {
                if (i == 0 || j == 0) {
                    dp[i][j] = 1;
                } else {
                    dp[i][j] = dp[i - 1][j] + dp[i][j - 1];
                }
            }
        }
        return dp[m - 1][n - 1];
    }
}

Edge Cases & FAANG Tips

ℹ️
Mathematical Solution: This is actually a combinatorics problem: C(m+n-2, m-1). For large grids, use BigInteger to avoid overflow.
Decode Ways
Count the number of ways to decode an encoded string (1='A', 2='B', ..., 26='Z').
Medium Meta
10
Time
O(n)
Space
O(1)
Type
1D DP

Problem Statement

A message containing letters from A-Z is encoded as numbers 1-26. Given a non-empty string containing only digits, return the number of ways to decode it.

DP State & Recurrence

State: dp[i] = number of ways to decode s[0...i-1].

Recurrence:
- If s[i-1] != '0': dp[i] += dp[i-1] (decode as single digit)
- If s[i-2:i] is valid (10-26): dp[i] += dp[i-2] (decode as two digits)

Solution

C# 
public class Solution {
    public int NumDecodings(string s) {
        if (s[0] == '0') return 0;

        long prev2 = 1, prev1 = 1;

        for (int i = 1; i < s.Length; i++) {
            long curr = 0;
            int oneDigit = s[i] - '0';
            int twoDigit = (int.Parse(s.Substring(i - 1, 2));

            if (oneDigit >= 1 && oneDigit <= 9) {
                curr += prev1;
            }
            if (twoDigit >= 10 && twoDigit <= 26) {
                curr += prev2;
            }

            prev2 = prev1;
            prev1 = curr;
        }
        return (int)prev1;
    }
}

Edge Cases & FAANG Tips

⚠️
Common Pitfall: Handle strings with leading zeros. "0" cannot be decoded. "10" and "20" are valid, but "00" is not.
Maximum Product Subarray
Find the contiguous subarray with the largest product.
Medium Amazon Google
11
Time
O(n)
Space
O(1)
Type
1D DP (Min/Max Track)

Problem Statement

Given an integer array, find the contiguous subarray with the largest product and return that product.

Key Insight

Unlike maximum subarray sum, we must track both max and min products, because a negative number can flip a negative minimum to a positive maximum.

DP State & Recurrence

State: maxProd[i] = max product ending at index i; minProd[i] = min product ending at index i.

Recurrence:
maxProd[i] = max(nums[i], nums[i] * maxProd[i-1], nums[i] * minProd[i-1])
minProd[i] = min(nums[i], nums[i] * maxProd[i-1], nums[i] * minProd[i-1])

Solution

C# 
public class Solution {
    public int MaxProduct(int[] nums) {
        int maxProd = nums[0];
        int minProd = nums[0];
        int result = nums[0];

        for (int i = 1; i < nums.Length; i++) {
            int newMax = Math.Max(Math.Max(
                nums[i],
                nums[i] * maxProd
            ), nums[i] * minProd);

            int newMin = Math.Min(Math.Min(
                nums[i],
                nums[i] * maxProd
            ), nums[i] * minProd);

            maxProd = newMax;
            minProd = newMin;
            result = Math.Max(result, maxProd);
        }
        return result;
    }
}

Edge Cases & FAANG Tips

💡
Interview Tip: Explain why we track both max and min. A negative number can flip the min to become the new max on the next iteration.
Partition Equal Subset Sum
Determine if an array can be partitioned into two subsets with equal sum (0/1 Knapsack variant).
Medium Google
12
Time
O(n × sum/2)
Space
O(sum/2)
Type
0/1 Knapsack

Problem Statement

Given an integer array, determine if it can be partitioned into two subsets such that the sum of elements in both subsets is equal.

Approach

If the total sum is odd, partitioning is impossible. Otherwise, check if we can achieve a subset with sum = total / 2 using 0/1 knapsack DP.

DP State & Recurrence

State: dp[i] = true if sum i is achievable.

Recurrence: dp[i] = dp[i] || dp[i - num] for each number in the array.

Solution

C# 
public class Solution {
    public bool CanPartition(int[] nums) {
        int total = 0;
        foreach (int num in nums) total += num;

        if (total % 2 != 0) return false;

        int target = total / 2;
        bool[] dp = new bool[target + 1];
        dp[0] = true;

        foreach (int num in nums) {
            // Iterate backwards to avoid using same element twice
            for (int i = target; i >= num; i--) {
                dp[i] = dp[i] || dp[i - num];
            }
        }
        return dp[target];
    }
}

Edge Cases & FAANG Tips

⚠️
Crucial: Iterate the dp array backwards when updating. This ensures each number is used at most once (0/1 knapsack). Forward iteration would allow reuse (unbounded knapsack).
Target Sum
Count ways to assign +/- signs to array elements to achieve a target sum.
Medium Meta
13
Time
O(n × (sum + target))
Space
O(sum + target)
Type
DP + Offset

Problem Statement

Given an integer array and a target, count the number of ways to assign +/- signs to each element such that the sum equals the target.

Approach

Convert to subset sum: if P is the sum of positive subset and N is the sum of negative subset, then P - N = target and P + N = sum. Solving: P = (target + sum) / 2. Count ways to achieve P using DP.

Solution

C# 
public class Solution {
    public int FindTargetSumWays(int[] nums, int target) {
        int sum = 0;
        foreach (int num in nums) sum += num;

        if (sum < Math.Abs(target) || (sum + target) % 2 != 0) return 0;

        int subsetSum = (sum + target) / 2;
        int[] dp = new int[subsetSum + 1];
        dp[0] = 1;

        foreach (int num in nums) {
            for (int i = subsetSum; i >= num; i--) {
                dp[i] += dp[i - num];
            }
        }
        return dp[subsetSum];
    }
}

Edge Cases & FAANG Tips

💡
Mathematical Reduction: This problem reduces to "count subsets with sum = (target + total) / 2". This elegant transformation converts a sign-assignment problem into a standard subset sum counting problem.
Burst Balloons
Maximize coins collected by bursting balloons (Interval DP).
Hard Meta
14
Time
O(n³)
Space
O(n²)
Type
Interval DP

Problem Statement

Given an array of balloons with values, burst them to maximize coins. When you burst balloon k, you get balloons[left] × balloons[k] × balloons[right] coins, where left and right are adjacent intact balloons.

Key Insight

Think backwards: consider which balloon to burst last in a range, not first. This avoids recalculating damaged states.

DP State & Recurrence

State: dp[i][j] = max coins from bursting balloons between i and j (exclusive).

Recurrence: For each balloon k in range, assume it's the last to burst: dp[i][j] = max(dp[i][k] + nums[i]*nums[k]*nums[j] + dp[k][j])

Solution

C# 
public class Solution {
    public int MaxCoins(int[] nums) {
        // Pad with 1s: simplifies boundary calculations
        int[] balloons = new int[nums.Length + 2];
        balloons[0] = 1;
        balloons[balloons.Length - 1] = 1;
        for (int i = 0; i < nums.Length; i++) {
            balloons[i + 1] = nums[i];
        }

        int[][] dp = new int[balloons.Length][];
        for (int i = 0; i < balloons.Length; i++) {
            dp[i] = new int[balloons.Length];
        }

        // len = distance between balloons (exclusive)
        for (int len = 2; len < balloons.Length; len++) {
            for (int i = 0; i < balloons.Length - len; i++) {
                int j = i + len;
                for (int k = i + 1; k < j; k++) {
                    dp[i][j] = Math.Max(
                        dp[i][j],
                        dp[i][k] + balloons[i] * balloons[k] * balloons[j] + dp[k][j]
                    );
                }
            }
        }
        return dp[0][balloons.Length - 1];
    }
}

Edge Cases & FAANG Tips

💡
Advanced Insight: Padding the array with 1s is a clever trick. It eliminates special boundary cases since balloons[0] and balloons[n+1] = 1 act as neutral multipliers for all bursts.
Regular Expression Matching
Implement regex matching with '.' (any char) and '*' (0+ of previous char).
Hard Meta Google
15
Time
O(m × n)
Space
O(m × n)
Type
2D DP (Complex)

Problem Statement

Given a string s and a pattern p, implement regex matching supporting '.' (matches any single char) and '*' (matches zero or more of the preceding element).

DP State & Recurrence

State: dp[i][j] = true if s[0...i-1] matches p[0...j-1].

Recurrence (complex): Consider p[j-1]:
— If it's '*': skip pair or match preceding;
— If it's '.' or matches s[i-1]: dp[i][j] = dp[i-1][j-1]

Solution

C# 
public class Solution {
    public bool IsMatch(string s, string p) {
        bool[][] dp = new bool[s.Length + 1][];
        for (int i = 0; i <= s.Length; i++) {
            dp[i] = new bool[p.Length + 1];
        }

        dp[0][0] = true;

        // Handle patterns like a*, a*b*, etc. at start
        for (int j = 2; j <= p.Length; j++) {
            if (p[j - 1] == '*') {
                dp[0][j] = dp[0][j - 2];
            }
        }

        for (int i = 1; i <= s.Length; i++) {
            for (int j = 1; j <= p.Length; j++) {
                if (p[j - 1] == '*') {
                    // Two cases: skip pair or match
                    dp[i][j] = dp[i][j - 2]; // Skip a*
                    if (p[j - 2] == '.' || p[j - 2] == s[i - 1]) {
                        dp[i][j] = dp[i][j] || dp[i - 1][j]; // Match 1+ chars
                    }
                } else {
                    // Char or dot
                    if (p[j - 1] == '.' || p[j - 1] == s[i - 1]) {
                        dp[i][j] = dp[i - 1][j - 1];
                    }
                }
            }
        }
        return dp[s.Length][p.Length];
    }
}

Edge Cases & FAANG Tips

⚠️
Tricky Cases: Empty pattern matches empty string. "a*" matches empty string. "*" never matches without a preceding char. Test: ("aa", "a"), ("aa", "a*"), ("ab", ".*").
SUMMARY

Key Takeaways

💡
Always start with state definition. Ambiguous state = guaranteed error. Write it clearly and test with an example before coding.
💡
Optimize space when possible. If DP only depends on previous rows/values, use a rolling array or two variables to reduce space.
💡
Master knapsack variants. 0/1, unbounded, and bounded knapsack appear frequently. Know when to iterate forward vs. backward.
ℹ️
Practice bottom-up first. Iterative DP is clearer for interviews and avoids stack overflow. Top-down memoization is more intuitive but riskier under time pressure.
