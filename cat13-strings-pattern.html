Core Concepts & Patterns

String problems sit at the intersection of algorithms, data structures, and careful implementation. C# provides powerful built-ins, but understanding the mechanics matters more.

C# String Fundamentals

Strings in C# are immutable. Every concatenation or modification creates a new string object. For intensive string building, use StringBuilder:

csharp 
// String concatenation (creates new object each time)
string result = "";
for (int i = 0; i < n; i++) {
    result += s[i];  // O(n²) time
}

// StringBuilder (efficient)
var sb = new StringBuilder();
for (int i = 0; i < n; i++) {
    sb.Append(s[i]);  // O(1) amortized
}
string result = sb.ToString();

Key Techniques

EXPAND AROUND CENTER
Palindrome Detection
For each possible center (char or pair), expand outward checking if characters match. O(n²) time, O(1) space.
SLIDING WINDOW
Substring Problems
Maintain a window with two pointers. Track character frequencies or constraints. Move right to expand, left to contract.
DYNAMIC PROGRAMMING
String Matching
Build tables for regex, wildcard, decode ways, word break. Each cell represents a state or subproblem solution.

Pattern Matching Algorithms

KMP (Knuth-Morris-Pratt): Linear-time string search using a failure function. Compute prefix table where pi[i] = longest proper prefix that is also suffix of s[0..i].

csharp 
private int[] BuildPrefixTable(string pattern) {
    int n = pattern.Length;
    int[] pi = new int[n];
    int j = 0;
    for (int i = 1; i < n; i++) {
        while (j > 0 && pattern[i] != pattern[j])
            j = pi[j - 1];
        if (pattern[i] == pattern[j])
            j++;
        pi[i] = j;
    }
    return pi;
}

Rabin-Karp Overview: Use rolling hash to find pattern efficiently. Compute hash of pattern and each substring of matching length. If hashes match, verify character-by-character.

💡
Interview Tip: For string problems, discuss space-time tradeoffs. Brute force is often acceptable for small inputs (n < 100). For larger inputs, optimize with sliding windows or DP. Always consider character set size—for small alphabets (26 letters), hash/frequency maps are powerful.
Longest Palindromic Substring

Find the longest contiguous substring that reads the same forwards and backwards.

Medium Amazon Google Microsoft
1

Problem

Given string s, return the longest palindromic substring. If multiple exist, return any one.

Example: Input: "babad" → Output: "bab" or "aba"

Approach: Expand Around Center

1
For each possible center, expand left and right while characters match.
2
Handle both odd-length (single char center) and even-length (two char center) palindromes.
3
Track maximum length palindrome found.

Solution

csharp 
public string LongestPalindrome(string s) {
    if (s == null || s.Length < 1) return "";

    int maxLen = 0, start = 0;

    for (int i = 0; i < s.Length; i++) {
        // Odd-length palindrome (single center)
        int len1 = ExpandAroundCenter(s, i, i);
        // Even-length palindrome (two-char center)
        int len2 = ExpandAroundCenter(s, i, i + 1);

        int len = Math.Max(len1, len2);
        if (len > maxLen) {
            maxLen = len;
            start = i - (len - 1) / 2;
        }
    }

    return s.Substring(start, maxLen);
}

private int ExpandAroundCenter(string s, int left, int right) {
    while (left >= 0 && right < s.Length && s[left] == s[right]) {
        left--;
        right++;
    }
    return right - left - 1;
}
Time
O(n²)
Space
O(1)

Edge Cases

  • Empty or single-character string → return as-is
  • Entire string is palindrome
  • No palindrome longer than 1 char
🔍
FAANG Tip: Mention Manacher's algorithm (O(n)) for follow-up, but expand-around-center is often sufficient and demonstrates clear thinking. At Google, interviewers appreciate seeing the tradeoff discussion.
Palindromic Substrings

Count or enumerate all palindromic substrings.

Medium Facebook Google
2

Problem

Given a string, return the count of palindromic substrings (including single characters).

Example: Input: "abc" → Output: 3 (a, b, c)

Example: Input: "aaa" → Output: 6 (a, a, a, aa, aa, aaa)

Approach: Expand Around Center

Similar to problem 1, but count all palindromes instead of finding the longest.

Solution

csharp 
public int CountSubstrings(string s) {
    int count = 0;

    for (int i = 0; i < s.Length; i++) {
        // Count odd-length palindromes
        count += CountPalindromes(s, i, i);
        // Count even-length palindromes
        count += CountPalindromes(s, i, i + 1);
    }

    return count;
}

private int CountPalindromes(string s, int left, int right) {
    int count = 0;
    while (left >= 0 && right < s.Length && s[left] == s[right]) {
        count++;
        left--;
        right++;
    }
    return count;
}
Time
O(n²)
Space
O(1)
⚠️
Interview Consideration: For very long strings, mention Manacher's algorithm achieves O(n). Expand-around-center is more intuitive and sufficient for most interviews.
Longest Common Prefix

Find the longest common prefix across all strings in an array.

Easy Google Amazon
3

Problem

Given array of strings, find the longest common prefix. If none exists, return empty string.

Example: Input: ["flower","flow","flight"] → Output: "fl"

Approach: Vertical Scan

Compare characters column-by-column (character at index i across all strings), stopping when a mismatch occurs.

Solution

csharp 
public string LongestCommonPrefix(string[] strs) {
    if (strs == null || strs.Length == 0) return "";

    for (int i = 0; i < strs[0].Length; i++) {
        char c = strs[0][i];

        for (int j = 1; j < strs.Length; j++) {
            // Check if string is too short or char doesn't match
            if (i >= strs[j].Length || strs[j][i] != c) {
                return strs[0].Substring(0, i);
            }
        }
    }

    return strs[0];
}
Time
O(S) *
Space
O(1)

* S = sum of all characters

Edge Cases

  • Empty array or array with empty string → ""
  • Single string → return it as-is
  • No common prefix → ""
Decode Ways

Count the number of ways to decode a string of digits to letters.

Medium Amazon Facebook
4

Problem

A string contains digits. '1' → 'A', '2' → 'B', ..., '26' → 'Z'. Count how many ways to decode the string.

Example: Input: "12" → Output: 2 ('1','2' or '12')

Example: Input: "226" → Output: 3 ('2','2','6' or '22','6' or '2','26')

Approach: Dynamic Programming

1
dp[i] = number of ways to decode s[0..i-1]
2
For each position, check if we can decode single digit or two-digit combination.
3
Transition: dp[i] += dp[i-1] if single digit valid, dp[i] += dp[i-2] if two digits valid.

Solution

csharp 
public int NumDecodings(string s) {
    if (s == null || s.Length == 0 || s[0] == '0') return 0;

    int n = s.Length;
    int[] dp = new int[n + 1];
    dp[0] = 1;
    dp[1] = 1;

    for (int i = 2; i <= n; i++) {
        int oneDigit = int.Parse(s.Substring(i - 1, 1));
        int twoDigits = int.Parse(s.Substring(i - 2, 2));

        if (oneDigit >= 1) dp[i] += dp[i - 1];
        if (twoDigits >= 10 && twoDigits <= 26) dp[i] += dp[i - 2];
    }

    return dp[n];
}
Time
O(n)
Space
O(n)
💡
Optimization: Space can be reduced to O(1) by tracking only previous two values instead of full DP array.
Word Break

Determine if a string can be segmented into dictionary words.

Medium Amazon Google Facebook
5

Problem

Given a string and list of dictionary words, determine if the string can be segmented into dictionary words.

Example: Input: s = "leetcode", wordDict = ["leet","code"] → Output: true

Approach: DP + HashSet

1
Store dictionary words in HashSet for O(1) lookup.
2
dp[i] = true if s[0..i-1] can be segmented.
3
For each position, check all previous positions where segmentation is valid.

Solution

csharp 
public bool WordBreak(string s, IList<string> wordDict) {
    var wordSet = new HashSet<string>(wordDict);
    bool[] dp = new bool[s.Length + 1];
    dp[0] = true;

    for (int i = 1; i <= s.Length; i++) {
        for (int j = 0; j < i; j++) {
            if (dp[j] && wordSet.Contains(s.Substring(j, i - j))) {
                dp[i] = true;
                break;
            }
        }
    }

    return dp[s.Length];
}
Time
O(n²)
Space
O(n + m)

m = dictionary size

🔍
Follow-up: If asked to return the actual segmentation, use backtracking or reconstruct from DP table.
Minimum Window Substring

Find the smallest substring containing all characters from a target string.

Hard Google Amazon Facebook
6

Problem

Given two strings s and t, find the minimum window substring of s that contains all characters in t.

Example: Input: s = "ADOBECODEBANC", t = "ABC" → Output: "BANC"

Approach: Sliding Window

1
Use two pointers and character frequency maps.
2
Expand right pointer until window contains all target characters.
3
Contract left pointer to find minimum valid window.

Solution

csharp 
public string MinWindow(string s, string t) {
    if (s.Length < t.Length) return "";

    var tCount = new Dictionary<char, int>();
    var windowCount = new Dictionary<char, int>();

    foreach (char c in t) {
        tCount[c] = (tCount.ContainsKey(c) ? tCount[c] : 0) + 1;
    }

    int formed = 0, required = tCount.Count;
    int left = 0, right = 0;
    int minLen = int.MaxValue, minLeft = 0;

    while (right < s.Length) {
        char c = s[right];
        windowCount[c] = (windowCount.ContainsKey(c) ? windowCount[c] : 0) + 1;

        if (tCount.ContainsKey(c) && windowCount[c] == tCount[c])
            formed++;

        while (formed == required && left <= right) {
            if (right - left + 1 < minLen) {
                minLen = right - left + 1;
                minLeft = left;
            }

            char lc = s[left];
            windowCount[lc]--;
            if (tCount.ContainsKey(lc) && windowCount[lc] < tCount[lc])
                formed--;
            left++;
        }

        right++;
    }

    return minLen == int.MaxValue ? "" : s.Substring(minLeft, minLen);
}
Time
O(|s| + |t|)
Space
O(1)*

* Fixed alphabet size (26 letters + digits)

⚠️
Tricky Aspect: The "formed" variable tracks unique characters with correct frequency. This is more efficient than counting total character occurrences.
Group Anagrams

Group strings that are anagrams of each other.

Medium Amazon Google Facebook
7

Problem

Given array of strings, group anagrams together.

Example: Input: ["eat","tea","tan","ate","nat","bat"] → Output: [["bat"],["nat","tan"],["ate","eat","tea"]]

Approach: Sort Key

Anagrams have the same characters when sorted. Use sorted string as the key in a dictionary.

Solution

csharp 
public IList<IList<string>> GroupAnagrams(string[] strs) {
    var map = new Dictionary<string, IList<string>>();

    foreach (string s in strs) {
        var sorted = new string(s.ToCharArray());
        var charArray = sorted.ToCharArray();
        System.Array.Sort(charArray);
        sorted = new string(charArray);

        if (!map.ContainsKey(sorted)) {
            map[sorted] = new List<string>();
        }
        map[sorted].Add(s);
    }

    return new List<IList<string>>(map.Values);
}
Time
O(n*k log k)
Space
O(n*k)

n = number of strings, k = average string length

💡
Alternative: Use character frequency map as key instead of sorting. Hash the frequencies for O(n*k) time (if hash computation is O(1)).
String to Integer (atoi)

Convert string to 32-bit signed integer with careful boundary handling.

Medium Google Amazon
8

Problem

Implement atoi: convert string to 32-bit integer. Handle leading whitespace, optional sign, non-digit chars, and overflow.

Rules: Strip leading whitespace, check for +/-, read digits, stop at non-digit, clamp to [-2³¹, 2³¹-1].

Example: Input: "42" → Output: 42

Example: Input: " -42" → Output: -42

Approach: State Machine

Track parsing state: whitespace, sign, digits. Stop when state changes invalidly or digits end.

Solution

csharp 
public int MyAtoi(string s) {
    int i = 0, sign = 1, result = 0;

    // Skip whitespace
    while (i < s.Length && s[i] == ' ')
        i++;

    // Check sign
    if (i < s.Length && (s[i] == '-' || s[i] == '+')) {
        sign = s[i] == '-' ? -1 : 1;
        i++;
    }

    // Read digits with overflow check
    while (i < s.Length && char.IsDigit(s[i])) {
        int digit = s[i] - '0';

        // Check overflow before multiplying
        if (result > int.MaxValue / 10 ||
            (result == int.MaxValue / 10 && digit > 7)) {
            return sign == 1 ? int.MaxValue : int.MinValue;
        }

        result = result * 10 + digit;
        i++;
    }

    return result * sign;
}
Time
O(n)
Space
O(1)

Edge Cases

  • Leading/trailing whitespace
  • Optional +/- sign
  • Non-digit characters → stop parsing
  • Integer overflow
  • Empty string or only whitespace → 0
🚨
Critical: Overflow check must happen before multiplying by 10. Use int.MaxValue / 10 to avoid overflow during the check itself.
Multiply Strings

Multiply two non-negative integers represented as strings.

Medium Google Amazon
9

Problem

Multiply two numbers as strings without converting to integers (handles numbers larger than 64-bit).

Example: Input: num1 = "123", num2 = "456" → Output: "56088"

Approach: Grade School Multiplication

Multiply each digit of num1 by each digit of num2, tracking position (place value). Sum results.

Solution

csharp 
public string Multiply(string num1, string num2) {
    if (num1 == "0" || num2 == "0") return "0";

    int m = num1.Length, n = num2.Length;
    int[] result = new int[m + n];

    int i1 = m - 1, i2 = n - 1;
    for (int i = m - 1; i >= 0; i--) {
        for (int j = n - 1; j >= 0; j--) {
            int mul = (num1[i] - '0') * (num2[j] - '0');
            int p1 = i + j, p2 = i + j + 1;
            int sum = mul + result[p2];

            result[p2] = sum % 10;
            result[p1] += sum / 10;
        }
    }

    var sb = new StringBuilder();
    foreach (int digit in result) {
        if (!(sb.Length == 0 && digit == 0)) {
            sb.Append(digit);
        }
    }

    return sb.Length == 0 ? "0" : sb.ToString();
}
Time
O(m*n)
Space
O(m+n)
🔍
Key Insight: Position i,j in digit multiplication maps to positions i+j (carry) and i+j+1 (product) in the result array.
Wildcard Matching

Pattern matching with '?' (single char) and '*' (zero or more chars).

Hard Google Amazon Facebook
10

Problem

Given string s and pattern p, implement wildcard matching. ? matches one char, * matches zero or more.

Example: Input: s = "aa", p = "a" → Output: false

Example: Input: s = "aa", p = "*" → Output: true

Approach: Dynamic Programming

1
dp[i][j] = true if s[0..i-1] matches p[0..j-1]
2
Handle empty string matching patterns like * or **.
3
For each position, check if chars match or use wildcard rules.

Solution

csharp 
public bool IsMatch(string s, string p) {
    int m = s.Length, n = p.Length;
    bool[,] dp = new bool[m + 1, n + 1];
    dp[0, 0] = true;

    // Handle patterns like * or **
    for (int j = 1; j <= n; j++) {
        if (p[j - 1] == '*')
            dp[0, j] = dp[0, j - 1];
    }

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (p[j - 1] == '*') {
                // * matches zero chars OR one or more chars
                dp[i, j] = dp[i, j - 1] || dp[i - 1, j];
            } else if (p[j - 1] == '?' || s[i - 1] == p[j - 1]) {
                // ? matches any char OR exact char match
                dp[i, j] = dp[i - 1, j - 1];
            }
        }
    }

    return dp[m, n];
}
Time
O(m*n)
Space
O(m*n)
⚠️
Tricky Part: * can match zero characters (dp[i, j-1]) or one or more (dp[i-1, j]). The OR captures both possibilities.
Regular Expression Matching

Pattern matching with '.' (single char) and '*' (zero or more of preceding).

Hard Google Facebook
11

Problem

Implement regex matching. '.' matches single char, '*' matches zero or more of the preceding element.

Example: Input: s = "aa", p = "a" → Output: false

Example: Input: s = "aa", p = "a*" → Output: true

Approach: Dynamic Programming

dp[i][j] = true if s[0..i-1] matches p[0..j-1]. Handle '*' carefully: it can match zero or more of previous char.

Solution

csharp 
public bool IsMatch(string s, string p) {
    int m = s.Length, n = p.Length;
    bool[,] dp = new bool[m + 1, n + 1];
    dp[0, 0] = true;

    // Handle patterns like a* or a*b* (zero occurrences)
    for (int j = 2; j <= n; j++) {
        if (p[j - 1] == '*')
            dp[0, j] = dp[0, j - 2];
    }

    for (int i = 1; i <= m; i++) {
        for (int j = 1; j <= n; j++) {
            if (p[j - 1] == '*') {
                // Match zero: skip pattern chars
                dp[i, j] = dp[i, j - 2];
                // Match one or more: check if previous chars match
                if (p[j - 2] == '.' || p[j - 2] == s[i - 1])
                    dp[i, j] = dp[i, j] || dp[i - 1, j];
            } else {
                // Normal char or dot
                if (p[j - 1] == '.' || p[j - 1] == s[i - 1])
                    dp[i, j] = dp[i - 1, j - 1];
            }
        }
    }

    return dp[m, n];
}
Time
O(m*n)
Space
O(m*n)

Key Differences from Wildcard

  • '*' applies to preceding element (must check p[j-2])
  • '.' matches exactly one character
  • More restrictive than wildcard matching
🚨
Critical Difference: Unlike wildcard, '*' here refers to the previous character. So when you see '*', you must check if the previous character in the pattern matches the current char in the string.
Implement strStr / KMP

Find first occurrence of needle substring in haystack (KMP algorithm).

Medium Google Microsoft
12

Problem

Return the index of the first occurrence of needle in haystack, or -1 if not found.

Example: Input: haystack = "sadbutsad", needle = "sad" → Output: 0

Approach: KMP Algorithm

1
Build prefix table: pi[i] = longest proper prefix of needle[0..i] that is also a suffix.
2
Scan haystack with two pointers, using prefix table to avoid redundant comparisons on mismatch.
3
When pattern fully matches, return haystack index.

Solution

csharp 
public int StrStr(string haystack, string needle) {
    if (needle.Length == 0) return 0;

    int[] pi = BuildPrefixTable(needle);
    int j = 0;

    for (int i = 0; i < haystack.Length; i++) {
        while (j > 0 && haystack[i] != needle[j])
            j = pi[j - 1];

        if (haystack[i] == needle[j])
            j++;

        if (j == needle.Length)
            return i - needle.Length + 1;
    }

    return -1;
}

private int[] BuildPrefixTable(string pattern) {
    int n = pattern.Length;
    int[] pi = new int[n];
    int j = 0;

    for (int i = 1; i < n; i++) {
        while (j > 0 && pattern[i] != pattern[j])
            j = pi[j - 1];

        if (pattern[i] == pattern[j])
            j++;

        pi[i] = j;
    }

    return pi;
}
Time
O(n + m)
Space
O(m)

n = haystack length, m = needle length

Prefix Table Intuition

For needle = "ABABAC":

i012345
needle[i]ABABAC
pi[i]001230
💡
Interview Tip: Most interviewers accept simpler solutions like haystack.IndexOf(needle) or two-pointer brute force (O(n*m)). KMP shows advanced understanding, but is not always necessary. Mention the tradeoff: KMP is faster asymptotically but has more complex code.
