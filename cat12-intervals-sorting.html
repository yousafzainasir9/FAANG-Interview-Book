Foundation & Patterns

Intervals and sorting problems dominate FAANG interviews because they combine algorithm design with practical real-world scenarios—calendar scheduling, resource allocation, data merging.

Core Patterns

Merge Pattern: Sort by start, iterate left-to-right, expand when overlapping. Used in 5+ FAANG problems annually.

Sweep-Line: Track events (start/end) on a timeline. Powers meeting room allocation and interval intersection.

Greedy Selection: For non-overlapping intervals, pick the earliest-ending option at each step.

Custom Comparators: C# Array.Sort() with IComparer<T> beats LINQ OrderBy() for large datasets (10K+ elements).

Sorting Algorithms in C#

C# uses Introsort (hybrid quicksort/heapsort) by default. For custom sorts:

  • Array.Sort(): O(n log n) average, in-place, faster for large arrays.
  • LINQ OrderBy(): Cleaner syntax but creates new array—use only for readability on small data.
  • PriorityQueue<T, P> (C# 10+): Min-heap, perfect for meeting rooms and event processing.
💡
FAANG Tip: Interviewers test your awareness of stability. Ask: "If two intervals have the same start, which should come first?" This shows you think about edge cases. C# sorts are not stable by default.

Merge Intervals

Given an array of intervals, merge overlapping ones and return the result.

Medium Google Amazon Meta
1

Problem Statement

Given an array of intervals intervals where intervals[i] = [starti, endi], merge all overlapping intervals, and return an array of the non-overlapping intervals.

Example: [[1,3],[2,6],[8,10],[15,18]] → [[1,6],[8,10],[15,18]]

Approach

1
Sort by start position. O(n log n).
2
Iterate. If current start ≤ last end, merge (extend end). Else, add new interval.
3
Return merged list.
Time
O(n log n)
Space
O(1) or O(n)

C# Solution

csharp 
public class Solution {
    public int[][] Merge(int[][] intervals) {
        if (intervals.Length <= 1) return intervals;

        // Sort by start position
        System.Array.Sort(intervals,
            (a, b) => a[0].CompareTo(b[0]));

        var result = new List<int[]>();
        int currentStart = intervals[0][0];
        int currentEnd = intervals[0][1];

        for (int i = 1; i < intervals.Length; i++) {
            // Overlapping: extend current end
            if (intervals[i][0] <= currentEnd) {
                currentEnd = Math.Max(currentEnd, intervals[i][1]);
            } else {
                // Non-overlapping: add merged interval
                result.Add(new int[] { currentStart, currentEnd });
                currentStart = intervals[i][0];
                currentEnd = intervals[i][1];
            }
        }
        result.Add(new int[] { currentStart, currentEnd });

        return result.ToArray();
    }
}

Edge Cases

  • Empty or single interval → return as-is.
  • All intervals overlap → return one merged interval.
  • Intervals with same start but different ends → sort by start, extend correctly.
⚠️
Common Pitfall: Forgetting to add the last merged interval after the loop. Always add outside the loop or track state carefully.
📌
FAANG Pattern: This is the foundation for 5+ derived problems. Master the mechanics: sort, iterate, compare current.start vs prev.end, merge or add.

Insert Interval

Insert a new interval into a list of non-overlapping intervals and merge if needed.

Medium Google Amazon
2

Problem Statement

Given a list of non-overlapping intervals sorted by start time and a new interval, insert it and merge overlapping intervals.

Example: [[1,2],[3,5],[6,9]], newInterval [4,8] → [[1,2],[3,9]]

Approach

1
Add non-overlapping before: intervals that end before newInterval starts.
2
Merge overlapping: expand newInterval to cover all overlaps.
3
Add remaining: all non-overlapping intervals after.
Time
O(n)
Space
O(n)

C# Solution

csharp 
public class Solution {
    public int[][] Insert(int[][] intervals, int[] newInterval) {
        var result = new List<int[]>();
        int i = 0;

        // Add all intervals that end before newInterval starts
        while (i < intervals.Length && intervals[i][1] < newInterval[0]) {
            result.Add(intervals[i]);
            i++;
        }

        // Merge overlapping intervals
        while (i < intervals.Length && intervals[i][0] <= newInterval[1]) {
            newInterval[0] = Math.Min(newInterval[0], intervals[i][0]);
            newInterval[1] = Math.Max(newInterval[1], intervals[i][1]);
            i++;
        }
        result.Add(newInterval);

        // Add remaining non-overlapping intervals
        while (i < intervals.Length) {
            result.Add(intervals[i]);
            i++;
        }

        return result.ToArray();
    }
}

Key Insight

No sorting needed—input is already sorted. Iterate once, partition into three groups: before, overlap, after. This linear approach beats sorting.

💡
FAANG Tip: Interviewer might ask: "What if we sorted everything and called Merge Intervals?" Answer: Yes, O(n log n) works but wastes the sorted input. Linear O(n) is better—show you recognize the constraint.

Non-Overlapping Intervals

Remove minimum number of intervals to make rest non-overlapping.

Medium Google
3

Problem Statement

Given an array of intervals, return the minimum number of intervals you need to remove to make the rest non-overlapping.

Example: [[1,2],[2,3],[3,4],[1,3]] → 1

Approach — Greedy

Sort by end time. At each step, keep the interval that ends earliest—it leaves max room for future intervals.

1
Sort by end position.
2
Track last kept end. If next start ≥ last end, keep it. Else skip.
3
Count removed = total − kept.
Time
O(n log n)
Space
O(1)

C# Solution

csharp 
public class Solution {
    public int EraseOverlapIntervals(int[][] intervals) {
        // Sort by end time
        System.Array.Sort(intervals,
            (a, b) => a[1].CompareTo(b[1]));

        int kept = 1;
        int lastEnd = intervals[0][1];

        for (int i = 1; i < intervals.Length; i++) {
            // Non-overlapping: start >= last end
            if (intervals[i][0] >= lastEnd) {
                kept++;
                lastEnd = intervals[i][1];
            }
        }

        return intervals.Length - kept;
    }
}

Why Greedy Works

Proof sketch: Suppose optimal solution picks interval X where end(X) > end(earliest). Swap X with earliest; you still have a valid solution, and earliest leaves more room. By induction, picking earliest-ending is optimal.

💡
FAANG Insight: This is a classic greedy + activity selection pattern. Interviewers love this because it tests intuition. Many candidates sort by start (wrong—doesn't guarantee optimality).

Meeting Rooms

Check if one person can attend all meetings.

Easy Google Facebook
4

Problem Statement

Given an array of meeting intervals, determine if a person can attend all meetings.

Example: [[0,30],[5,10],[15,20]] → false (overlaps)

Approach

Sort by start time. Check if any two consecutive meetings overlap.

Time
O(n log n)
Space
O(1)

C# Solution

csharp 
public class Solution {
    public bool CanAttendMeetings(int[][] intervals) {
        System.Array.Sort(intervals,
            (a, b) => a[0].CompareTo(b[0]));

        for (int i = 1; i < intervals.Length; i++) {
            if (intervals[i][0] < intervals[i - 1][1]) {
                return false;
            }
        }
        return true;
    }
}

Meeting Rooms II

Find minimum number of meeting rooms needed.

Medium Google Amazon
5

Problem Statement

Given an array of meeting intervals, find the minimum number of meeting rooms required.

Example: [[0,30],[5,10],[15,20]] → 2 (at time 5-10, need 2 rooms)

Approach — Min-Heap (Sweep-Line)

Sort by start. Use heap to track which room frees up earliest. When a new meeting starts, reuse earliest-free room if possible.

Time
O(n log n)
Space
O(n)

C# Solution

csharp 
public class Solution {
    public int MinMeetingRooms(int[][] intervals) {
        System.Array.Sort(intervals,
            (a, b) => a[0].CompareTo(b[0]));

        // Min-heap: track end times of rooms
        var minHeap = new PriorityQueue<int, int>();

        foreach (var interval in intervals) {
            // Room available: reuse it
            if (minHeap.Count > 0 && minHeap.Peek() <= interval[0]) {
                minHeap.Dequeue();
            }
            // Allocate/extend room
            minHeap.Enqueue(interval[1], interval[1]);
        }

        return minHeap.Count;
    }
}
📌
C# 10+ Tip: PriorityQueue<T, TPriority> is built-in. In earlier versions, use SortedSet<T> or implement a min-heap.

Minimum Number of Arrows to Burst Balloons

Find minimum arrows to burst all balloons, greedy selection.

Medium Google
6

Problem Statement

Given balloon positions (intervals), find minimum arrows. One arrow bursts all overlapping balloons.

Example: [[10,16],[2,8],[1,6],[7,12]] → 2

Approach

Sort by end position. Greedily shoot at the rightmost position of first balloon, burst all overlapping. Repeat.

Time
O(n log n)
Space
O(1)

C# Solution

csharp 
public class Solution {
    public int FindMinArrowShots(int[][] points) {
        System.Array.Sort(points,
            (a, b) => a[1].CompareTo(b[1]));

        int arrows = 1;
        int lastArrow = points[0][1];

        for (int i = 1; i < points.Length; i++) {
            // No overlap: need new arrow
            if (points[i][0] > lastArrow) {
                arrows++;
                lastArrow = points[i][1];
            }
        }

        return arrows;
    }
}
⚠️
Integer Overflow: Comparing end positions may overflow with large int values. Use long for comparators or compare differently.

Employee Free Time

Find common free time across employee schedules (hard merge problem).

Hard Google Amazon
7

Problem Statement

Given schedules of N employees (each with multiple meetings), find intervals when all are free.

Example: Employee 1: [[1,3],[4,6]], Employee 2: [[2,5],[7,9]] → common free: [[6,7]]

Approach

Flatten all intervals, merge overlaps to get busy time, then find gaps.

Time
O(n log n)
Space
O(n)

C# Solution

csharp 
public class Solution {
    public IList<Interval> EmployeeFreeTime(IList<IList<Interval>> schedule) {
        var allIntervals = new List<Interval>();

        // Flatten all employee schedules
        foreach (var emp in schedule) {
            foreach (var interval in emp) {
                allIntervals.Add(interval);
            }
        }

        // Sort by start
        allIntervals.Sort((a, b) => a.start.CompareTo(b.start));

        // Merge overlapping
        var merged = new List<Interval>();
        int currentStart = allIntervals[0].start;
        int currentEnd = allIntervals[0].end;

        for (int i = 1; i < allIntervals.Count; i++) {
            if (allIntervals[i].start <= currentEnd) {
                currentEnd = Math.Max(currentEnd, allIntervals[i].end);
            } else {
                merged.Add(new Interval(currentStart, currentEnd));
                currentStart = allIntervals[i].start;
                currentEnd = allIntervals[i].end;
            }
        }
        merged.Add(new Interval(currentStart, currentEnd));

        // Find gaps (free time)
        var result = new List<Interval>();
        for (int i = 0; i < merged.Count - 1; i++) {
            result.Add(new Interval(merged[i].end, merged[i + 1].start));
        }

        return result;
    }
}

Interval List Intersections

Find overlapping intervals between two sorted lists using two pointers.

Medium Google Amazon
8

Problem Statement

Given two lists of sorted intervals, find all their intersections.

Example: A = [[0,2],[5,10],[13,23],[24,25]], B = [[1,5],[8,12],[15,24],[25,26]] → [[1,2],[5,5],[8,10],[15,23],[24,24],[25,25]]

Approach — Two Pointers

Iterate both lists. At each step, check overlap. Move the pointer with earlier end.

Time
O(m + n)
Space
O(1)

C# Solution

csharp 
public class Solution {
    public int[][] IntervalIntersection(int[][] firstList, int[][] secondList) {
        var result = new List<int[]>();
        int i = 0, j = 0;

        while (i < firstList.Length && j < secondList.Length) {
            // Find intersection range
            int start = Math.Max(firstList[i][0], secondList[j][0]);
            int end = Math.Min(firstList[i][1], secondList[j][1]);

            // Overlap exists
            if (start <= end) {
                result.Add(new int[] { start, end });
            }

            // Move pointer of interval that ends first
            if (firstList[i][1] < secondList[j][1]) {
                i++;
            } else {
                j++;
            }
        }

        return result.ToArray();
    }
}
💡
Key Insight: Intersection of [a,b] and [c,d] is [max(a,c), min(b,d)] if that's valid (start ≤ end). This pattern applies to many geometric/interval problems.

Sort an Array

Implement merge sort or use built-in sorting efficiently.

Medium Google Amazon
9

Problem Statement

Sort an array using merge sort (or compare Array.Sort vs LINQ OrderBy performance).

Approach — Merge Sort

Divide-and-conquer: split array, recursively sort halves, merge. O(n log n) guaranteed, stable.

Time
O(n log n)
Space
O(n)

C# Solution

csharp 
public class Solution {
    public int[] SortArray(int[] nums) {
        if (nums.Length <= 1) return nums;
        MergeSort(nums, 0, nums.Length - 1);
        return nums;
    }

    private void MergeSort(int[] nums, int left, int right) {
        if (left >= right) return;

        int mid = left + (right - left) / 2;
        MergeSort(nums, left, mid);
        MergeSort(nums, mid + 1, right);
        Merge(nums, left, mid, right);
    }

    private void Merge(int[] nums, int left, int mid, int right) {
        int[] temp = new int[right - left + 1];
        int i = left, j = mid + 1, k = 0;

        // Merge two sorted halves
        while (i <= mid && j <= right) {
            if (nums[i] <= nums[j]) {
                temp[k++] = nums[i++];
            } else {
                temp[k++] = nums[j++];
            }
        }

        // Copy remaining
        while (i <= mid) temp[k++] = nums[i++];
        while (j <= right) temp[k++] = nums[j++];

        // Copy back
        System.Array.Copy(temp, 0, nums, left, temp.Length);
    }
}

Real-World: Array.Sort vs OrderBy

Array.Sort(nums) is faster for 10K+ elements (in-place, no allocation). nums.OrderBy(x => x) creates new array (memory hit) but cleaner syntax for small data.

⚠️
Interview Tip: If asked to implement sorting, always use merge sort (stable, O(n log n) guaranteed). Quicksort is risky (O(n²) worst case).

Maximum Gap

Find maximum gap between sorted array elements using bucket or radix sort.

Medium Google
10

Problem Statement

Find the maximum gap between consecutive elements after sorting the array. Constraint: solve in O(n) time.

Example: [3,9,3,20] → sorted [3,3,9,20] → max gap = 11 (between 9 and 20)

Approach — Radix Sort

Classic O(n) sort: no comparisons, sort digit-by-digit. Then find gap.

Time
O(n)
Space
O(n)

C# Solution (Counting Sort for simplicity)

csharp 
public class Solution {
    public int MaximumGap(int[] nums) {
        if (nums.Length < 2) return 0;

        // Radix sort
        List<int> arr = new List<int>(nums);
        RadixSort(arr);

        // Find max gap
        int maxGap = 0;
        for (int i = 1; i < arr.Count; i++) {
            maxGap = Math.Max(maxGap, arr[i] - arr[i - 1]);
        }
        return maxGap;
    }

    private void RadixSort(List<int> nums) {
        int maxNum = int.MinValue;
        foreach (int num in nums) {
            maxNum = Math.Max(maxNum, num);
        }

        // Sort by each digit
        for (int exp = 1; maxNum / exp > 0; exp *= 10) {
            CountingSort(nums, exp);
        }
    }

    private void CountingSort(List<int> nums, int exp) {
        int[] output = new int[nums.Count];
        int[] count = new int[10];

        // Store frequencies
        foreach (int num in nums) {
            count[(num / exp) % 10]++;
        }

        // Cumulative count
        for (int i = 1; i < 10; i++) {
            count[i] += count[i - 1];
        }

        // Build output
        for (int i = nums.Count - 1; i >= 0; i--) {
            output[count[(nums[i] / exp) % 10] - 1] = nums[i];
            count[(nums[i] / exp) % 10]--;
        }

        // Copy back
        for (int i = 0; i < nums.Count; i++) {
            nums[i] = output[i];
        }
    }
}
📌
FAANG Insight: Radix sort beats comparison-based sorts when dealing with non-negative integers. This is a rare O(n) sort problem—shows deep algorithm knowledge.

Largest Number

Arrange numbers to form the largest possible number using custom comparator.

Medium Google Amazon
11

Problem Statement

Given an array of integers, arrange them to form the largest number possible.

Example: [3,30,34,5,9] → "9534330"

Approach

Sort with custom comparator: for two numbers a, b, compare concatenations ab vs ba. If ab > ba, a comes first.

Time
O(n log n)
Space
O(n)

C# Solution

csharp 
public class Solution {
    public string LargestNumber(int[] nums) {
        // Convert to strings
        string[] strs = Array.ConvertAll(nums, x => x.ToString());

        // Custom sort: compare concatenations
        System.Array.Sort(strs, (a, b) => {
            string ab = a + b;
            string ba = b + a;
            return ba.CompareTo(ab); // Descending
        });

        // Handle edge case: all zeros
        if (strs[0][0] == '0') return "0";

        return string.Join("", strs);
    }
}

Edge Cases

  • All zeros: [0,0] → "0" (not "00")
  • Mixed sizes: [3,30,34,5,9] requires careful ordering
💡
Why This Works: If ab > ba, placing a before b gives a larger result. This order is transitive and defines a total order on the numbers.

Wiggle Sort II

Rearrange array so that nums[0] < nums[1] > nums[2] < nums[3]...

Medium Google
12

Problem Statement

Given an array, rearrange so that nums[0] < nums[1] > nums[2] < nums[3] > ...

Example: [1,5,1,1,6,4] → [1,6,1,5,1,4]

Approach

Sort array. Map elements cleverly: first half goes to even indices, second half to odd indices in reverse. Avoids duplicates side-by-side.

Time
O(n log n)
Space
O(n)

C# Solution

csharp 
public class Solution {
    public void WiggleSort(int[] nums) {
        Array.Sort(nums);
        int[] result = new int[nums.Length];

        // Place smaller half at even indices
        int mid = (nums.Length - 1) / 2;
        int idx = 0;
        for (int i = 0; i <= mid; i++) {
            result[idx] = nums[i];
            idx += 2;
        }

        // Place larger half at odd indices in reverse
        idx = nums.Length % 2 == 0 ? nums.Length - 1 : nums.Length - 2;
        for (int i = nums.Length - 1; i > mid; i--) {
            result[idx] = nums[i];
            idx -= 2;
        }

        Array.Copy(result, 0, nums, 0, nums.Length);
    }
}

Why This Works

After sort: smaller values at front, larger at back. Placing small values at even indices and large at odd indices ensures every even index is less than adjacent odd indices.

⚠️
Index Mapping: Tricky to get right. Many candidates confuse the index placement logic. Draw it out for small examples first.
