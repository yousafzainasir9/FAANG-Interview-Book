INTRO

Tree Fundamentals

Before tackling the 15 problems, understand the core patterns and structures that make tree problems solvable.

TreeNode Class in C#

All problems in this chapter use this standard TreeNode definition:

C# 
public class TreeNode {
    public int val;
    public TreeNode left;
    public TreeNode right;
    public TreeNode(int val = 0, TreeNode left = null, TreeNode right = null) {
        this.val = val;
        this.left = left;
        this.right = right;
    }
}

Traversal Patterns

Depth-First Search (DFS) explores as far as possible along each branch before backtracking. Three main variants:

1
Preorder (Root-Left-Right): Process node before children. Used for copying trees, prefix expressions.
2
Inorder (Left-Root-Right): Process node between children. For BSTs, yields values in sorted order.
3
Postorder (Left-Right-Root): Process node after children. Used for deletion, computing heights.

Breadth-First Search (BFS) explores level by level using a queue. Essential for level-order problems.

💡 Recursion vs Iteration: Recursive DFS is cleaner but risks stack overflow on deep trees. Iterative approaches with explicit stacks handle large trees safely.
1. Invert Binary Tree

Flip a tree horizontally so all left and right children swap positions.

Easy Google Amazon
01

Problem Statement

Given the root of a binary tree, invert the tree and return its root. Inversion means every left child becomes a right child and vice versa, recursively for the entire tree.

Approach

Use DFS recursion: visit each node, swap its children, then recursively invert the left and right subtrees. This achieves complete inversion in a single pass.

Solution

C# 
public TreeNode InvertTree(TreeNode root) {
    if (root == null) return null;

    // Swap children
    TreeNode temp = root.left;
    root.left = root.right;
    root.right = temp;

    // Recursively invert subtrees
    InvertTree(root.left);
    InvertTree(root.right);

    return root;
}
Time Complexity
O(n)
Space Complexity
O(h)
Edge Cases
Empty tree, single node
⭐ FAANG Tip: Mention you can do this iteratively with a queue/stack if the interviewer asks about space. The recursive solution is preferred for its clarity and is O(h) space due to call stack.
2. Maximum Depth of Binary Tree

Find the height of the tree—the longest path from root to any leaf.

Easy Google Meta
02

Problem Statement

Given a binary tree, return its maximum depth—the number of nodes along the longest path from the root to a leaf node.

Approach

Recursively compute the maximum depth of left and right subtrees, then return 1 + max(left_depth, right_depth). Base case is null nodes returning 0.

Solution

C# 
public int MaxDepth(TreeNode root) {
    if (root == null) return 0;

    // Recursively find max depth of subtrees
    int leftDepth = MaxDepth(root.left);
    int rightDepth = MaxDepth(root.right);

    return 1 + Math.Max(leftDepth, rightDepth);
}
Time Complexity
O(n)
Space Complexity
O(h)
Edge Cases
Null root, skewed tree
ℹ️ Variant: Minimum depth counts nodes on shortest path to any leaf. Return 0 if root is null, return 1 + min(left, right) only if both children exist; else return 1 + max(left, right).
3. Same Tree

Determine if two binary trees are structurally identical with the same values.

Easy Amazon Apple
03

Problem Statement

Given two binary trees, return true if they are identical (same structure and same node values at each position).

Approach

Compare node values at each position using DFS. Both nodes must be null, both must have equal values, and their left and right subtrees must be identical.

Solution

C# 
public bool IsSameTree(TreeNode p, TreeNode q) {
    // Both null: identical
    if (p == null && q == null) return true;

    // One null or values differ: not identical
    if (p == null || q == null || p.val != q.val) return false;

    // Recursively check left and right subtrees
    return IsSameTree(p.left, q.left) && IsSameTree(p.right, q.right);
}
Time Complexity
O(min(m,n))
Space Complexity
O(h)
Edge Cases
Both null, one null, different values
⭐ FAANG Tip: Short-circuit evaluation early—if values don't match or one is null, return false immediately rather than exploring further.
4. Subtree of Another Tree

Check if one tree is a subtree of another (same structure and values).

Easy Amazon Facebook
04

Problem Statement

Given two trees root and subRoot, return true if subRoot is a subtree of root. A subtree must have the same structure and values; it doesn't have to be rooted at the root of root.

Approach

Traverse the main tree and at each node, check if the subtree rooted at that node is identical to subRoot using the same tree comparison logic.

Solution

C# 
public bool IsSubtree(TreeNode root, TreeNode subRoot) {
    if (root == null) return subRoot == null;

    // Check if subtree matches at current node or in children
    if (IsSameTree(root, subRoot)) return true;

    return IsSubtree(root.left, subRoot) || IsSubtree(root.right, subRoot);
}

private bool IsSameTree(TreeNode p, TreeNode q) {
    if (p == null && q == null) return true;
    if (p == null || q == null || p.val != q.val) return false;
    return IsSameTree(p.left, q.left) && IsSameTree(p.right, q.right);
}
Time Complexity
O(m*n)
Space Complexity
O(h)
Edge Cases
Null trees, identical trees
⚠️ Note: Worst-case O(m*n) because we potentially call IsSameTree at each of m nodes. Advanced: use hashing to achieve O(m+n).
5. Lowest Common Ancestor of BST

Find the deepest node that is an ancestor of both p and q in a BST.

Medium Google Amazon
05

Problem Statement

Given a binary search tree (BST) and two nodes p and q, return their lowest common ancestor (LCA)—the deepest node that is an ancestor of both.

Approach

Exploit BST property: if both p and q are less than current node, go left; if both are greater, go right; otherwise, the current node is their LCA.

Solution

C# 
public TreeNode LowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
    // Both p and q are smaller: LCA is in left subtree
    if (p.val < root.val && q.val < root.val) {
        return LowestCommonAncestor(root.left, p, q);
    }

    // Both p and q are larger: LCA is in right subtree
    if (p.val > root.val && q.val > root.val) {
        return LowestCommonAncestor(root.right, p, q);
    }

    // One on each side or at current node: current is LCA
    return root;
}
Time Complexity
O(log n)
Space Complexity
O(log n)
Edge Cases
p or q is root, same node
⭐ FAANG Tip: This is much faster than a general binary tree LCA (which requires O(n)). Always leverage BST properties. Mention you can do iterative version to save call stack space.
6. Binary Tree Level Order Traversal

Visit all nodes level by level, returning them as a list of lists.

Medium Google Microsoft
06

Problem Statement

Given a binary tree, return the level order traversal (BFS result) as a list of lists, where each inner list contains all values at that depth level.

Approach

Use a queue: enqueue the root, then process each level by dequeueing all current nodes, collecting their values, and enqueueing their children.

Solution

C# 
public IList<IList<int>> LevelOrder(TreeNode root) {
    IList<IList<int>> result = new List<IList<int>>();
    if (root == null) return result;

    Queue<TreeNode> queue = new Queue<TreeNode>();
    queue.Enqueue(root);

    while (queue.Count > 0) {
        int levelSize = queue.Count;
        IList<int> level = new List<int>();

        // Process all nodes at current level
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.Dequeue();
            level.Add(node.val);

            if (node.left != null) queue.Enqueue(node.left);
            if (node.right != null) queue.Enqueue(node.right);
        }

        result.Add(level);
    }

    return result;
}
Time Complexity
O(n)
Space Complexity
O(w)
Edge Cases
Null root, single node
ℹ️ Key Point: The queue size at each iteration tells you how many nodes are at that level. Use this to properly separate levels without extra markers.
7. Validate Binary Search Tree

Verify that a tree satisfies the BST property: left < parent < right.

Medium Google Amazon
07

Problem Statement

Given a binary tree, determine if it is a valid BST. A valid BST must have all left descendants less than the node, all right descendants greater, and this must hold recursively.

Approach

Recursively validate with bounds: each node must be within a valid range. For left children, update the upper bound; for right children, update the lower bound.

Solution

C# 
public bool IsValidBST(TreeNode root) {
    return Validate(root, null, null);
}

private bool Validate(TreeNode node, TreeNode min, TreeNode max) {
    if (node == null) return true;

    // Check if node violates bounds
    if (min != null && node.val <= min.val) return false;
    if (max != null && node.val >= max.val) return false;

    // Left subtree: update max bound
    // Right subtree: update min bound
    return Validate(node.left, min, node) && Validate(node.right, node, max);
}
Time Complexity
O(n)
Space Complexity
O(h)
Edge Cases
Invalid BST, single node, all same values
⚠️ Common Mistake: Checking only immediate children (left.val < root.val && right.val > root.val) is wrong. All left descendants must be smaller, all right descendants larger.
8. Kth Smallest Element in BST

Find the kth smallest value in a binary search tree.

Medium Google Microsoft
08

Problem Statement

Given a BST and an integer k, return the value of the kth smallest element in the tree (1-indexed).

Approach

Use inorder traversal (Left-Root-Right) which visits nodes in sorted order. Count nodes visited and return the kth one.

Solution

C# 
private int count = 0;
private int result = 0;

public int KthSmallest(TreeNode root, int k) {
    count = 0;
    Inorder(root, k);
    return result;
}

private void Inorder(TreeNode node, int k) {
    if (node == null) return;

    Inorder(node.left, k);

    count++;
    if (count == k) {
        result = node.val;
        return;
    }

    Inorder(node.right, k);
}
Time Complexity
O(k)
Space Complexity
O(h)
Edge Cases
k=1, k=n, single node
⭐ FAANG Tip: Inorder traversal of a BST gives sorted order—this is a key property to exploit. Early termination (return when count==k) avoids unnecessary traversal.
9. Construct Binary Tree from Preorder and Inorder

Rebuild a tree given its preorder and inorder traversals.

Medium Google Amazon
09

Problem Statement

Given preorder and inorder traversal arrays, construct and return the binary tree. Preorder is [root | left | right], inorder is [left | root | right].

Approach

The first element of preorder is always the root. Find it in inorder to split left and right subtrees, then recursively build subtrees using corresponding array slices.

Solution

C# 
public TreeNode BuildTree(int[] preorder, int[] inorder) {
    if (preorder.Length == 0) return null;

    // Root is first element of preorder
    int rootVal = preorder[0];
    TreeNode root = new TreeNode(rootVal);

    // Find root position in inorder
    int rootIdx = Array.IndexOf(inorder, rootVal);

    // Split arrays and recursively build subtrees
    root.left = BuildTree(
        preorder.Skip(1).Take(rootIdx).ToArray(),
        inorder.Take(rootIdx).ToArray()
    );

    root.right = BuildTree(
        preorder.Skip(rootIdx + 1).ToArray(),
        inorder.Skip(rootIdx + 1).ToArray()
    );

    return root;
}
Time Complexity
O(n²)
Space Complexity
O(n)
Edge Cases
Empty arrays, single element
⚠️ Optimization: This solution is O(n²) due to array operations and IndexOf. Use a HashMap to store inorder indices for O(n) lookup, achieving O(n) overall.
10. Binary Tree Right Side View

Return the rightmost node at each level when viewing the tree from the right.

Medium Google Meta
10

Problem Statement

Given a binary tree, imagine standing on the right side and looking at the tree. Return a list of values you can see, ordered from top to bottom.

Approach

Use BFS level-order traversal and record the last node value at each level, or use DFS with preorder traversal, visiting right before left.

Solution

C# 
public IList<int> RightSideView(TreeNode root) {
    IList<int> result = new List<int>();
    if (root == null) return result;

    Queue<TreeNode> queue = new Queue<TreeNode>();
    queue.Enqueue(root);

    while (queue.Count > 0) {
        int levelSize = queue.Count;

        // Process all nodes at current level
        for (int i = 0; i < levelSize; i++) {
            TreeNode node = queue.Dequeue();

            // Record rightmost node (last one processed at each level)
            if (i == levelSize - 1) {
                result.Add(node.val);
            }

            if (node.left != null) queue.Enqueue(node.left);
            if (node.right != null) queue.Enqueue(node.right);
        }
    }

    return result;
}
Time Complexity
O(n)
Space Complexity
O(w)
Edge Cases
Single node, left-skewed tree
ℹ️ Alternative: Use DFS with preorder (visit right first), tracking depth. Add to result if we're seeing this depth for the first time.
11. Count Good Nodes in Binary Tree

Count nodes where the node value is greater than or equal to all ancestor values.

Medium Google Amazon
11

Problem Statement

Given a binary tree, a "good" node is one with a value that is greater than or equal to all ancestor values on the path from the root. Count the number of good nodes.

Approach

Use DFS, tracking the maximum value seen so far on the current path from the root. A node is "good" if its value is >= the max on its path.

Solution

C# 
public int GoodNodes(TreeNode root) {
    return CountGood(root, int.MinValue);
}

private int CountGood(TreeNode node, int maxSoFar) {
    if (node == null) return 0;

    // Current node is good if value >= max on path
    int count = 0;
    if (node.val >= maxSoFar) {
        count = 1;
        maxSoFar = node.val;
    }

    // Recursively count good nodes in subtrees
    count += CountGood(node.left, maxSoFar);
    count += CountGood(node.right, maxSoFar);

    return count;
}
Time Complexity
O(n)
Space Complexity
O(h)
Edge Cases
Null root, all nodes good, no nodes good
⭐ FAANG Tip: Key insight is passing maxSoFar through the recursion to track path context. Update it only if the current node is good—this ensures we're comparing against ancestors.
12. Serialize and Deserialize Binary Tree

Encode and decode a tree to/from a string representation.

Hard Google Amazon
12

Problem Statement

Design an algorithm to serialize a binary tree to a string and deserialize the string back to the tree. The format is arbitrary but must support null nodes.

Approach

Serialize using preorder DFS with null markers (e.g., "#"). Deserialize by parsing the string into tokens and reconstructing via preorder traversal.

Solution

C# 
public string Serialize(TreeNode root) {
    StringBuilder sb = new StringBuilder();
    SerializeHelper(root, sb);
    return sb.ToString();
}

private void SerializeHelper(TreeNode node, StringBuilder sb) {
    if (node == null) {
        sb.Append("#,");
        return;
    }

    sb.Append(node.val).Append(",");
    SerializeHelper(node.left, sb);
    SerializeHelper(node.right, sb);
}

public TreeNode Deserialize(string data) {
    Queue<string> tokens = new Queue<string>(data.Split(','));
    return DeserializeHelper(tokens);
}

private TreeNode DeserializeHelper(Queue<string> tokens) {
    string val = tokens.Dequeue();
    if (val == "#") return null;

    TreeNode root = new TreeNode(int.Parse(val));
    root.left = DeserializeHelper(tokens);
    root.right = DeserializeHelper(tokens);

    return root;
}
Time Complexity
O(n)
Space Complexity
O(n)
Edge Cases
Null root, single node, duplicate values
⚠️ Complexity: Both serialize and deserialize are O(n). The queue ensures preorder reconstruction without backtracking.
13. Binary Tree Maximum Path Sum

Find the path with maximum sum; path doesn't need to pass through root.

Hard Google Meta
13

Problem Statement

Given a binary tree, find the maximum path sum. A path is any sequence of nodes connected by edges; it can start and end at any node (doesn't need to go through root or reach a leaf).

Approach

Use DFS to track the maximum path sum ending at each node. At each node, compute the max path that goes through it by combining max left and max right paths with the current value.

Solution

C# 
private int maxSum = int.MinValue;

public int MaxPathSum(TreeNode root) {
    maxSum = int.MinValue;
    MaxGain(root);
    return maxSum;
}

private int MaxGain(TreeNode node) {
    if (node == null) return 0;

    // Max gain from left/right; if negative, don't include
    int leftGain = Math.Max(0, MaxGain(node.left));
    int rightGain = Math.Max(0, MaxGain(node.right));

    // Max path through current node
    int pathSum = node.val + leftGain + rightGain;
    maxSum = Math.Max(maxSum, pathSum);

    // Return max path that extends upward (can't use both left and right)
    return node.val + Math.Max(leftGain, rightGain);
}
Time Complexity
O(n)
Space Complexity
O(h)
Edge Cases
Negative values, single node, all negatives
⚠️ Tricky Detail: We return the max path extending upward (can't use both children), but we check the path going through the node (using both) against the global max. This allows the answer to be any internal path.
14. Diameter of Binary Tree

Find the longest path between any two nodes (passing through their LCA).

Easy Google Facebook
14

Problem Statement

Given a binary tree, return the diameter—the length of the longest path between any two nodes. The path may or may not pass through the root.

Approach

At each node, compute the height of left and right subtrees. The diameter through that node is the sum of both heights. Track the maximum diameter seen.

Solution

C# 
private int maxDiameter = 0;

public int DiameterOfBinaryTree(TreeNode root) {
    maxDiameter = 0;
    Height(root);
    return maxDiameter;
}

private int Height(TreeNode node) {
    if (node == null) return 0;

    int leftHeight = Height(node.left);
    int rightHeight = Height(node.right);

    // Diameter through this node
    maxDiameter = Math.Max(maxDiameter, leftHeight + rightHeight);

    return 1 + Math.Max(leftHeight, rightHeight);
}
Time Complexity
O(n)
Space Complexity
O(h)
Edge Cases
Single node, null root, linear tree
ℹ️ Related: This is similar to Maximum Path Sum but we count edges instead of node values. Diameter is height(left) + height(right), not node.val + gains.
15. Balanced Binary Tree

Check if a tree is balanced: height difference between left and right subtrees is at most 1 at every node.

Easy Google Amazon
15

Problem Statement

Given a binary tree, determine if it is height-balanced. A tree is height-balanced if for every node, the heights of its left and right subtrees differ by at most 1.

Approach

Use DFS to compute heights recursively. At each node, check if the height difference is valid; if not, mark the tree as unbalanced. Return both the height and balance status.

Solution

C# 
public bool IsBalanced(TreeNode root) {
    return CheckBalance(root).Item1;
}

private Tuple<bool, int> CheckBalance(TreeNode node) {
    if (node == null)
        return new Tuple<bool, int>(true, 0);

    // Check left and right subtrees
    Tuple<bool, int> left = CheckBalance(node.left);
    Tuple<bool, int> right = CheckBalance(node.right);

    // If either subtree is unbalanced, propagate unbalanced status
    bool isBalanced = left.Item1 && right.Item1 &&
                       Math.Abs(left.Item2 - right.Item2) <= 1;

    int height = 1 + Math.Max(left.Item2, right.Item2);

    return new Tuple<bool, int>(isBalanced, height);
}
Time Complexity
O(n)
Space Complexity
O(h)
Edge Cases
Null root, single node, skewed tree
⭐ FAANG Tip: Returning a tuple (isBalanced, height) avoids traversing the entire tree if unbalanced. Once we detect imbalance, we propagate it up without extra work. This is more efficient than separate height and balance checks.
