Foundation & Techniques

Greedy algorithms make locally optimal choices, hoping for a global optimum. Backtracking explores all possibilities systematically, pruning branches that cannot lead to a solution. Both require strong proof intuition and pattern recognition.

Greedy Proof Techniques

Not every greedy approach works. Two classical proof methods ensure correctness:

Exchange Argument
Assume an optimal solution exists that differs from the greedy choice. Show you can swap part of it with the greedy choice without losing optimality, contradicting the assumption.
Example: Activity Selection. If an optimal solution picks a different first activity, swap it with the greedy pick (earliest end); the swapped solution is still valid.
Stays-Ahead Argument
Prove the greedy solution always stays "ahead" of any other solution. After each step, show the greedy choice achieves a better objective value than any alternative.
Example: Jump Game. The greedy furthest reachable position is always ≥ any other strategy at each step.

Backtracking Template

All backtracking problems follow this pattern: explore candidates, commit to a choice, recurse, then undo the choice.

C# 
void Backtrack(int idx, List<int> path) {
    // Base case: found a valid combination
    if (idx == candidates.Count) {
        if (IsValid(path)) result.Add(new List<int>(path));
        return;
    }

    // Pruning: early exit if impossible to meet goal
    if (!CanProceed(path)) return;

    // Choice: try each candidate
    for (int i = idx; i < candidates.Count; i++) {
        path.Add(candidates[i]);           // Choose
        Backtrack(i + 1, path);            // Explore
        path.RemoveAt(path.Count - 1);     // Unchoose
    }
}

Pruning Strategies

Backtracking becomes exponential without pruning. Key strategies:

PRUNING
Constraint-Based
Skip candidates that immediately violate a constraint (e.g., sum exceeds target in Combination Sum).
PRUNING
Feasibility Check
Use bounds (e.g., max remaining sum in Combination Sum) to detect infeasible branches early.
PRUNING
Memoization
Cache subproblems (Palindrome Partitioning uses DP to pre-compute palindromes).

Greedy vs. DP vs. Backtracking

Greedy: Make a locally optimal choice once. Use when you can prove optimality (Jump Game, Gas Station). O(n) or O(n log n).

Dynamic Programming: Optimal substructure with overlapping subproblems. Trade off space and time for correctness (Coin Change, Edit Distance).

Backtracking: Explore all possibilities with pruning. Use when all solutions matter or proof is hard (N-Queens, Sudoku, Permutations). Exponential time but manageable with strong pruning.

Jump Game
Given an array of non-negative integers, return True if you can reach the last index starting from index 0. Each element is the max jump length from that position.
Medium Amazon Apple Google
1
Time
O(n)
Space
O(1)
Approach
Greedy (Stays Ahead)

Approach

Track the furthest index reachable. If it ever becomes less than the current index, we're stuck. Proof: at each index, pick the jump that reaches the furthest position ahead (greedy). This maximizes our forward progress, so if any strategy reaches the end, so does ours.

C# 
public class Solution {
    public bool CanJump(int[] nums) {
        int maxReach = 0;
        for (int i = 0; i < nums.Length; i++) {
            // If current index is beyond reach, impossible
            if (i > maxReach) return false;
            // Update furthest reachable index
            maxReach = Math.Max(maxReach, i + nums[i]);
            // Early termination
            if (maxReach >= nums.Length - 1) return true;
        }
        return true;
    }
}
💡
Edge Cases: Single element always True. All zeros except last index: False if length > 1. Large jumps: early termination saves time.
🎯
FAANG Tip: Interviewers often ask: "Can you jump to ANY index ≤ max_jump?" (yes) vs. "EXACTLY max_jump?" (hard, use BFS). Clarify the problem statement.
Jump Game II
Return the minimum number of jumps needed to reach the last index. Guaranteed reachable.
Medium Google Meta ByteDance
2
Time
O(n)
Space
O(1)
Approach
BFS / Greedy Levels

Approach

Think of it as BFS: each level in the BFS represents one jump. Track the furthest index reachable in the current level, and the furthest reachable in the next level. When current position exits the current level, increment jumps and move to the next level.

C# 
public class Solution {
    public int Jump(int[] nums) {
        if (nums.Length <= 1) return 0;

        int jumps = 0, currentMaxReach = 0, nextMaxReach = 0;

        for (int i = 0; i < nums.Length - 1; i++) {
            // Expand next level's reach
            nextMaxReach = Math.Max(nextMaxReach, i + nums[i]);

            // Exiting current level, move to next
            if (i == currentMaxReach) {
                jumps++;
                currentMaxReach = nextMaxReach;
            }
        }

        return jumps;
    }
}
💡
Key Insight: Don't need to explore all jumps from a position. Just track when we exhaust the current level and move to the next. This is "BFS without explicit queue."
🎯
FAANG Tip: Mention the BFS connection during explanation. Greedy optimization is a sign of deep algorithm thinking.
Gas Station
Given gas and cost arrays, find the starting gas station index from which you can complete a circular route. Return -1 if impossible.
Medium Amazon Microsoft
3
Time
O(n)
Space
O(1)
Approach
Greedy (Exchange)

Approach

Key observations: (1) If total gas < total cost, no solution exists. (2) If a greedy start position fails, no earlier position will work (exchange argument). Start at index 0; if balance goes negative, jump to the next unvisited position.

C# 
public class Solution {
    public int CanCompleteCircuit(int[] gas, int[] cost) {
        int totalGas = 0, totalCost = 0, currentGas = 0, startIdx = 0;

        for (int i = 0; i < gas.Length; i++) {
            totalGas += gas[i];
            totalCost += cost[i];
            currentGas += gas[i] - cost[i];

            // If balance negative, can't start from 0..i; try next
            if (currentGas < 0) {
                startIdx = i + 1;
                currentGas = 0;
            }
        }

        // If total is negative, no solution exists
        return (totalGas < totalCost) ? -1 : startIdx;
    }
}
⚠
Proof: If we fail at station i, then no station j ≤ i can be a valid start. Because if j worked for 0..i-1, then j also works for 0..i (contradiction). So we reset to i+1.
🎯
FAANG Tip: Explain the exchange argument clearly. Interviewers love to see proof intuition, not just code.
Task Scheduler
Schedule tasks with cooldown. Given task list and cooldown interval n, find the minimum time to complete all tasks (or return -1 if impossible).
Medium Google Bloomberg
4
Time
O(m + n log n)
Space
O(m)
Approach
Greedy + Heap

Approach

Always execute the task with the highest remaining count (greedy: minimize idle time). Use a max heap to track task counts. Simulate the schedule: in each cycle, pop the n+1 most frequent tasks, decrement counts, and push back if > 0. Total time = max(schedule time, total tasks).

C# 
public class Solution {
    public int LeastInterval(char[] tasks, int n) {
        var counts = new Dictionary<char, int>();
        foreach (var t in tasks) {
            if (!counts.ContainsKey(t)) counts[t] = 0;
            counts[t]++;
        }

        var heap = new PriorityQueue<int, int>();
        foreach (var cnt in counts.Values) {
            heap.Enqueue(cnt, -cnt);
        }

        int time = 0;
        while (heap.Count > 0) {
            var temp = new List<int>();
            for (int i = 0; i <= n && heap.Count > 0; i++) {
                var cnt = heap.Dequeue();
                if (cnt - 1 > 0) temp.Add(cnt - 1);
            }
            foreach (var cnt in temp) {
                heap.Enqueue(cnt, -cnt);
            }
            time += temp.Count > 0 ? n + 1 : heap.Count == 0 ? 0 : n + 1;
        }

        return time + tasks.Length - 1;
    }
}
💡
Math Shortcut: (maxFreq - 1) * (n + 1) + numMaxFreq. Minimum slots needed: the most frequent task requires maxFreq-1 gaps + n+1 slots each, plus the last execution.
🎯
FAANG Tip: Mention the mathematical formula as a cleaner alternative. Shows deep understanding of the structure.
Hand of Straights
Divide cards into groups such that each group is a consecutive sequence. Return True if possible, False otherwise.
Medium Meta Google
5
Time
O(n log n)
Space
O(n)
Approach
Greedy + Sorted Map

Approach

Greedy: always start a new sequence with the smallest available card. If the next card doesn't exist, it's impossible. Count cards, then repeatedly form sequences starting from the minimum available card.

C# 
public class Solution {
    public bool IsStrightHand(int[] hand, int groupSize) {
        if (hand.Length % groupSize != 0) return false;

        var counts = new SortedDictionary<int, int>();
        foreach (var card in hand) {
            if (!counts.ContainsKey(card)) counts[card] = 0;
            counts[card]++;
        }

        while (counts.Count > 0) {
            int firstCard = counts.Keys.Min();

            for (int i = 0; i < groupSize; i++) {
                int card = firstCard + i;
                if (!counts.ContainsKey(card)) return false;

                counts[card]--;
                if (counts[card] == 0) counts.Remove(card);
            }
        }

        return true;
    }
}
💡
Greedy Proof: Any valid solution can be transformed to use the minimum card in a group. By induction, our greedy approach is optimal.
🎯
FAANG Tip: Mention SortedDictionary (C#) / TreeMap (Java) for efficient min extraction. Or sort once and use a pointer.
Subsets
Generate all subsets of a given array (no duplicates). Return list of all subsets.
Medium Amazon Apple Google
6
Time
O(n·2^n)
Space
O(2^n)
Approach
Backtracking

Approach

For each element, decide: include it or skip it. Backtrack explores both choices. No pruning needed; explore all 2^n subsets. Alternative: iterative approach (build subsets incrementally).

C# 
public class Solution {
    public IList<IList<int>> Subsets(int[] nums) {
        var result = new List<IList<int>>();
        Backtrack(0, nums, new List<int>(), result);
        return result;
    }

    private void Backtrack(int idx, int[] nums, List<int> curr, List<IList<int>> result) {
        // Every subset is valid
        result.Add(new List<int>(curr));

        // Try including each remaining element
        for (int i = idx; i < nums.Length; i++) {
            curr.Add(nums[i]);
            Backtrack(i + 1, nums, curr, result);
            curr.RemoveAt(curr.Count - 1);
        }
    }
}
💡
Iterative Alternative: Start with [[]]. For each number, duplicate all current subsets and add the number to each copy. This builds 2^n subsets bottom-up.
🎯
FAANG Tip: Show both recursive (backtracking) and iterative approaches. Demonstrates flexibility and deep understanding.
Combination Sum
Find all unique combinations that sum to target. Elements can be reused. Return all combinations.
Medium Amazon Google Meta
7
Time
O(N^(T/M))
Space
O(T/M)
Approach
Backtracking + Pruning

Approach

Backtrack with pruning: for each candidate, decide how many times to use it (0, 1, 2, ...). Prune if current sum > target. Early termination when sum == target (found valid combo).

C# 
public class Solution {
    public IList<IList<int>> CombinationSum(int[] candidates, int target) {
        var result = new List<IList<int>>();
        Array.Sort(candidates);
        Backtrack(0, candidates, target, new List<int>(), result);
        return result;
    }

    private void Backtrack(int idx, int[] candidates, int remaining,
                                List<int> curr, List<IList<int>> result) {
        // Pruning: if remaining is negative, backtrack
        if (remaining < 0) return;

        // Found a valid combination
        if (remaining == 0) {
            result.Add(new List<int>(curr));
            return;
        }

        // Try each candidate (can reuse)
        for (int i = idx; i < candidates.Length; i++) {
            // Pruning: candidates are sorted
            if (candidates[i] > remaining) break;

            curr.Add(candidates[i]);
            // Reuse same element: pass i, not i+1
            Backtrack(i, candidates, remaining - candidates[i], curr, result);
            curr.RemoveAt(curr.Count - 1);
        }
    }
}
❌
Common Mistake: Passing i+1 instead of i. Elements CAN be reused, so pass i to allow re-selection of the same element.
🎯
FAANG Tip: Ask clarifying questions: "Can elements repeat?" and "Can each element be used once or multiple times?" Candidates with duplicates require different handling.
Permutations
Generate all permutations of a list of distinct integers. Return list of all permutations.
Medium Amazon Apple Google
8
Time
O(n·n!)
Space
O(n!)
Approach
Backtracking + Used Set

Approach

Use a "used" set to track which elements are in the current permutation. For each position, try all unused elements. Once we've placed n elements, record the permutation.

C# 
public class Solution {
    public IList<IList<int>> Permute(int[] nums) {
        var result = new List<IList<int>>();
        Backtrack(nums, new List<int>(), new HashSet<int>(), result);
        return result;
    }

    private void Backtrack(int[] nums, List<int> curr, HashSet<int> used,
                                List<IList<int>> result) {
        // All elements placed
        if (curr.Count == nums.Length) {
            result.Add(new List<int>(curr));
            return;
        }

        // Try each unused element
        for (int i = 0; i < nums.Length; i++) {
            if (used.Contains(nums[i])) continue;

            curr.Add(nums[i]);
            used.Add(nums[i]);
            Backtrack(nums, curr, used, result);
            curr.RemoveAt(curr.Count - 1);
            used.Remove(nums[i]);
        }
    }
}
💡
Swap Alternative: Swap element at idx with each subsequent position (in-place). Avoids HashSet overhead. Less intuitive but faster.
🎯
FAANG Tip: Mention both approaches: HashSet-based (clear logic) and swap-based (optimal). Interviewers appreciate knowing trade-offs.
Word Search
Search for a word in a 2D grid. Word is formed by adjacent cells (up, down, left, right). Return True if found, False otherwise.
Medium Amazon Microsoft Google
9
Time
O(m·n·4^L)
Space
O(L)
Approach
DFS Backtracking on Grid

Approach

For each cell, start a DFS search. Mark visited cells to avoid revisiting in the same path. Backtrack by unmarking. If we match all characters of the word, return True. Prune early if character doesn't match.

C# 
public class Solution {
    public bool Exist(char[][] board, string word) {
        for (int i = 0; i < board.Length; i++) {
            for (int j = 0; j < board[0].Length; j++) {
                if (board[i][j] == word[0] && DFS(board, word, i, j, 0))
                    return true;
            }
        }
        return false;
    }

    private bool DFS(char[][] board, string word, int r, int c, int idx) {
        // Matched entire word
        if (idx == word.Length) return true;

        // Out of bounds or wrong character
        if (r < 0 || r >= board.Length || c < 0 || c >= board[0].Length) return false;
        if (board[r][c] != word[idx]) return false;
        if (board[r][c] == '#') return false; // Already visited

        // Mark as visited
        char orig = board[r][c];
        board[r][c] = '#';

        // Explore 4 directions
        bool found =
            DFS(board, word, r + 1, c, idx + 1) ||
            DFS(board, word, r - 1, c, idx + 1) ||
            DFS(board, word, r, c + 1, idx + 1) ||
            DFS(board, word, r, c - 1, idx + 1);

        // Backtrack: restore cell
        board[r][c] = orig;

        return found;
    }
}
⚠
Key Detail: Mark visited during recursion, then restore on backtrack. This prevents revisiting the same cell in the same path but allows sharing cells across different search paths.
🎯
FAANG Tip: Mention pruning: check if the cell is the starting position OR matches the word[idx] before exploring neighbors. This reduces unnecessary recursive calls.
N-Queens
Place n queens on an n×n chessboard such that no two queens attack each other. Return all distinct solutions.
Hard Amazon Google
10
Time
O(N!)
Space
O(N^2)
Approach
Backtracking + Constraint Tracking

Approach

Place queens row-by-row. For each row, try each column. Use sets to track occupied columns, diagonals (r - c), and anti-diagonals (r + c). Prune branches where a queen conflicts. When all rows are filled, record the solution.

C# 
public class Solution {
    public IList<IList<string>> SolveNQueens(int n) {
        var result = new List<IList<string>>();
        var cols = new HashSet<int>();
        var diag1 = new HashSet<int>(); // r - c
        var diag2 = new HashSet<int>(); // r + c
        Backtrack(0, n, new List<string>(), result, cols, diag1, diag2);
        return result;
    }

    private void Backtrack(int row, int n, List<string> board,
                                List<IList<string>> result, HashSet<int> cols,
                                HashSet<int> diag1, HashSet<int> diag2) {
        if (row == n) {
            result.Add(new List<string>(board));
            return;
        }

        // Try each column in this row
        for (int col = 0; col < n; col++) {
            // Pruning: check if column or diagonals are occupied
            if (cols.Contains(col) || diag1.Contains(row - col) ||
                diag2.Contains(row + col)) continue;

            // Place queen
            var rowStr = new StringBuilder();
            for (int c = 0; c < n; c++)
                rowStr.Append(c == col ? 'Q' : '.');

            board.Add(rowStr.ToString());
            cols.Add(col);
            diag1.Add(row - col);
            diag2.Add(row + col);

            Backtrack(row + 1, n, board, result, cols, diag1, diag2);

            // Backtrack
            board.RemoveAt(board.Count - 1);
            cols.Remove(col);
            diag1.Remove(row - col);
            diag2.Remove(row + col);
        }
    }
}
💡
Diagonal Encoding: Top-left to bottom-right diagonal has constant (r - c). Top-right to bottom-left has constant (r + c). These encode all cells on each diagonal uniquely.
🎯
FAANG Tip: This is a classic backtracking problem. Emphasize the pruning strategy (tracking constraints with sets). Mention complexity: N! solutions, each board takes O(N^2) to construct.
Sudoku Solver
Fill a 9×9 Sudoku grid such that each row, column, and 3×3 box contains digits 1–9 exactly once.
Hard Amazon Google
11
Time
O(9^N)
Space
O(N)
Approach
Backtracking + Constraint Sets

Approach

Fill cells one by one (left-to-right, top-to-bottom). For each empty cell, try digits 1–9. Check if the digit is valid (not in row, column, or 3×3 box). Prune invalid choices early. Use constraint sets to track used digits in each row, column, and box.

C# 
public class Solution {
    public void SolveSudoku(char[][] board) {
        var rows = new HashSet<char>[9];
        var cols = new HashSet<char>[9];
        var boxes = new HashSet<char>[9];

        // Initialize constraint sets
        for (int i = 0; i < 9; i++) {
            rows[i] = new HashSet<char>();
            cols[i] = new HashSet<char>();
            boxes[i] = new HashSet<char>();
        }

        for (int i = 0; i < 9; i++) {
            for (int j = 0; j < 9; j++) {
                if (board[i][j] != '.') {
                    char c = board[i][j];
                    rows[i].Add(c);
                    cols[j].Add(c);
                    boxes[(i / 3) * 3 + j / 3].Add(c);
                }
            }
        }

        Backtrack(board, rows, cols, boxes);
    }

    private bool Backtrack(char[][] board, HashSet<char>[] rows,
                                 HashSet<char>[] cols, HashSet<char>[] boxes) {
        // Find next empty cell
        for (int i = 0; i < 9; i++) {
            for (int j = 0; j < 9; j++) {
                if (board[i][j] != '.') continue;

                // Try digits 1-9
                for (char c = '1'; c <= '9'; c++) {
                    int boxIdx = (i / 3) * 3 + j / 3;

                    // Pruning: check if digit is valid
                    if (rows[i].Contains(c) || cols[j].Contains(c) ||
                        boxes[boxIdx].Contains(c)) continue;

                    // Place digit
                    board[i][j] = c;
                    rows[i].Add(c);
                    cols[j].Add(c);
                    boxes[boxIdx].Add(c);

                    if (Backtrack(board, rows, cols, boxes)) return true;

                    // Backtrack
                    board[i][j] = '.';
                    rows[i].Remove(c);
                    cols[j].Remove(c);
                    boxes[boxIdx].Remove(c);
                }

                return false; // No valid digit found for this cell
            }
        }

        return true; // All cells filled
    }
}
⚠
Box Index: The 3×3 box containing (i, j) is at index (i / 3) * 3 + j / 3. This maps each cell to one of 9 boxes (0–8).
🎯
FAANG Tip: Mention optimization: maintain a list of empty cells and always pick the cell with the fewest valid candidates (minimum remaining values heuristic). This prunes exponentially faster.
Palindrome Partitioning
Partition a string into palindromic substrings. Return all possible partitions.
Medium Amazon Google Meta
12
Time
O(N·2^N)
Space
O(N^2)
Approach
Backtracking + DP Memoization

Approach

Backtrack through all partition points. For each starting position, try all ending positions. Prune if substring is not a palindrome. Use a 2D DP table to memoize palindrome checks (O(N^2) preprocessing), avoiding O(N) checks during backtracking.

C# 
public class Solution {
    public IList<IList<string>> Partition(string s) {
        // Precompute palindromes using DP
        bool[][] isPalin = new bool[s.Length][];
        for (int i = 0; i < s.Length; i++)
            isPalin[i] = new bool[s.Length];

        // Every single character is a palindrome
        for (int i = 0; i < s.Length; i++)
            isPalin[i][i] = true;

        // Check all substrings of length > 1
        for (int len = 2; len <= s.Length; len++) {
            for (int i = 0; i <= s.Length - len; i++) {
                int j = i + len - 1;
                if (s[i] == s[j] && (len == 2 || isPalin[i + 1][j - 1]))
                    isPalin[i][j] = true;
            }
        }

        var result = new List<IList<string>>();
        Backtrack(0, s, isPalin, new List<string>(), result);
        return result;
    }

    private void Backtrack(int idx, string s, bool[][] isPalin,
                                List<string> curr, List<IList<string>> result) {
        if (idx == s.Length) {
            result.Add(new List<string>(curr));
            return;
        }

        // Try all palindromic substrings starting at idx
        for (int end = idx; end < s.Length; end++) {
            // Pruning: skip if not a palindrome
            if (!isPalin[idx][end]) continue;

            curr.Add(s.Substring(idx, end - idx + 1));
            Backtrack(end + 1, s, isPalin, curr, result);
            curr.RemoveAt(curr.Count - 1);
        }
    }
}
💡
DP Optimization: Preprocessing palindromes in O(N^2) time saves O(N) per check during backtracking. Without it, checking if a substring is a palindrome costs O(N), making backtracking O(N^2 · 2^N). With memoization, it's O(N · 2^N).
🎯
FAANG Tip: Emphasize the DP + backtracking combination. This pattern appears in many hard problems. Show understanding of when to use memoization to optimize backtracking.
