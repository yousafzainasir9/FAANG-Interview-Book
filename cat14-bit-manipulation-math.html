INTRO

Bit Operations & Common Patterns

Bit manipulation is a low-level skill that reveals elegant solutions to problems that seem complex at first glance. Understanding these operations unlocks a whole category of interview problems.

Core Bit Operations in C#

& (AND): Both bits must be 1. Use to isolate bits or check flags.
| (OR): At least one bit is 1. Use to set bits.
^ (XOR): Bits differ. Use for toggling or finding unique elements.
~ (NOT): Flip all bits. Watch for sign extension in two's complement.
<< (Left Shift): Multiply by 2^n.
>> (Right Shift): Divide by 2^n (arithmetic, preserves sign).

Essential Bit Tricks

XOR Property
a ^ a = 0, a ^ 0 = a
Perfect for finding single unpaired elements or swapping without temp variables.
n & (n-1) = 0
Check Power of Two
Removes the lowest set bit. Repeat until n becomes 0 to count all set bits.
Count Set Bits
Brian Kernighan's Algorithm
while (n > 0) { count++; n &= n - 1; } — O(number of set bits)

Math Patterns

GCD & Modular Arithmetic: Use Euclidean algorithm (GCD via subtraction/modulo) for divisibility problems. Modular arithmetic prevents overflow in range queries.
Overflow Detection: Check if a + b overflows before summing. In two's complement, overflow occurs when signs differ after operation.
Fast Exponentiation: Binary exponentiation reduces O(n) to O(log n) by leveraging bit structure of the exponent.

Single Number

Given a non-empty array where every element appears twice except one, find that single element in O(1) space and O(n) time.

Easy Google Apple
1

Problem Statement

Input: [2, 2, 1, 1, 4]
Output: 4

Approach: XOR Property

1
Recall that a ^ a = 0 and a ^ 0 = a.
2
XOR is commutative and associative, so paired elements cancel out.
3
XOR all elements; result is the unpaired number.
Time
O(n)
Space
O(1)

Solution

csharp 
public int SingleNumber(int[] nums)
{
    int result = 0;
    foreach (int num in nums)
    {
        result ^= num;
    }
    return result;
}

Complexity & Edge Cases

Time: O(n)
Single pass through the array, XOR operation is constant time per element.
Space: O(1)
Only one variable result, no additional data structures.
💡
FAANG Tip: Interviewers love this problem because it tests understanding of bit operations beyond basic counting. Mention why XOR works: it leverages the mathematical property of self-cancellation in even occurrences.

Number of 1 Bits

Count the number of set bits (1s) in the binary representation of an unsigned integer.

Easy Facebook Amazon
2

Problem Statement

Input: 11 (binary: 1011)
Output: 3

Approach: Brian Kernighan's Algorithm

1
The operation n & (n-1) removes the rightmost set bit.
2
Repeat until n becomes 0, counting iterations.
3
This is more efficient than checking every bit position.
Time
O(k) k=set bits
Space
O(1)

Solution

csharp 
public int HammingWeight(uint n)
{
    int count = 0;
    while (n > 0)
    {
        count++;
        n &= n - 1;  // Remove rightmost set bit
    }
    return count;
}

Complexity & Edge Cases

Time: O(k)
Where k is the number of set bits. Best case O(1) if no bits, worst case O(log n) with all bits.
Space: O(1)
No extra space beyond counters.
💡
FAANG Tip: Brian Kernighan's algorithm is faster than bit-by-bit checking for sparse bit patterns. Show you understand why n-1 removes the lowest bit (borrow propagation in two's complement).

Counting Bits

For every number i from 0 to n, count set bits. Return an array of results.

Easy Apple Amazon
3

Problem Statement

Input: n = 5
Output: [0, 1, 1, 2, 1, 2] (counts for 0, 1, 2, 3, 4, 5)

Approach: DP + Bit Pattern

1
Observe: dp[i] = dp[i >> 1] + (i & 1).
2
Right shift removes the last bit; (i & 1) checks if it was set.
3
Build array incrementally, reusing previously computed counts.
Time
O(n)
Space
O(n)

Solution

csharp 
public int[] CountBits(int n)
{
    int[] dp = new int[n + 1];
    for (int i = 1; i <= n; i++)
    {
        dp[i] = dp[i >> 1] + (i & 1);  // Count of i = count of i/2 + last bit
    }
    return dp;
}

Complexity & Edge Cases

Time: O(n)
Single pass from 1 to n, each computation is O(1).
Space: O(n)
Output array of size n+1; not counting this as extra space in some contexts.
💡
FAANG Tip: This is a classic DP + bit manipulation hybrid. The key insight is recognizing the relationship between i and i>>1 in the bit representation.

Reverse Bits

Reverse the bits of a 32-bit unsigned integer.

Easy Microsoft Apple
4

Problem Statement

Input: 00000010100101000001111010011100
Output: 00111001011110000010100101000000

Approach: Bit Shifting

1
Extract the rightmost bit of input: n & 1.
2
Shift result left and add the extracted bit.
3
Shift input right and repeat 32 times.
Time
O(1)
Space
O(1)

Solution

csharp 
public uint ReverseBits(uint n)
{
    uint result = 0;
    for (int i = 0; i < 32; i++)
    {
        result = (result << 1) | (n & 1);  // Add rightmost bit
        n >>= 1;  // Shift input right
    }
    return result;
}

Complexity & Edge Cases

Time: O(1)
Always exactly 32 iterations for a 32-bit integer, constant work per iteration.
Space: O(1)
Only integer variables, no extra structures.
💡
FAANG Tip: Understand the loop must run exactly 32 times; you can't optimize away early even if n becomes 0, because leading zeros matter in bit reversal.

Missing Number

Given an array containing n distinct numbers from 0 to n, find the missing one in O(n) time and O(1) space.

Easy Google LinkedIn
5

Problem Statement

Input: [3, 0, 1]
Output: 2

Approach: XOR or Gauss Sum

1
XOR approach: XOR all array elements with all numbers 0 to n.
2
Paired numbers cancel; missing number remains.
3
Alternatively, use Gauss formula: expected sum - actual sum = missing.
Time
O(n)
Space
O(1)

Solution (XOR)

csharp 
public int MissingNumber(int[] nums)
{
    int result = nums.Length;  // Start with n
    for (int i = 0; i < nums.Length; i++)
    {
        result ^= i ^ nums[i];  // XOR with index and value
    }
    return result;
}

Complexity & Edge Cases

Time: O(n)
Single pass through array.
Space: O(1)
Only a result variable, no extra storage.
ℹ️
Alternative (Gauss): sum = n*(n+1)/2; return sum - array.Sum(). Simple but risks integer overflow in languages like Java/C# with large n (use long casting).
💡
FAANG Tip: The XOR approach avoids overflow entirely. Mention both and explain trade-offs in your interview.

Sum of Two Integers

Calculate the sum of two integers without using + or - operators.

Medium Google Facebook
6

Problem Statement

Input: a = 1, b = 2
Output: 3

Approach: Bit-by-Bit Addition

1
XOR gives sum without carry: a ^ b.
2
AND + left shift gives carry: (a & b) << 1.
3
Repeat until carry is 0. Handle negative numbers in two's complement.
Time
O(1)
Space
O(1)

Solution

csharp 
public int GetSum(int a, int b)
{
    while (b != 0)
    {
        int carry = (a & b) << 1;  // Compute carry
        a ^= b;  // a becomes XOR (sum without carry)
        b = carry;  // b becomes carry for next iteration
    }
    return a;
}

Complexity & Edge Cases

Time: O(1)
Fixed 32 iterations for 32-bit integers in worst case.
Space: O(1)
Only integer variables.
⚠️
Negative Numbers: Two's complement handles negatives naturally. The loop terminates when carry becomes 0. For overflow, C# integers wrap, making this work correctly.
💡
FAANG Tip: This is an elegant problem that tests deep bit understanding. Explain the algorithm clearly: XOR for sum, AND for detecting carry positions.

Power of Two

Determine if a given integer is a power of two.

Easy Google Amazon
7

Problem Statement

Input: n = 16
Output: true (16 = 2^4)

Approach: n & (n-1) == 0

1
Powers of two have exactly one bit set: 1, 2, 4, 8, 16... (binary: 1, 10, 100, 1000, 10000...).
2
Subtracting 1 from a power of two flips all bits after the single set bit.
3
AND of n and n-1 is 0 if and only if n is a power of two.
Time
O(1)
Space
O(1)

Solution

csharp 
public bool IsPowerOfTwo(int n)
{
    return n > 0 && (n & (n - 1)) == 0;
}

Complexity & Edge Cases

Time: O(1)
Single bitwise operation.
Space: O(1)
No extra storage.
⚠️
Edge Case: n must be > 0. Zero and negative numbers are not powers of two. Always check n > 0 first.
💡
FAANG Tip: This one-liner is deceptively powerful. Understand why n-1 flips the bits and why AND with 0 reveals a power of two. Explain with examples (e.g., 8 & 7 = 1000 & 0111 = 0).

Pow(x, n)

Compute x raised to the power n in O(log n) time. Handle negative exponents and overflow.

Medium Microsoft Apple
8

Problem Statement

Input: x = 2.0, n = 10
Output: 1024.0
Input: x = 2.0, n = -2
Output: 0.25

Approach: Fast Exponentiation (Binary Exponentiation)

1
Use the binary representation of n. If n is even, x^n = (x^2)^(n/2).
2
If n is odd, x^n = x * x^(n-1).
3
For negative n, compute x^(-n) then take reciprocal. Handle n = int.MinValue specially.
Time
O(log n)
Space
O(log n)

Solution

csharp 
public double MyPow(double x, int n)
{
    if (n == 0) return 1.0;

    long N = n;
    if (N < 0)
    {
        x = 1 / x;
        N = -N;
    }

    return FastPow(x, N);
}

private double FastPow(double x, long n)
{
    if (n == 0) return 1.0;

    double half = FastPow(x, n / 2);
    if (n % 2 == 0)
        return half * half;
    else
        return half * half * x;
}

Complexity & Edge Cases

Time: O(log n)
Exponent halves at each step, resulting in log n recursion depth.
Space: O(log n)
Recursion stack depth proportional to log n.
⚠️
int.MinValue Overflow: -int.MinValue overflows to int.MinValue. Cast n to long before negating: N = -N ensures correctness.
💡
FAANG Tip: Mention you can also implement iteratively by reading n's bits. Recursive is cleaner; iterative avoids stack. Both are O(log n).

Happy Number

A happy number reaches 1 by repeatedly summing the squares of its digits. Detect if a number is happy using Floyd's cycle.

Easy LinkedIn Amazon
9

Problem Statement

Input: n = 19
Output: true
Trace: 19 → 1+81=82 → 64+4=68 → 36+64=100 → 1+0+0=1 ✓

Approach: Floyd's Cycle Detection (Tortoise & Hare)

1
If a number is not happy, it enters a cycle.
2
Use slow and fast pointers: slow moves one step, fast moves two steps.
3
If they meet, there's a cycle (unhappy). If slow reaches 1, it's happy.
Time
O(k)
Space
O(1)

Solution

csharp 
public bool IsHappy(int n)
{
    int slow = n, fast = n;
    do
    {
        slow = SumOfSquares(slow);
        fast = SumOfSquares(SumOfSquares(fast));
    } while (slow != fast);

    return slow == 1;
}

private int SumOfSquares(int n)
{
    int sum = 0;
    while (n > 0)
    {
        int digit = n % 10;
        sum += digit * digit;
        n /= 10;
    }
    return sum;
}

Complexity & Edge Cases

Time: O(k)
k depends on the number; typically converges quickly. Worst case is proportional to log(n) iterations.
Space: O(1)
Only pointers and temporary variables, no extra structures.
ℹ️
Alternative: Use a HashSet to detect cycles: store visited sums. If you revisit, there's a cycle. Slightly easier to understand but uses O(k) space.
💡
FAANG Tip: Demonstrate understanding of cycle detection algorithms. Floyd's method is elegant and space-efficient; know when to use it vs. HashSet.

Reverse Integer

Reverse the digits of a 32-bit integer. Return 0 if overflow occurs.

Medium Google Microsoft
10

Problem Statement

Input: x = 123
Output: 321
Input: x = -123
Output: -321
Input: x = 120
Output: 21

Approach: Digit Extraction & Overflow Detection

1
Extract digits from right to left: digit = x % 10, x /= 10.
2
Build reversed result: result = result * 10 + digit.
3
Before each multiplication, check for overflow: if result > int.MaxValue / 10 or (== int.MaxValue / 10 and digit > 7), return 0.
Time
O(log x)
Space
O(1)

Solution

csharp 
public int Reverse(int x)
{
    int result = 0;
    while (x != 0)
    {
        int digit = x % 10;
        x /= 10;

        // Check overflow before multiplication
        if (result > int.MaxValue / 10 ||
            (result == int.MaxValue / 10 && digit > 7))
            return 0;
        if (result < int.MinValue / 10 ||
            (result == int.MinValue / 10 && digit < -8))
            return 0;

        result = result * 10 + digit;
    }
    return result;
}

Complexity & Edge Cases

Time: O(log x)
Number of digits in x, loop runs once per digit.
Space: O(1)
Only integer variables.
⚠️
Overflow Cases: int.MaxValue = 2147483647, int.MinValue = -2147483648. Check before each step: (int.MaxValue / 10 = 214748364, and 7 is the max last digit for positive; -2147483648 / 10 = -214748364, and -8 for negative).
💡
FAANG Tip: This problem is a favorite for testing overflow awareness. Show you understand the exact conditions for overflow. Many candidates skip or get the boundary cases wrong—get them right to stand out.
